## ゲーム概要

### 基本コンセプト

未解明の古代遺跡に隊を率いて潜り、お宝を持ち帰るカスタマイズ重視のローグライク。
プレイヤーはリーダーとして先頭で探索し、隊員はポリシーに基づいて自動で作業する。
リーダーが未知の領域を切り開き、隊員が探索済みエリアで戦闘・回収・運搬をこなす。

### ジャンル・特徴
- **ジャンル**: ローグライク・隊マネジメントRPG（ターンベース）
- **視点**: 1人称レイキャスティング（3D風2D）。リーダーの視点で探索する
- **HUD**: ミニマップで隊員・敵の位置を表示。隊員HP一覧で状態を常時把握する
- **探索**: グリッドベース・ターン制移動
- **戦闘**: 同一マップ上でのターンベース統合戦闘。隊員が自律的に戦い、リーダーはアイテムで支援する
- **成長**: 隊員ごとの使用ベーススキル成長（永続） + ランを跨ぐ身体改造の蓄積
- **マネジメント**: 隊の編成、装備配分、指示、撤退判断

### 参考作品
- **RimWorld**: 複数キャラの間接操作、使用ベースのスキル成長、キャラロスの重み
- **Elona**: 使用ベースのスキル成長、大量のパラメータが育つ楽しさ
- **Darkest Dungeon**: 隊員の雇用・育成・喪失のサイクル

## 核心的な面白さ

### 3つの柱

#### 1. 隊のマネジメント
- 2〜10人の隊員を雇用・育成・改造する
- 軽量な使い捨て偵察要員と、重投資した精鋭を組み合わせる
- 隊員の死は永続。育てた隊員とサイボーグ投資を失う緊張感
- 何人連れていくか自体がビルド判断になる。多ければ維持コストが重い

#### 2. カスタマイズ
- 隊員ごとに身体改造（内蔵パーツ）と装備（外付け）の2層でビルドを構築する
- 内蔵パーツは電力を共有する。全部入れたいが動力が足りない、というトレードオフ
- 隊全体の役割分担を設計する。近接特化、射撃手、偵察要員、荷物持ちなど

#### 3. リスクとリターンの判断
- もう1階潜れば報酬が増えるが、隊員を失うかもしれない
- 隊員が負傷したとき、撤退するか続行するか
- リーダーが死んだらラン失敗。自身の安全と隊の前進を天秤にかける

## 隊編成システム

### リーダー（プレイヤー）

リーダーはプレイヤーが直接操作するキャラクター。未知の領域を先頭で切り開く探索者。

- 1人称視点でダンジョンを移動し、未探索エリアを開拓する
- 武器・防具を装備できるが、隊員より弱い。ステータス補正にペナルティがかかる
- アイテムを使って隊員を支援する（回復薬を投げる、バフアイテムなど）
- 隊員にポリシーと命令を設定する
- リーダーが死亡するとラン失敗。通貨獲得なし、バックパック内アイテム消滅。ストレージと生存隊員は保持される

### 隊員（NPC）

隊員はリーダーが設定したポリシーとAIに基づいて、探索済みエリアで自律的に行動する。

- リーダーが探索済みにしたエリアでのみ行動できる。未探索の暗闇には入らない
- ポリシーに基づいて戦闘・アイテム回収・運搬を自動でこなす
- それぞれ個別にスキルが使用ベースで成長する
- 身体改造・装備はリーダーがハブで事前に設定する
- 死亡は永続。サイボーグ投資・スキル蓄積ごと失われる

### 隊員の個性

隊員は雇用時にランダム生成される最低限の個性を持つ。

- **名前**: ランダム生成
- **初期属性**: STR/SEN/DEX/AGI/VIT/DEFがランダムに分配される
- **得意分野**: 初期属性の傾向から決まる（近接向き、射撃向き、器用タイプなど）
- **外見**: ランダム生成（ダンジョン内で1人称視点の隊員として描画される）

性格や人間関係のシステムはない。個性は能力面の差異と名前・外見で表現する。
長く使い込んだ隊員には育成の歴史が蓄積され、プレイヤーにとっての愛着が生まれる。

### 隊員の雇用

ハブの酒場でランダム生成された候補から選んで雇用する。

- 候補は名前、初期属性、得意分野がランダムに生成される
- 通貨を支払って雇用する
- 雇用後は個別に育成する（装備付与、身体改造、ダンジョンでのスキル成長）
- 酒場の候補はランごとに入れ替わる

### コスト構造

隊員は雇用コストと維持コストを持つ。

```
軽量NPC（安い傭兵・新人）
  - 雇用コスト: 低
  - 維持コスト: 低（食料少、電力少）
  - 能力: 低い。スキル0から育てる
  - 死んでも損害が小さい → 偵察・荷物持ち要員

重量NPC（改造済みベテラン）
  - 雇用コスト: 高（改造投資の累積）
  - 維持コスト: 高（食料多、高出力動力炉の燃料）
  - 能力: 高い。サイボーグ強化済み
  - 死ぬと大損害 → 慎重に運用する
```

維持コストはラン中に隊全体で食料と凍晶バッテリーを消費する形で発生する。
10人連れていくと維持コストで物資が尽きる。3人精鋭だけだと1人の死で崩壊する。

### エリア制限

隊員はリーダーが探索済みにしたエリアでのみ行動できる。

- リーダーが通過して視界に入れたマスが「探索済み」になる
- 隊員は探索済みマスの範囲内で自律行動する。未探索の暗闇には入らない
- リーダーが前線を押し広げるほど、隊員の行動範囲も広がる
- リーダーの役割は「どこに進むかを決めて未知の領域を切り開くこと」

### ポリシーシステム

隊員の自律行動はポリシーで制御する。ポリシーは事前に設定しておくルールであり、隊員はポリシーに従って自動で判断する。

#### 位置ポリシー

- **護衛**: リーダーから離れない。一定距離以内にとどまる
- **前衛**: リーダーの前方に展開し、敵と交戦する
- **巡回**: 探索済みエリア内を自律的に巡回する
- **待機**: その場にとどまる
- **撤退**: 出口に向かって移動する

#### 戦闘ポリシー

- **攻撃**: 視界内の敵を攻撃する
- **回避**: 戦闘を避けて逃げる

#### アイテムポリシー

- **回収**: 探索済みエリアのアイテムを自動で拾う
- **無視**: アイテムを拾わない
- フィルタ条件（武器だけ拾う、素材だけ拾う等）を設定できる

#### アイテム処理ポリシー

- **保持**: 拾ったアイテムを持ち続ける
- **分配**: 運搬役の隊員にアイテムを渡す
- **分解**: 拾ったアイテムをその場で分解して素材にする

複数のポリシーを組み合わせて隊員の役割を定義する（例: 「前衛」+「攻撃」+「無視」= 戦闘特化、「巡回」+「回避」+「回収」+「分配」= 回収要員）。
隊員はポリシーの範囲内で移動・攻撃・回収をAIで判断する。行動はシンプルで予測可能にする。

### 直接命令

ポリシーに加えて、リーダーはダンジョン内で隊員に直接命令を出せる。

- 攻撃対象の指定
- 撤退指示
- 特定位置への移動指示

直接命令はポリシーより優先される。命令を実行し終えるとポリシーに戻る。

## レベル構造

### 大域移動: ハブ型

```
           [廃坑 10F]
         /
[ハブ] ──┼─ [研究所 15F]
         \
           [深淵 20F]
          ...
```

- ハブから解禁済みのダンジョンに出発する
- ダンジョンの数と深さはダンジョンごとに異なる
- ダンジョンをクリアすると次のダンジョンが解禁される
- ハブで買い物、身体改造、隊編成、アイテム配分をしてから出発する

### ダンジョン設計原則

- ダンジョンごとに環境ギミック、敵種、有効な戦術が異なる
- ダンジョン数と深さはデータ追加で拡張できる
- 最深部にボスがいる

### ダンジョン内: 階層選択

```
[1F]
  ↓ 選択肢提示
 ┌┴┐
[2A][2B]  ← どちらかを選ぶ
  ↓
 ...
  ↓
[最深部 ボス]
```

- 各階層クリア後、次の階層の選択肢が提示される
- 選んだ方に進む（戻れない）
- 選ばなかった方は行けない
- 最深部でボス戦

### 小域探索

各階層内での自由探索。

- グリッドベースのマップ
- ターン制移動
- リソース収集、隊員による戦闘

## ランサイクル

「ラン」はハブを出発してからダンジョンを終了するまでの1回の探索を指す。
装備・アイテムは全て持ち帰れる。隊員のスキル値・身体改造・装備は永続する。

### ランの流れ

```
1. ハブにスポーン（リーダー+生存隊員+前回の持ち物を所持した状態）
      ↓
2. 酒場で新しい隊員を雇用する
   店で装備・アイテムを購入し、隊員に配分する
   施術所で隊員の身体改造パーツを装着・交換する
      ↓
3. 隊の編成を決める（誰を連れていくか）
      ↓
4. ダンジョンを選択
      ↓
5. ダンジョンに入る（マップがランダム生成される）
      ↓
6. 階層を探索（リーダーが前線を切り開き、隊員がポリシーに基づいて戦闘・回収）
   リーダーは未探索エリアの開拓、アイテム支援、直接命令を行う
      ↓
7. 次の階層を選択
      ↓
8. 繰り返し...
      ↓
9. 最深部ボス撃破 → ボーナス通貨獲得、次のダンジョン解禁
   or 撤退 → 到達階層に応じた通貨を獲得
   or リーダー死亡 → ラン失敗（通貨獲得なし、バックパック内アイテム消滅）
   or 隊員全滅 → リーダーだけで脱出を試みる
      ↓
10. ハブに戻る（生存隊員・身体改造・装備中アイテムは保持）
```

### アイテムの持ち帰り

- ラン終了時、生存者の所持アイテムは全て持ち帰れる
- 売りたいものだけハブの店で売却する
- 残りはストレージに保管するかバックパックに入れたまま次のランに持ち込む
- 隊全体の重量制約が持ち帰り量の自然な上限になる
- 死亡した隊員の所持品は消滅する

### マップ生成

- ダンジョンに入るたびにマップが新規にランダム生成される
- 一度探索した階層に戻ることはできない
- クリア済みのダンジョンにも再挑戦できる

### 通貨システム

- ダンジョンクリア時にボーナス通貨を獲得する
- 撤退時は到達階層に応じた通貨を獲得する
- リーダー死亡時はラン失敗。通貨を獲得できず、バックパック内アイテムも消滅する。ただしハブのストレージと生存隊員は保持される
- ハブの店でアイテムを売却して通貨を得る
- 通貨で装備・アイテム・身体改造パーツ・隊員雇用に使う

### 永続解禁（将来実装）

通貨を使って段階的に解禁できる永続要素。プレイを続ける動機になる。

- **店売り商品の追加** — 購入可能な装備・アイテムの品揃えが増える
- **レシピ解禁** — 合成で作れるものが増える
- **ハブ施設の拡張** — 鍛冶場、図書館、施術所などの施設を建設する
- **ダンジョン情報の開示** — 未踏ダンジョンの環境・敵種のヒントを事前に見られる

### 永続する要素と一時的な要素

| 要素 | 永続性 | 説明 |
|------|--------|------|
| 通貨 | 永続 | ダンジョンクリア・撤退・売却で獲得する |
| 隊員 | 永続（死亡で永久喪失） | 酒場で雇用。死亡すると改造投資ごと失われる |
| 隊員の身体改造パーツ | 永続（隊員に紐づく） | 施術所で装着・交換する |
| ストレージ内アイテム | 永続 | ハブのストレージに保管したアイテム |
| ダンジョン解禁実績 | 永続 | クリアしたダンジョンの記録 |
| 隊員のスキル値 | 永続（隊員に紐づく） | 使用ベースで成長する。隊員の死亡で失われる |
| バックパック内アイテム | 条件付き | 生還すれば持ち帰れる。所持者の死亡で消滅 |
| ダンジョンのマップ | ラン内 | 入るたびに新規生成 |
| ダンジョン内の進行 | ラン内 | 撤退・死亡で1Fからやり直し |

## 身体改造システム

身体改造（サイボーグ化）がRUINSのコアメカニクス。隊員の身体にパーツを埋め込んで能力を拡張する。
リーダーは身体改造を受けられない。

### スロット構成

外付け装備と内蔵パーツの2層構造。隊員1人ごとに独立したスロットを持つ。

```
外付け（装備）          内蔵（身体改造）
┌──────────┐     ┌──────────┐
│ 頭: ヘルメット │     │ 脳: 演算装置   │
│ 胴: 鎧      │     │ 眼: 暗視レンズ  │
│ 腕: 篭手     │     │ 腕: 油圧シリンダ │
│ 脚: ブーツ    │     │ 脚: 跳躍装置   │
│ 武器: 剣     │     │ 胸: 動力炉     │
│ 装飾: 指輪    │     │ 背: 排熱機構   │
└──────────┘     └──────────┘
```

### 外付け装備と内蔵パーツの違い

| | 外付け（装備） | 内蔵（身体改造） |
|--|------------|-------------|
| 着脱 | いつでも自由 | ハブの施術所でのみ |
| 入手 | ラン中に拾う・店で買う | ラン中に拾う・店で買う |
| 死亡時 | 所持者が死亡すると消滅 | 所持者が死亡すると消滅 |
| 効果 | 防御・攻撃の基本性能 | 特殊能力・ステータス強化 |
| 電力 | 不要 | 動力炉の電力を消費する |

### 電力管理

内蔵パーツは電力を消費する。胸スロットの動力炉が電力を供給する。

- 動力炉の出力が全パーツの消費電力の上限
- 全部入れたいが電力が足りない、というトレードオフがビルドの核
- 凍晶が動力炉の燃料。ダンジョンで採取するか店で購入する
- 電力が尽きると内蔵パーツが全停止し、素の能力値だけで戦う
- 隊全体の電力消費が維持コストの一部になる

### 内蔵パーツの効果例

| スロット | パーツ例 | 効果 |
|---------|---------|------|
| 脳 | 演算装置 | Sensation+、ギミック事前察知 |
| 脳 | 翻訳モジュール | 碑文解読、情報収集 |
| 眼 | 暗視レンズ | 暗闇エリアで視界確保 |
| 眼 | 熱感知器 | 壁越しに敵を感知 |
| 腕 | 油圧シリンダ | Strength+、壁の破壊 |
| 腕 | 精密マニピュレータ | Dexterity+、錠前解除 |
| 脚 | 跳躍装置 | 1マス飛び越え、罠回避 |
| 脚 | 静音歩行 | 敵に気づかれにくい |
| 胸 | 小型動力炉 | 電力供給（低出力・軽量） |
| 胸 | 大型動力炉 | 電力供給（高出力・重い） |
| 背 | 排熱機構 | 寒冷ペナルティ軽減 |
| 背 | フィルター | 毒ガスエリア耐性 |

## ステージギミック

ダンジョンごとに環境ギミックが異なる。1つのギミックに対して複数の突破方法があり、どれが使えるかは隊の編成依存。突破できないギミックに遭遇したら撤退して隊を組み直す。

### ギミック一覧

| ギミック | 効果 | 突破方法の例 |
|---------|------|------------|
| 氷結エリア | 移動コスト増加、体温低下 | 排熱パーツ / 防寒装備 / 高Speed走破 |
| 水没エリア | 通行不可 | 浮遊パーツ / 排水バルブ操作 / 迂回路探索 |
| 毒ガス | 毎ターンHP減少 | フィルターパーツ / 解毒薬 / 高Speed走破 |
| 暗闇 | 視界1マス | 暗視パーツ / ライト装備 / 高Sensation |
| 電磁障害 | 内蔵パーツ一時停止 | シールドパーツ / パーツなしで素の能力で突破 |
| 封鎖通路 | 瓦礫で通行不可 | 油圧パーツで破壊 / 爆破アイテム / 迂回路 |
| 高温エリア | 体温上昇、装備劣化 | 冷却パーツ / 耐熱装備 / 高Speed走破 |
| 重力異常 | 重量ペナルティ倍増 | 軽量ビルド / 浮遊パーツ / アイテム破棄 |

### 設計原則

- 1つのギミックに最低3つの突破方法を用意する
- 「パーツで突破」「アイテムで突破」「ステータスで突破」の3系統
- どんな編成でも撤退は常に選択肢にある。詰みは発生しない
- 深層ほど複数のギミックが同時に出現する
- 隊の編成によって有効な突破方法が変わる。偵察要員に暗視パーツ、前衛に排熱パーツなど役割分担で対処する

## 装備システム

### 外付け装備カテゴリ

#### 戦闘装備
- 武器（近接・遠距離）— リーダーも装備可能だが性能にペナルティがかかる
- 防具（頭・胴・腕・脚）— リーダーも装備可能だが性能にペナルティがかかる

#### 探索装備
- ライト（暗闇対策）
- 防寒具（氷結対策）
- ガスマスク（毒ガス対策）

#### 消耗品
- 食料（空腹対策、隊全体の維持コスト）
- 凍晶バッテリー（電力補充、隊全体の維持コスト）
- 医療品（HP回復）— リーダーが隊員に使用する
- 解毒薬（毒ガス対策）— リーダーが隊員に使用する
- 爆破チャージ（封鎖通路対策）
- バフアイテム（一時的な能力強化）— リーダーが隊員に使用する

### 装備スロット

隊員のスロット:
- 頭部
- 胴体
- 腕部
- 脚部
- 武器（最大5枠）
- 装飾品

リーダーのスロット:
- 頭部、胴体、腕部、脚部、武器（最大5枠）、装飾品（隊員と同じスロット構成だが、装備の性能にペナルティがかかる）

## 戦闘システム

### 特徴
- **マップ統合**: 探索画面で直接戦闘実行
- **ターン制**: リーダー行動→隊員行動（AI）→敵行動
- **リーダーは前線探索者**: 弱い戦闘能力で前線を切り開く。アイテム支援と直接命令も行う
- **隊員はポリシー自律行動**: 探索済みエリア内でポリシーに従って戦闘・回収を自動で行う

### リーダーのターン行動

- 移動する（未探索エリアを開拓する）
- 攻撃する（隊員より弱いが戦闘は可能）
- アイテムを使用する（回復薬を隊員に投げる、バフアイテムを使うなど）
- 隊員に直接命令を出す（攻撃対象の指定、撤退指示など）
- 何もしない（隊員のAIに任せる）

### 隊員の戦闘AI

隊員はポリシーと直接命令に基づいてシンプルな行動を実行する。

- 「攻撃」ポリシー: 視界内の敵を攻撃する。直接命令で指定された敵がいれば優先する
- 「回避」ポリシー: 敵から距離を取る。攻撃しない
- 「護衛」ポリシー: リーダーの近くにとどまりつつ、近づく敵を迎撃する
- 「前衛」ポリシー: リーダーの前方で敵と交戦する
- HPが低下すると自発的に後退する
- 装備した武器の射程に応じて位置取りを行う
- 探索済みエリアの外には出ない

行動はシンプルで予測可能にする。隊員の動きを見て「なぜそう動いたか」が常にわかる状態を目指す。

### 敵の種類

#### 結晶生物
- 凍晶に飲まれた動物や探索者の成れの果て
- 浅い階層から出現

#### 古代機構
- 遺跡の防衛装置や暴走した機械
- 中層以降に出現

#### ボス
- 遺跡の最深部を支配する強大な存在
- ダンジョンごとに固有のボス

## スキルシステム

### 使用ベースの成長

RPG的な経験値・キャラクターレベルは存在しない。
隊員ごとにスキルが使用に応じて成長し、スキル値はランを跨いで永続する。
隊員が死亡するとその隊員のスキル蓄積は全て失われる。
全36スキルが7カテゴリに分類される。詳細は `docs/design/20260319_13.md` を参照。

隊員が自律的に行動するため、スキルは隊員の行動に応じて自然に育つ。
近接武器を持たせた隊員は刀剣スキルが伸び、暗所を偵察させた隊員は暗視スキルが伸びる。
長く生き延びた隊員ほどスキルが高く、失ったときの損害が大きい。

### 本による成長

本はダンジョン内でのみ発見できる。店では購入できない。
本を読むと一定量のスキル経験値を獲得できる。実戦では伸ばしにくい知識系スキルは本が主要な成長手段になる。
読書にはAPを消費し、読了した本は消滅する。
どの隊員にどの本を読ませるかがビルド判断になる。

### 隊員の初期傾向

隊員は雇用時にランダムな初期属性を持つ（隊員の個性を参照）。
属性値が高いほど対応スキルの成長が速いため、初期属性が隊員の得意分野を決める。
雇用時の候補を見て、隊に足りない役割を埋められる隊員を選ぶ。

## ターンベース探索システム

### 基本仕様
- **グリッド移動**: 1マス単位での移動
- **アクションコスト**: 全行動に時間コストを設定
- **ターン制**: リーダー行動→隊員行動→敵行動→環境変化
- **統合戦闘**: 探索マップ上で直接戦闘実行

### アクションコストシステム

**基本メカニズム:**
- **APの獲得**: 毎ターン開始時に「敏捷性」値分のAPを獲得
- **基本移動コスト**: 通常地形での1マス移動 = 100AP
- **ターン終了条件**: APが0以下になるとターン終了

**行動コスト例:**
```
基本行動:
- 移動（平地）: 100AP
- 移動（対角）: 142AP（√2倍）
- 通常攻撃: 100AP
- アイテム使用: 80-120AP
```

- 持ち帰り量は隊全体のWeightCapacityで決まる
- 身体改造の脚パーツでWeightCapacityを拡張できる

## サバイバルシステム

体温と食料はステージギミックの一種として機能する。全てのダンジョンで常に脅威になるのではなく、特定のダンジョンで環境ギミックとして登場する。

### 体温

氷結エリアや高温エリアで体温が変動する。対策なしで長時間滞在するとペナルティが発生する。

- 排熱/冷却パーツ、防寒/耐熱装備で対策する
- 高Speedで走破してダメージを最小化する方法もある
- 隊員ごとに対策が必要。全員分の耐寒装備を揃えるか、対策済みの隊員だけ先行させるか

### 食料管理

- 空腹度は時間経過で低下する
- 空腹状態ではSpeed低下ペナルティ
- 食料アイテムで回復する
- 隊員が多いほど食料消費が増える。維持コストの主要因

## 難易度設計

### ダンジョン別難易度

各ダンジョンは独自の環境ギミックと難易度を持つ。
階層が深くなるほどダンジョン内でも難易度が上昇する。

ダンジョンの具体的な構成（名称、階層数、環境ギミック、敵種）は別途定義する。

### 難易度の軸

| 軸 | 浅いダンジョン | 深いダンジョン |
|----|--------------|--------------|
| ギミック | 単一・穏やか | 複合・過酷 |
| 敵の強さ | 弱い | 強い |
| 必要編成 | 少人数でも可 | 役割分担した大人数が必要 |
| 突破方法 | 汎用的 | 特定パーツ・特定編成が求められる |
| 維持コスト | 低い | 大人数の維持が重荷になる |

### バランス方針

- 通貨を貯めて隊員を雇用・改造し、難しいダンジョンに挑むプレイが成立する
- 隊員の死亡は永続。無謀な挑戦には育成投資を失うリスクが伴う
- 撤退は常に選択肢にある。少しずつ稼ぐプレイも有効
- ギミックに対応できない編成で来てしまった場合は撤退して隊を組み直して再挑戦する
- 使い捨て偵察隊員でダンジョンの情報を集め、本命の精鋭隊で攻略する戦略も有効
