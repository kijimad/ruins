# 敵・アイテムスポーンに偏りを導入する

## 背景

従来の敵・アイテムのスポーンはマップ全体からランダムに位置を選択していた。廊下にも部屋にも均等にスポーンし、偏りがなかった。

多くのローグライクでは敵やアイテムは偏って配置され、かたまりやすい傾向がある。完全にランダムな分散配置はゲームプレイとしておもしろくない。

## 設計

### 配置戦略の選択

スポーンの偏りを実現する方法は複数ある。今回は部屋ベースを採用したが、マップタイプやゲームデザインの方向性によって異なる戦略が適する場合がある。

| 戦略 | 特徴 | 適するケース |
|------|------|-------------|
| 部屋ベース（今回採用） | 部屋を選び、その中にスポーン | 部屋構造が明確なダンジョン |
| Perlinノイズ確率マップ | ノイズで密度マップを生成 | 自然地形や広大なオープンマップ |
| ポアソンディスク分布 | 最小距離を保ちつつ自然な分布 | 均等だが規則的でない配置 |
| 固定スポーンポイント | テンプレートで位置を指定 | ボスフロアや演出重視のフロア |
| 部屋サイズ重み付け | 面積比でスポーン数を按分 | 大小混在するダンジョン |

### 現在の実装: 部屋ベーススポーン

スポーン処理を「まず部屋を選び、次にその部屋内でランダム位置を決定する」方式に変更した。

```
変更前: マップ全体からランダム座標 → スポーン可能か判定
変更後: 部屋をランダム選択 → その部屋内からランダム座標 → スポーン可能か判定
```

敵もアイテムも同じ方式を適用する。部屋がない場合（テンプレートマップなど）は従来の全体ランダム配置にフォールバックする。

### クラスタリングの仕組み

`selectRoom()` は部屋リストから均等にランダム選択する。これにより自然な偏りが生まれる:

- 同じ部屋が複数回選ばれると、その部屋にエンティティが集中する
- 小さい部屋が選ばれるとさらに密度が高くなる
- 一部の部屋は何も選ばれず空のまま残る

追加の重み付けやパラメータなしで、部屋数とエンティティ数の比率から自然にクラスタリングが発生する。

### 同種クラスタ（敵NPC）

敵NPCは部屋単位で同種クラスタを形成する。1つの部屋が選ばれると、まずその部屋に配置する敵の種類を重み付き抽選で1つ決定し、次にその種類の敵を1〜3体まとめて配置する。

```
部屋をランダム選択 → 敵の種類を1つ決定 → 同種の敵を1〜3体配置
```

これにより「スライムの巣」「ゴブリンの集落」のような自然な群れが発生する。アイテムは種類ごとのクラスタリングは行わず、部屋内にランダム配置する。

## コメント

- 洞窟やBigRoomなどRoomsが少数の巨大矩形になるケースでは、偏りの効果は薄い。こうしたマップタイプには別の戦略が適する可能性がある
- PlannerType単位で配置戦略を切り替える設計に発展させることもできる
