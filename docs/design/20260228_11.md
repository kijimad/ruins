# APシステム改善設計

## 概要

Cataclysm-DDA（CDDA）のAPシステムを参考に、ruinsのアクションポイントシステムを改善する設計。

## 現状の問題点

| 問題 | 現状 | CDDAの方式 |
|------|------|-----------|
| AP回復 | 毎ターン100にリセット | 毎ターン speed 分を加算 |
| 行動コスト | 固定値 | 地形・姿勢・状態で動的変化 |
| 複数行動 | 1ターン1行動 | moves >= cost なら連続行動可能 |
| 状態異常影響 | AGI低下のみ | 詳細なspeedペナルティ |
| 地形コスト | なし | 地形ごとに移動コスト設定 |

## 改善設計

### Phase 1: AP加算方式への変更

#### 1.1 概念の整理

```
speed: キャラクターの行動速度（毎ターンの AP 回復量）
  = base_speed(100) + speed_bonus

speed_bonus: 状態異常・装備・特性によるボーナス/ペナルティ
  = AGI * 3 + DEX * 1 + 状態異常ペナルティ + 装備ボーナス

move_cost: 各行動に必要なコスト
  = base_cost * 地形修正 * 状態修正

行動可否: AP.Current >= move_cost
```

#### 1.2 TurnBased コンポーネント変更

```go
// 現状
type TurnBased struct {
    AP gc.Pool  // Current, Max
}

// 変更後
type TurnBased struct {
    AP    gc.Pool // Current: 現在AP, Max: 上限（200など）
    Speed int     // 毎ターンの回復量
}
```

#### 1.3 AP回復処理の変更

```go
// 現状: RestoreAllActionPoints
actionPoints.AP.Current = maxAP
actionPoints.AP.Max = maxAP

// 変更後
speed := calculateSpeed(world, entity)
actionPoints.Speed = speed
actionPoints.AP.Current = min(actionPoints.AP.Current + speed, actionPoints.AP.Max)
```

#### 1.4 Speed計算

```go
func CalculateSpeed(world w.World, entity ecs.Entity) int {
    baseSpeed := 100

    // 属性ボーナス
    attrs := getAttributes(entity)
    attrBonus := attrs.Agility.Total * 2 + attrs.Dexterity.Total * 1

    // 状態異常ペナルティ
    statusPenalty := calculateStatusPenalty(world, entity)

    // 過積載ペナルティ
    carryPenalty := calculateCarryPenalty(world, entity)

    speed := baseSpeed + attrBonus + statusPenalty + carryPenalty

    // 最小値: 基本速度の25%
    minSpeed := baseSpeed / 4
    return max(speed, minSpeed)
}
```

### Phase 2: 状態異常のSpeed影響

#### 2.1 ペナルティ計算

```go
func calculateStatusPenalty(world w.World, entity ecs.Entity) int {
    penalty := 0

    // 空腹ペナルティ
    if hunger := getHunger(entity); hunger != nil {
        penalty += hungerSpeedPenalty(hunger.Current)
    }

    // 体温ペナルティ（低体温/高体温）
    if bt := getBodyTemperature(entity); bt != nil {
        penalty += temperatureSpeedPenalty(bt)
    }

    // 痛みペナルティ
    if pain := getPain(entity); pain > 10 {
        penalty += painSpeedPenalty(pain)
    }

    return penalty
}
```

#### 2.2 空腹ペナルティ（CDDAスタイル）

| 空腹度 | ペナルティ |
|--------|-----------|
| 0-1000 | 0% |
| 1000-2000 | -10% |
| 2000-3000 | -25% |
| 3000-4000 | -50% |
| 4000+ | -75% |

```go
func hungerSpeedPenalty(hunger int) int {
    switch {
    case hunger < 1000:
        return 0
    case hunger < 2000:
        return -10
    case hunger < 3000:
        return -25
    case hunger < 4000:
        return -50
    default:
        return -75
    }
}
```

#### 2.4 過積載ペナルティ

```go
func calculateCarryPenalty(world w.World, entity ecs.Entity) int {
    pools := getPools(entity)
    if pools == nil {
        return 0
    }

    // 最大重量を超えた分の25%をペナルティ
    if pools.Weight.Current > pools.Weight.Max {
        overweight := pools.Weight.Current - pools.Weight.Max
        penalty := (overweight * 25) / pools.Weight.Max
        return -min(penalty, 75) // 最大-75
    }
    return 0
}
```

#### 3.3 移動コスト取得

```go
func GetTileMoveCost(world w.World, x, y int) int {
    // タイルエンティティから MoveCost を取得
    var cost int = 100 // デフォルト

    world.Manager.Join(
        world.Components.GridElement,
        world.Components.TileInfo,
    ).Visit(func(entity ecs.Entity) {
        grid := getGridElement(entity)
        if grid.X == x && grid.Y == y {
            tileInfo := getTileInfo(entity)
            cost = tileInfo.MoveCost
        }
    })

    return cost
}
```

#### 3.4 MoveActivity での使用

func (ma *MoveActivity) Info() ActivityInfo {

// 実際のコストは実行時に計算
func (ma *MoveActivity) GetActualCost(world w.World, dest Position) int {
    baseCost := 100
    tileCost := GetTileMoveCost(world, dest.X, dest.Y)
    return baseCost * tileCost / 100

### Phase 4: APマイナス許容とターン自動経過（CDDA方式）

#### 4.1 基本仕様

CDDAと同様に、APが足りなくても行動を実行できる。
APはマイナスになることを許容し、0以上になるまで自動でターンが経過する。

```
行動可否: AP.Current >= 0 （マイナスでなければ行動可能）
行動実行: AP.Current -= cost （マイナスになっても良い）
ターン経過: AP.Current < 0 の間、自動でターンを進める
```

#### 4.2 行動ループ

```go
// PlayerTurn フェーズでの処理
case turns.PlayerTurn:
    // AP が 0 以上なら行動可能
    if turnBased.AP.Current >= 0 {
        // プレイヤー入力を待つ
    } else {
        // AP がマイナス、自動でAIターンへ
        turnManager.AdvanceToAITurn()
    }
```

#### 4.3 行動後のAP消費

```go
func consumeActionCost(world w.World, entity ecs.Entity, cost int) {
    turnBased := getTurnBased(entity)
    turnBased.AP.Current -= cost
    // APがマイナスになっても良い（次ターン以降で回復）
}
```

#### 4.4 例：冷えた状態での攻撃

```
Speed: 25（体温ペナルティで低下）
攻撃コスト: 100
現在AP: 0

1. プレイヤーが攻撃を選択
2. AP: 0 - 100 = -100（マイナスになる）
3. AP < 0 なので自動でターン経過
4. ターン終了時: AP: -100 + 25 = -75
5. 自動でターン経過（繰り返し）
6. 4ターン後: AP: -100 + 25*4 = 0
7. AP >= 0 なので再び行動可能
```
