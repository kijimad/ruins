# シームレスワールド Phase 8: 状態統合・ポータルの行き先保持・オーバーマップ中心化

## 背景

[20260719_63.md](./20260719_63.md) で共存方式のステージ機構を実装し、ダンジョンの階層往復が動く。全ステージのエンティティを同一 world に共存させ、`StageBound` で所属を、`Suspended` で稼働可否を表し、往復を `stage.SwapTo` に集約した。63.md は「遺跡は『隣のステージがオーバーワールド帯』という特殊ケースにすぎない」「往復はマーカー付け替えに還元」と述べた。

本 Phase はその論理を最後まで進め、**世界をオーバーマップ中心に一本化**する。到達したい姿:

- 状態は1つ。オーバーワールドも遺跡も同じ探索状態で扱う。
- ポータル、すなわち階段・入口が行き先を保持し、往復は「触れて行き先へ swapTo」に還元される。
- 街はオーバーマップ上に生成され、新規ゲームの開始点になる。
- 「戻れない通常ダンジョン」と、その起動口であるダンジョン選択・街からの起動を廃止する。`StageKindDungeon` を撤廃する。

これは [[project-stagekind-dungeon-transitional]] に記録した「StageKindDungeon は過渡的、将来オーバーワールド+遺跡へ寄せて撤廃」の実行である。

## 前提の再確認（設計が依拠する事実）

- **`OverworldState` は既に `DungeonState` を埋め込む薄いラッパー**。`OverworldState.Update` は `DungeonState.Update` へ委譲し、末尾で `updateFront` と `maybeShift`（帯シフト）を足すだけ。描画も DungeonState へ委譲。よって統合は「この2フックを DungeonState 側へ畳む」だけで済む。
- **共存方式の忠実復元はエンティティが serde されるから**であり、**State マシンのスタックは serde 対象でない**。ゆえにステージの継続情報はスタックに預けてはならない。ECS 側、コンポーネントとシングルトンに置く。前設計案の TransPush/Pop 退避はここで reload に破綻したため棄却する。
- **`findPortalPosition` は ActiveFilter で Suspended を除外**する。退避中ステージのポータルは検索に出ない。帰還は「入口を探す」でなく「行き先をポータルが保持」で解く。

## 要件

- 状態を統合する。オーバーワールドと遺跡フロアを単一の探索状態が扱う。挙動は現在ステージの種別で分岐する。
- ポータルは行き先 `StageKey` と行き先座標を保持する。触れると `stage.SwapTo(行き先)` し、行き先座標へプレイヤーを配置する。入口・上り階段・下り階段のすべてがこの1機構に還元される。
- 遺跡はオーバーワールドの入口から入り、上り階段でオーバーワールドへ戻る。戻り先座標は入口の座標で、入場時に遺跡の上り階段へ焼き込む。
- 街はオーバーマップの開始チャンクに生成し、新規ゲームはそこで始まる。
- `StageKindDungeon`・ダンジョン選択・街からのダンジョン起動を廃止する。ステージ種別は Overworld と Ruin の2つにする。
- すべてのステージは離れても保持され、徒歩で到達し直せる。セーブ・ロードで共存状態が復元される。

## 設計

### 1. 状態統合 — DungeonState に一本化

`OverworldState` を廃し、`DungeonState` に統合する。`updateFront`・`maybeShift`・帯生成・`restoreFromSave` を DungeonState へ移し、`CurrentStage.Kind == Overworld` のときだけ実行する。

```go
func (st *DungeonState) Update(world w.World) (...) {
    // 既存の探索更新（入力・ターン・視界・カメラ・HUD・死亡判定）
    ...
    if query.GetDungeon(world).CurrentStage.Kind == gc.StageKindOverworld {
        st.updateFront(world)
        st.maybeShift(world)
    }
}
```

帯ドライバ、ChunkGen と Band は DungeonState のフィールドとして持つ。enter/exit は同一 State 内の `swapTo`（TransNone）なので State は破棄されず、遺跡滞在中もドライバはメモリに残る。退避中の帯はシフトしないので、戻ったとき ECS とドライバが一致する。reload 時は `restoreFromSave` が SeamlessBand から再構築する。State 名は責務が広がるため将来 `FieldState` 等へ改名を検討するが、本 Phase では `DungeonState` のままにして差分を絞る。

### 2. ポータルの行き先保持 — 往復の一般化

ポータルを「行き先を持つステージ・グラフの辺」に一般化する。ポータルのエンティティに行き先を持たせる。

| ポータル | 行き先(StageKey) | 行き先座標 |
|---|---|---|
| オーバーワールドの遺跡入口 | 遺跡1階 | 遺跡の入口 |
| 遺跡の上り階段(最上階) | オーバーワールド | 入った入口の座標 |
| 下り階段 | 1つ深い階 | その階の入口 |
| 上り階段(中間階) | 1つ浅い階 | その階の下り階段 |

- ポータルに触れる → そのポータルの行き先 `StageKey` へ `stage.SwapTo` → 行き先座標へ `MovePlayerToPosition`。種別分岐なしの単一機構。
- 未訪問の行き先は SwapTo の generate で生成し、生成時に「戻りのポータル」へ戻り先座標を焼き込む。
- 行き先座標はポータルのコンポーネントに持つので serde 対象。reload を跨いで戻り先が保たれる。「どの入口から入ったか」を階段自身が覚える。
- これで `findPortalPosition` の先着ヒューリスティックを廃し、複数入口があっても曖昧さが無い。

### 3. 街をオーバーマップに生成

- 街はステージでなくオーバーワールドの地物。開始チャンク、`eastIndex==0` 付近に街の構造物、店・収納・隊員雇用などの prop を決定的に配置する。
- 新規ゲームは、キャラ作成後に街を含むオーバーワールドを生成し、街の位置でプレイヤーを開始する。`TownState` と、そこへ至る導線を廃止する。街の機能は overworld 上の prop への相互作用で提供する。

### 4. 破棄スコープ

一方向ダンジョンと街への完全離脱が無くなるため、「潜行を丸ごと捨てる」完全離脱が消える。全ステージは共存・徒歩往復可能で保持される。`DungeonState.OnStop` の全フィールド一括 purge を廃止する。world を捨てるのはタイトルへ戻る・ロードのときで、そこは world リセットが担う。ステージ単位の破棄が要る場合は `stage.Purge` をハンドラで明示的に呼ぶ。

### 5. StageBound タグ付け（前提）

- チャンク生成直後に `stage.Bind(world, gc.NewOverworldStage())` で帯エンティティを束縛する。
- `CurrentStage` は開始時に `NewOverworldStage()`。以降は swapTo が更新する。

## 採用しなかった方法

- **OverworldState を残し TransPush/Pop で遺跡へ出入り**: 前案。State スタックが serde 外なので、遺跡中セーブ→ロードで戻り先 State が消え TransPop が破綻する。共存方式の「継続は ECS に置く」原則に反するため棄却。
- **専用 RuinState を新設**: 遺跡専用状態。DungeonState と描画・往復・入力をまるごと重複させる。遺跡はダンジョンと構造が同じで、統合 State + ポータル行き先で足りるため不要。
- **帰還を入口 prop 探索で解く**: `findPortalPosition` で入口を探して戻る。入口が複数あると先着で誤る。ポータルが行き先座標を持つほうが曖昧さゼロで serde 安全。
- **街を専用ステージにする**: 街を Ruin と同様の別ステージにして入口から入る。だが「開始点」なので入場でなく初期在場が要り、オーバーマップ地物とするほうが素直。

## コメント

- 統合は `OverworldState` が既に `DungeonState` ラッパーなので小さい。まず挙動保存の統合から入り、その後にポータル一般化・遺跡・街生成・旧ダンジョン撤廃を重ねる。
- 撤廃(旧ダンジョン・選択・街起動)は代替(遺跡・街オーバーマップ化)が入ってから最後に行い、常にゲームが動く状態を保つ。
- 新たな不変条件: Update は `CurrentStage.Kind` で分岐する。swapTo が CurrentStage を更新するので、遷移中に古い種別で誤動作しないようテストで固める。
