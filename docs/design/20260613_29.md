# UIテーマ統一: 残ハードコード色の置換

## 背景

UIの色定数を `widgets/theme` パッケージに一元化する作業を進めてきた。`consts/colors.go` の色をすべて適切なパッケージに移動し、ebitenui系ウィジェットのハードコード色をtheme定数に置換した。しかしHUD系の手描画コードに `color.White` / `color.Black` が残っている。

## 設計

### 対象: `color.White` の置換（10箇所）

HUDテキスト描画で `color.White` が直接使われている。すべて `theme.TextPrimary` に置換する。

### 対象: `color.Black` の置換（1箇所）

### 対象外

- `hud/game_info.go` のHPバーグラデーション色（動的計算）
- `hud/minimap.go` のタイル色（データ駆動）
- `debug_overlay.go:103` の `whiteImg.Fill(color.White)` は `DrawTriangles` のソーステクスチャ用1x1白画像であり、頂点カラーで実際の色を制御するため純白のままにする
- `debug_overlay.go` の視界円の頂点カラー（緑、半透明0.3）はデバッグ専用で固有の値であるため対象外

### 施策2: 透明背景 `color.NRGBA{}` の定数化

`color.NRGBA{}` が ebitenui の透明背景として6箇所で使われている。`theme.Transparent` として定数化する。

| ファイル | 行 | 用途 |
|---|---|---|
| `styled/eui.go` | 193 | NewListContainer の背景 |
| `styled/eui.go` | 239 | NewListItemText の背景 |
| `styled/table.go` | 48 | NewTableRow の背景 |
| `styled/table.go` | 115 | NewTableHeaderRow の背景 |
| `styled/table.go` | 164 | NewSeparatorRow の背景 |
| `tabmenu/ui_builder.go` | 113 | 非フォーカスリストの背景 |

加えて `messagewindow/window.go:759` の `color.NRGBA{R:0,G:0,B:0,A:0}` も同じ透明色であり、`theme.Transparent` に統一する。

### 施策2 対象外

- `fadeout_animation.go:88` の `color.RGBA{0,0,0,uint8(alpha*255)}` は動的アルファのフェードアウト演出であり定数化不可
- 構造体フィールドの `color.RGBA` 型定義は色の値ではなく型宣言のため対象外

### 施策3-4 対象外

- HUD描画コードのピクセルオフセット（位置計算であり、UIスペーシングではない）
- `Spacing(0)` は意図的なゼロ間隔
- `MinSize`, `MaxHeight` 等のウィジェット固有サイズ
- `Spacing(5)`, `Bottom: 15` は各1箇所のみで定数化の効果が薄い。最も近いtheme定数に丸める

### 施策5: コード重複パターン（将来課題として記録）

以下は将来的にリファクタリングすべきだが、今回のスコープ外とする。

- **カテゴリタブ構築**: inventory/shop/craft/status で同じ「選択状態によるテキスト色切替+リストアイテム生成」パターンが4回繰り返されている
- **能力値テーブル構築**: equip_menu と status_state で同じ6能力値の表示ロジック
- **ページネーション付きリスト構築**: 4箇所で同パターン（ページテキスト+テーブル行生成+空リスト判定）
- **アクションウィンドウ構築**: 4箇所で同パターン（ウィンドウ生成+リストアイテム追加+中央配置）

## コメント

- `color.White` と `theme.TextPrimary` は微妙に異なる（`#ffffff` vs `#f5f5f5`）が、HUDテキストの視覚的統一のためtheme定数に揃える
- SpaceML(10), SpaceLL(20) の命名は既存スケール（XS=2, S=4, M=8, L=16, XL=24）の間を埋める形にする。12pxと15pxは各1箇所のみのため定数化しない
