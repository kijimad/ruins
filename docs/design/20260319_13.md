# スキル・職業システム設計

## 背景

現在の戦闘システムには以下の問題がある。

- Sensation属性が戦闘で未使用である。全武器のダメージがStrengthで計算されている
- AttackTypeにMELEE/RANGEDの区分があるのに、計算式が同一である
- 装備の重さに戦術的な選択がない。重装備のAGIペナルティを軽減する手段がない
- 職業を変えても属性が同じなら戦闘体験が変わらない

並列ダンジョン構成（ダンジョンごとに異なる環境・敵種）において、ラン内でのビルド構築を実現するためにスキルシステムが必要になる。

### 参考作品

- Elonaの使用ベースのスキル成長を参考にする
- CDDAの職業ごとのテーマに沿った装備セットを参考にする
- Vampire Survivorsの「大量のパラメータが成長・変動するのを見る楽しさ」を参考にする
- Slay the Spireの「毎ランゼロからビルドを組む」構造を参考にする

RPG的な経験値によるキャラクターレベルは導入しない。
成長は「スキルを使えばそのスキルが伸びる」方式で行う。

### 設計思想

スキルはアイテムではなくパラメータとして獲得・成長させる。
大量のスキルが並んでおり、それぞれの数字が使い込みで育っていくのを見ること自体が楽しさになる。
スキルはゲームの全システム（戦闘、温度、空腹、合成、悪魔モード、AP、重量、属性元素）に接続する。

### 永続性モデル

スキルは毎ラン完全リセットされる。ハブで職業選択や購入でき、ダンジョン内で使用と読書によって成長させる。

永続するのは通貨とダンジョン解禁実績のみ。通貨でハブの店から装備・アイテム・本を購入し、ランの初期ビルドをコントロールする。

この設計により:
- 毎ランのビルド判断が意味を持つ。ダンジョンの環境に合わせてスキルの育成方針を変える
- バランスが取りやすい。各ダンジョンの想定スキル値帯が明確になる
- 通貨による準備フェーズでコントロール感を確保できる

### スキルの効果式

すべての効果は倍率で表現し、合計・内訳を表示できるようにする。
各アクションの最終倍率は、健康・装備・スキルなど各要因の倍率の合算で決まる。
近接武器はSTR、射撃武器はSENがダメージ基礎値になる。

```
近接攻撃力:     x(100 + 武器種スキル値 × 5) / 100
射撃攻撃力:     x(100 + 武器種スキル値 × 5) / 100
命中率:         x(100 + 武器種スキル値 × 3) / 100
元素耐性:       x(100 - 元素耐性スキル値 × 3) / 100
低体温進行速度: x(100 - 耐寒 ×3) / 100
高体温進行速度: x(100 - 耐暑 ×3) / 100
空腹減少速度:   x(100 - 耐餓 ×2) / 100
回復効果:       x(100 + 治療 ×5) / 100
最大所持重量:   x(100 + 荷重 ×4) / 100
アイテム発見率: x(100 + 探索 ×4) / 100
敵視界距離:     x(100 - 隠密 ×3) / 100
暗所視界範囲:   x(100 + 暗視 ×5) / 100
移動APコスト:   x(100 - 走破 ×2) / 100
素材消費量:     x(100 - 合成 ×3) / 100
合成品質:       x(100 + 調合 ×3) / 100
買値:           x(100 - 交渉 ×2) / 100
売値:           x(100 + 交渉 ×2) / 100
重装備AGIペナ:  x(100 - 重装 ×5) / 100
スキル成長速度: x(100 + 対応属性値 × 5) / 100
```

これらはスキルによる倍率のみを示す。最終的なアクション効率は、健康・装備・スキルの各倍率を合算して決まる。

### スキル成長

Elonaの成長曲線を簡略化したもの。スキル値が高いほど成長が鈍化する。
毎ラン0からスタートし、ダンジョン内での使用と読書で成長させる。

#### 属性による成長速度補正

各スキルには対応する属性があり、その属性値が高いほどスキル経験値の獲得量が増える。
対応はスキル一覧の属性グループと同一。
職業は属性値を補正する主要な要素であり、結果として職業の得意分野のスキルが自然と伸びやすくなる。
成長速度倍率は効果式の「スキル成長速度」を参照。

#### 成長曲線

個別のスキルが使用回数に応じて上がっていく。RPGでいうキャラクターレベルの概念はない。
スキル値が高いほど成長が鈍化する。

```
獲得経験値 = 基本経験値(10) × スキル成長速度倍率 × 100 / (100 + 現在スキル値 × 20)
スキルアップ必要経験値 = 100（固定）
```

| スキル値 | 1回の獲得量 | スキルアップに必要な回数 |
|----------|-----------|----------------------|
| 0→1 | 10.0 | 10回 |
| 5→6 | 5.0 | 20回 |
| 10→11 | 3.3 | 30回 |
| 15→16 | 2.5 | 40回 |
| 20→21 | 2.0 | 50回 |

1ランの想定到達スキル値は5～10程度。上限は設けないが、1ランの行動回数で自然に制限される。

### 本による成長

本を読むことでスキル経験値を獲得できる。ダンジョン内で拾うか、ハブの店で通貨を使って購入する。

```
本の例:
  射撃教本 → 拳銃 +50exp
  錬金術入門 → 調合 +80exp
  悪魔学概論 → 悪魔制御 +60exp, 悪魔戦闘 +40exp
```

本の特徴:
- 読書にはAPを消費する（ダンジョン内で読む場合はターンを使うリスクがある）
- 読了した本は消滅する
- 実戦で伸ばしにくい知識系スキル（錬金、悪魔学、罠学など）は本が主要な成長手段になる
- 逆に身体系スキル（格闘、回避など）は本だけでは上限が低く、実戦が必要になる

### 職業

職業は属性値（STR, SEN, DEX, AGI, VIT, DEF）、スキル初期値、初期装備のセットで差別化する。
属性値の違いがスキル成長速度に影響するため、職業ごとに得意なスキルが自然と伸びやすくなる。
職業の初期スキルはラン序盤の方向性を決めるが、ラン中に別方向に育てることも可能。
職業と特質はハブで通貨を支払って変更できる。

### ダンジョンとスキルの関係

ダンジョンごとに有効なスキルが異なる。
プレイヤーはダンジョンの環境を見て、ラン内でどのスキルを育てるか方針を立てる。

1ランでは一部のスキルしか育てられないため、ダンジョンに合ったビルドを組む判断が重要になる。
ハブの店でどの本・装備を購入するかも、ダンジョンの環境に合わせて選ぶ。

```
例:
  廃坑: 近接敵+FIRE属性が多い → 刀剣/格闘/重装/耐火が有効、火撃の本を持ち込むと有利
  研究所: 射撃敵+THUNDER属性が多い → 回避/拳銃/耐電が有効、合成素材が多い
  凍土: 極寒環境 → 耐寒が必須、暗視/隠密が活きる、耐寒の本が重要
  深淵: 全属性の敵 → 高い戦闘スキルと元素耐性が必要
```

### ラン内の成長曲線

```
1F:  スキル値0～2  本を読んで方向性を決める
      ↓
中層: スキル値3～5  使用で伸ばしつつ装備も強化
      ↓
深層: スキル値6～8  ビルドが完成してくる
      ↓
ボス: 育てたビルドで挑む
```

### 特質（将来実装）

スキルとは別に、キャラクター固有の固定的な性質を表す「特質」を導入する。

- 職業選択の後にポイント制で特質を選択する
- 有利な特質はポイントを消費し、不利な特質はポイントを獲得する
- トレードオフの判断でキャラの個性が生まれる
- 特質はラン中に変化しない

詳細な特質一覧やポイントコストは別途設計する。

### ステータス画面

スキル一覧画面では全スキルが属性グループ別に表示され、各スキルの値と経験値バーが見える。
ダンジョン探索中にスキルが上がるたびに通知が表示される。

### アクション効率と内訳表示

ステータス画面で各アクションの効率を倍率で一覧表示する。
各項目を選択すると倍率変動の内訳が見える。
内訳の要因は健康・装備・スキルの3種類で構成される。
プレイヤーがビルドの効果を把握し、次の判断に活かせるようにする。

```
ステータス画面:
  移動速度       110%
  近接攻撃力     125%
  射撃攻撃力     100%
  採掘速度       150%
  合成品質       109%
  回復効果       115%
  体温低下耐性    55%
  空腹耐性       90%
  ...

「採掘速度 150%」の内訳:
  基本:              100%
  健康: 負傷なし      +0%
  装備: 採掘道具      +25%
  スキル: 荷重Lv3     +5%
  スキル: 合成Lv4    +20%

「体温低下耐性 130%」の内訳:
  基本:              100%
  健康: 負傷なし      +0%
  装備: 防寒装備      +20%
  スキル: 耐寒Lv2    +10%
```

## コメント

### 採用しなかった案

| 案 | 不採用理由 |
|---|---|
| スキル永続（Elona型） | バランス調整が困難。ワールドマップなしの構成では毎ランリセットが適切 |
| 装備・アイテム永続 | 永続側が支配的になりリセット側の意味が薄れる。通貨のみ永続にすることで解決 |
| 本を倉庫に保管可能にする | 「毎ラン序盤に貯めた本を全部読む」が最適解になり、リセットの意味がなくなる |
| スキル8種に絞る | 「パラメータがたくさん育つのを見る楽しさ」が損なわれる |
| 近接/射撃の2分類に統合 | 武器種ごとに独立しているほうが育成の実感がある |
| 属性の自動成長（スキル使用で関連属性も上がるElona方式） | 成長軸が多すぎて調整困難。スキルだけで十分な数がある |
| ポテンシャル制（Elona式の成長率管理） | 単純な減衰式で同じ効果を得られる |
| 職業ごとの属性補正 | スキル初期値で差別化できるので不要 |
| レベルキャップ | 自然な減衰で十分。キャップを設けるとやり込みの動機が削がれる |
| RPG的なキャラクターレベル | 使用ベースのスキル成長で十分。レベルを分離する意味がない |
| アイテムとしてのスキル獲得（Vampire Survivors式） | ruinsではスキルは使用ベースで自然に育つ方が探索ゲームと合う |
