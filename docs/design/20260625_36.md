# 小部屋マップの入口にランダムにドアを生成する

## 背景

小部屋タイプのマップビルダー（`NewSmallRoomPlanner`）で生成されるダンジョンには、部屋と廊下が存在するが、部屋の入口にドアがない。ローグライクとして、閉じた扉が部屋の入口にあることで探索の緊張感や戦術性が生まれる。

### 要件

- 部屋の入口にランダムな確率でドアを配置する
- 3タイル幅の廊下入口を1タイル幅に狭めてドアを設置する
- 隣接する部屋間（壁を挟まず直接つながっている）にはドアを配置しない
- 既存の扉システム（開閉、ロック、BlockPass/BlockView）をそのまま活用する

## 設計

### 方針: DoorPlannerで部屋境界を走査してドアを配置する

廊下が部屋の壁を貫通した箇所（入口）を検出し、入口を1タイルに狭めてドアを置く。PlannerChainの一段として実装し、廊下生成後・壁変換前に実行する。

### アルゴリズム

1. 各部屋の4辺の境界（部屋の外側1タイル）を走査する
2. 境界上の連続する床タイル（= 廊下が壁を貫通した箇所）をグループ化する
3. 以下のタイルではグループを分割する
   - 壁タイル
   - 処理済みタイル（他の部屋の走査で既に処理された入口）
   - 別の部屋の内部タイル（部屋同士が隣接している場合）
4. 各グループについて `DoorChance`（0.5 = 50%）の確率でドアを配置する
5. ドア配置時は中央タイルにドアを置き、他のタイルを壁に戻して入口を1タイル幅に狭める

### PlannerChain内の位置

```
RectRoomPlanner        // 部屋を計画
FillAll("wall")        // 全体を壁で埋める
RoomDraw               // 部屋を描画
LineCorridorPlanner    // 廊下を作成
DoorPlanner            // 入口にランダムにドアを配置  ← 新規
ConvertIsolatedWalls   // 床に隣接しない壁をvoidに変換
EnvironmentPlanner     // 環境情報を計算
```

DoorPlannerは廊下生成後に実行する必要がある（入口検出のため）。ConvertIsolatedWallsの前に配置することで、狭めた壁タイルが適切に評価される。

### データ構造

```go
// DoorSpec はドア配置仕様を表す
type DoorSpec struct {
    consts.Coord[int]
}

// MetaPlan に追加
Doors []DoorSpec
```

### スポーン

`mapspawner/spawner.go` に `spawnDoors` を追加する。既存の `detectDoorOrientation` で隣接タイルから扉の向きを自動判定し、`worldhelper.SpawnDoor` でエンティティを生成する。

入口の狭め処理により、ドア位置の隣接タイルは以下のパターンになる。
- 上辺/下辺の入口: 左右が壁 → 縦向きドア（`DoorOrientationVertical`）
- 左辺/右辺の入口: 上下が壁 → 横向きドア（`DoorOrientationHorizontal`）

### 採用しなかった方法

| 方法 | 不採用の理由 |
|------|-------------|
| ボトルネック検出（廊下のくびれ地点を検出してドアを置く） | 3タイル幅の廊下ではくびれが発生しないため適用できない。廊下幅の変更が前提になる |
| LineCorridorPlanner内で入口座標を記録する | 廊下生成ロジックと入口検出ロジックが混在して責務が曖昧になる。DoorPlannerが独立して走査する方が疎結合で、廊下アルゴリズムを変更しても影響を受けない |
| 廊下を1タイル幅に狭める（入口だけでなく全体を） | 移動の自由度が大幅に下がり、ゲームプレイに悪影響。入口のみの狭めで十分 |

## コメント

- `isInAnyRoom`（既存）は半開区間 `[X1, X2)` だが、`RoomDraw`は `[X1, X2]` の包含範囲で描画する。DoorPlannerでは包含的な判定 `isInAnyRoomInclusive` を用意して、隣接部屋の誤検出を防いでいる
- DoorPlannerのRNG消費により、固定シードを使うテスト（`TestHostileNPCPlanner_PlanMeta/部屋内は同種クラスタになる`）のシードを99999→100に変更した
- `DoorChance: 0.5` は暫定値。ゲームバランスに応じて調整可能
