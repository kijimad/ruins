# コンポーネントベースアーキテクチャへの移行設計

## 概要

現在の `TurnManager` と `ActivityManager` を ECS アーキテクチャの原則に従って
コンポーネント（データ）とシステム（ロジック）に分離する設計。

## 現状の問題

### 1. 循環依存

```
testutil → activity → worldhelper → testutil
```

- `Resources` にマネージャーを保持することで循環参照が発生しやすい
- テストヘルパーでマネージャーを初期化できない

### 2. 初期化タイミングの不明瞭さ

- マネージャーがどこで初期化されるべきか不明確
- テスト毎に手動で初期化する必要がある

### 3. ECS原則からの逸脱

- ECS: Component = Data, System = Logic
- 現状: Manager = Data + Logic（混在）

## 提案する新アーキテクチャ

### 原則

1. **コンポーネント** = 純粋なデータ構造（`internal/components/`）
2. **システム** = コンポーネントを操作するロジック（`internal/systems/`）
3. **マネージャー廃止** = Resources からマネージャーを削除

### 構成図

```
Before:
┌─────────────────────────────────────┐
│ Resources                           │
│  ├─ TurnManager (Data + Logic)      │
│  └─ ActivityManager (Data + Logic)  │
└─────────────────────────────────────┘

After:
┌──────────────────────────┐     ┌─────────────────────────┐
│ Components (Data)        │     │ Systems (Logic)         │
│  ├─ TurnState            │ ←── │  ├─ TurnSystem          │
│  └─ CurrentActivity      │ ←── │  └─ ActivitySystem      │
└──────────────────────────┘     └─────────────────────────┘
```

## 新コンポーネント定義

### 1. TurnState（グローバル状態）

```go
// internal/components/turn_state.go

package components

// TurnPhase はターンの段階を表す
type TurnPhase int

const (
    TurnPhasePlayer TurnPhase = iota // プレイヤーのターン
    TurnPhaseAI                      // AIのターン
    TurnPhaseEnd                     // ターン終了処理
)

// TurnState はグローバルなターン状態を保持する
// 唯一のエンティティ（シングルトン）に付与して使用する
type TurnState struct {
    Phase      TurnPhase // 現在のターンフェーズ
    TurnNumber int       // ターン番号
}
```

### 2. CurrentActivity（エンティティ毎）

```go
// internal/components/current_activity.go

package components

import ecs "github.com/x-hgg-x/goecs/v2"

// ActivityState はアクティビティの実行状態を表す
type ActivityState int

const (
    ActivityRunning ActivityState = iota
    ActivityPaused
    ActivityCompleted
    ActivityCanceled
)

// ActivityType はアクティビティの種類を表す
type ActivityType int

const (
    ActivityNone ActivityType = iota
    ActivityMove
    ActivityAttack
    ActivityRest
    ActivityUseItem
    ActivityInteract
    // 追加アクティビティは列挙で定義
)

// CurrentActivity は実行中のアクティビティを表す
// 各エンティティに付与する（1エンティティ最大1アクティビティ）
type CurrentActivity struct {
    Type         ActivityType // アクティビティの種類
    State        ActivityState
    TurnsTotal   int          // 総必要ターン数
    TurnsLeft    int          // 残りターン数
    Target       *ecs.Entity  // 対象エンティティ
    TargetPos    *Position    // 対象位置
    CancelReason string       // キャンセル理由
}
```

### 3. TurnBased の拡張（既存）

```go
// internal/components/turn_based.go（既存ファイルを拡張）

// TurnBased はターンベース行動に必要なデータを保持する
type TurnBased struct {
    AP    ActionPoints // 現在/最大AP
    Speed int          // 毎ターンのAP回復量
}

// ActionPoints はアクションポイントを表す
type ActionPoints struct {
    Current int
    Max     int
}
```

## 新システム定義

### 1. TurnSystem

```go
// internal/systems/turn_system.go

package systems

import (
    gc "github.com/kijimaD/ruins/internal/components"
    "github.com/kijimaD/ruins/internal/contract"
    ecs "github.com/x-hgg-x/goecs/v2"
)

// GetTurnState はワールドからターン状態コンポーネントを取得する
func GetTurnState(world contract.World) *gc.TurnState {
    // シングルトンエンティティから取得
    var turnState *gc.TurnState
    world.GetManager().Join(
        world.GetComponents().(*gc.Components).TurnState,
    ).Visit(ecs.Visit(func(entity ecs.Entity) {
        turnState = world.GetComponents().(*gc.Components).TurnState.Get(entity).(*gc.TurnState)
    }))
    return turnState
}

// IsPlayerTurn はプレイヤーのターンかを判定する
func IsPlayerTurn(world contract.World) bool {
    state := GetTurnState(world)
    return state != nil && state.Phase == gc.TurnPhasePlayer
}

// IsAITurn はAIのターンかを判定する
func IsAITurn(world contract.World) bool {
    state := GetTurnState(world)
    return state != nil && state.Phase == gc.TurnPhaseAI
}

// AdvanceToAITurn はAIターンに移行する
func AdvanceToAITurn(world contract.World) {
    state := GetTurnState(world)
    if state != nil {
        state.Phase = gc.TurnPhaseAI
    }
}

// AdvanceToTurnEnd はターン終了に移行する
func AdvanceToTurnEnd(world contract.World) {
    state := GetTurnState(world)
    if state != nil {
        state.Phase = gc.TurnPhaseEnd
    }
}

// StartNewTurn は新しいターンを開始する
func StartNewTurn(world contract.World) {
    state := GetTurnState(world)
    if state != nil {
        state.TurnNumber++
        state.Phase = gc.TurnPhasePlayer
    }
}

// CanEntityAct はエンティティが行動可能かを判定する
func CanEntityAct(world contract.World, entity ecs.Entity) bool {
    components := world.GetComponents().(*gc.Components)

    // プレイヤーの場合はターンフェーズをチェック
    if entity.HasComponent(components.Player) && !IsPlayerTurn(world) {
        return false
    }

    // TurnBasedコンポーネントをチェック
    tbComp := components.TurnBased.Get(entity)
    if tbComp == nil {
        return false
    }

    tb := tbComp.(*gc.TurnBased)
    return tb.AP.Current >= 0 // 閾値以上なら行動可能
}

// ConsumeActionPoints はAPを消費する
func ConsumeActionPoints(world contract.World, entity ecs.Entity, cost int) bool {
    components := world.GetComponents().(*gc.Components)
    tbComp := components.TurnBased.Get(entity)
    if tbComp == nil {
        return false
    }

    tb := tbComp.(*gc.TurnBased)
    tb.AP.Current -= cost
    return true
}

// RestoreAllActionPoints は全エンティティのAPを回復する
func RestoreAllActionPoints(world contract.World) {
    components := world.GetComponents().(*gc.Components)
    world.GetManager().Join(components.TurnBased).Visit(ecs.Visit(func(entity ecs.Entity) {
        tb := components.TurnBased.Get(entity).(*gc.TurnBased)
        newAP := tb.AP.Current + tb.Speed
        if newAP > tb.AP.Max {
            newAP = tb.AP.Max
        }
        tb.AP.Current = newAP
    }))
}
```

### 2. ActivitySystem

```go
// internal/systems/activity_system.go

package systems

import (
    gc "github.com/kijimaD/ruins/internal/components"
    "github.com/kijimaD/ruins/internal/contract"
    ecs "github.com/x-hgg-x/goecs/v2"
)

// HasActivity はエンティティがアクティビティを実行中かを返す
func HasActivity(world contract.World, entity ecs.Entity) bool {
    components := world.GetComponents().(*gc.Components)
    actComp := components.CurrentActivity.Get(entity)
    if actComp == nil {
        return false
    }
    act := actComp.(*gc.CurrentActivity)
    return act.State == gc.ActivityRunning
}

// StartActivity はアクティビティを開始する
func StartActivity(world contract.World, entity ecs.Entity, actType gc.ActivityType, turns int) error {
    components := world.GetComponents().(*gc.Components)

    // 既存のアクティビティを中断
    if HasActivity(world, entity) {
        CancelActivity(world, entity, "新しいアクティビティを開始")
    }

    // コンポーネントを設定
    act := &gc.CurrentActivity{
        Type:       actType,
        State:      gc.ActivityRunning,
        TurnsTotal: turns,
        TurnsLeft:  turns,
    }
    entity.AddComponent(components.CurrentActivity, act)

    return nil
}

// ProcessActivityTurn は全アクティビティの1ターン分を処理する
func ProcessActivityTurn(world contract.World) {
    components := world.GetComponents().(*gc.Components)

    world.GetManager().Join(components.CurrentActivity).Visit(ecs.Visit(func(entity ecs.Entity) {
        act := components.CurrentActivity.Get(entity).(*gc.CurrentActivity)

        if act.State != gc.ActivityRunning {
            return
        }

        // ターン消費
        act.TurnsLeft--

        // 完了判定
        if act.TurnsLeft <= 0 {
            act.State = gc.ActivityCompleted
        }
    }))
}

// CancelActivity はアクティビティをキャンセルする
func CancelActivity(world contract.World, entity ecs.Entity, reason string) {
    components := world.GetComponents().(*gc.Components)
    actComp := components.CurrentActivity.Get(entity)
    if actComp == nil {
        return
    }

    act := actComp.(*gc.CurrentActivity)
    act.State = gc.ActivityCanceled
    act.CancelReason = reason
}

// CompleteActivity はアクティビティを完了状態にする
func CompleteActivity(world contract.World, entity ecs.Entity) {
    components := world.GetComponents().(*gc.Components)
    actComp := components.CurrentActivity.Get(entity)
    if actComp == nil {
        return
    }

    act := actComp.(*gc.CurrentActivity)
    act.State = gc.ActivityCompleted
    act.TurnsLeft = 0
}

// GetActivity はエンティティのアクティビティを取得する
func GetActivity(world contract.World, entity ecs.Entity) *gc.CurrentActivity {
    components := world.GetComponents().(*gc.Components)
    actComp := components.CurrentActivity.Get(entity)
    if actComp == nil {
        return nil
    }
    return actComp.(*gc.CurrentActivity)
}

// CleanupCompletedActivities は完了/キャンセルされたアクティビティを削除する
func CleanupCompletedActivities(world contract.World) {
    components := world.GetComponents().(*gc.Components)
    var toRemove []ecs.Entity

    world.GetManager().Join(components.CurrentActivity).Visit(ecs.Visit(func(entity ecs.Entity) {
        act := components.CurrentActivity.Get(entity).(*gc.CurrentActivity)
        if act.State == gc.ActivityCompleted || act.State == gc.ActivityCanceled {
            toRemove = append(toRemove, entity)
        }
    }))

    for _, entity := range toRemove {
        entity.RemoveComponent(components.CurrentActivity)
    }
}
```

## 移行戦略

### Phase 1: 新コンポーネント追加（互換性維持）

1. `TurnState`, `CurrentActivity` コンポーネントを追加
2. 新システム関数を追加（既存マネージャーと並行稼働）
3. ワールド初期化時にシングルトンを作成

### Phase 2: 段階的置換

1. 新規コードで新システム関数を使用
2. 既存コードを段階的に移行
3. テストを新アーキテクチャに移行

### Phase 3: マネージャー削除

1. `Resources` から `TurnManager`, `ActivityManager` を削除
2. `contract.World` インターフェースから `GetTurnManager()` を削除
3. 古いマネージャーコードを削除

## メリット

### 循環依存の解消

- `components` パッケージは依存なし（純粋データ）
- `systems` パッケージは `components` と `contract` のみに依存
- テストヘルパーは `components` を直接使用可能

### 初期化の明確化

- コンポーネントはワールド初期化時に自動登録
- シングルトンエンティティで状態を管理
- テスト毎の手動初期化が不要

### ECS原則の遵守

- データ（Component）とロジック（System）の明確な分離
- 状態の可視性向上（コンポーネントで一元管理）
- テスト容易性の向上

### 拡張性

- 新しいターンシステムの追加が容易
- アクティビティタイプの追加が列挙型で管理
- モジュール間の疎結合

## 注意点

### Behavior インターフェースの扱い

現在の `activity.Behavior` インターフェースは複雑なアクティビティロジックを担当している。
完全にコンポーネント化すると複雑になりすぎるため、以下の方針を推奨する：

1. **シンプルなアクティビティ**: コンポーネント + システム関数で処理
2. **複雑なアクティビティ**: `Behavior` インターフェースを維持し、
   `ActivityType` で振り分けてシステム内で呼び出す

```go
// 複雑なアクティビティの処理例
func ProcessActivityTurn(world contract.World) {
    // ...
    switch act.Type {
    case gc.ActivityMove:
        // シンプル: 直接処理
        processMoveActivity(world, entity, act)
    case gc.ActivityUseItem:
        // 複雑: Behaviorに委譲
        behavior := getBehaviorForType(act.Type)
        behavior.DoTurn(/* ... */)
    }
}
```

## 結論

この設計により：

1. **循環依存が解消** - コンポーネントとシステムの分離
2. **初期化が明確化** - ワールド初期化時に自動設定
3. **テストが容易に** - マネージャー不要でコンポーネント直接操作
4. **ECS原則に準拠** - データとロジックの分離

段階的な移行により、既存コードを破壊せずに新アーキテクチャへ移行できる。
