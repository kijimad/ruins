# ECS コンポーネント取得の型安全化（型付きアクセサの生成）

## 背景

ECS のコンポーネント取得は現在、呼び出し側で手書きの型アサーションを行っている。

```go
name := world.Components.Name.Get(entity).(*gc.Name)
```

問題点:

- `SliceComponent.Get(entity)` は `interface{}` を返すため、`.(*gc.Name)` を毎回手書きする。
- エンティティが当該コンポーネントを保持しない場合、`Get` が `nil` を返し、アサーションで **panic** する。
- フィールド（`Name`）とデータ型（`*gc.Name`）の対応が **コンパイラに保証されず規約頼み**。取り違えても型検査で防げない。
- 非テストで約271箇所、テスト含め約574箇所存在する。

この改善は事前にプロトタイプ済みで、`internal/systems` パッケージ（65箇所）で実証し `make check` が 0 issues で通ることを確認している。本ドキュメントはその設計を確定し、全パッケージへ展開する計画を記す。

## 要件

- コンポーネント取得を **コンパイラが型検査できる** 形にし、手書きアサーションを排除する。
- **reflect / save / Join / AddComponent の既存挙動を変更しない**（save のシリアライズには golden/roundtrip テストがあり、壊すリスクを避ける）。
- 段階的に移行し、各段階で `make check`（全テスト + lint）が通ること。
- 移行完了後、`forcetypeassert` を有効化して生の型アサーションを恒久的に禁止する（設計 + linter の合わせ技）。

## 設計

### 方式: 型付きアクセサメソッドをコード生成する（案C）

`Components` 構造体のフィールドは `*ecs.SliceComponent` の**まま**にし、型付きの取得メソッド `GetX` のみを生成する。

```go
// Code generated by gen; DO NOT EDIT.

// GetName はエンティティのNameコンポーネントを取得する。
// 保持していない場合はpanicする（呼び出し側が保持を保証している場合に使う）。
func (c *Components) GetName(entity ecs.Entity) *Name {
	return c.Name.Get(entity).(*Name)
}
```

呼び出し側:

```go
name := world.Components.Name.Get(entity).(*gc.Name) // 旧
name := world.Components.GetName(entity)             // 新（アサーション消滅、型はコンパイラ保証）
```

`GetX` は生アサーションと同じ「保持前提・違反時 panic」挙動を保つ。Join でエンティティを絞った後の取得が大半（544箇所）で、保持は構造的に保証されるため panic は fail-fast として妥当。

**安全形 `TryX` は生成しない。** optional 取得（comma-ok）は現状わずか2箇所（`SpriteRender`, `TileTemperature`）のみで、YAGNI に従い未使用の `TryX` を全コンポーネントに生成しない。該当2箇所は `HasComponent` チェック + `GetX` に書き換える。

```go
// 旧: comma-ok
if tileTemp, ok := world.Components.TileTemperature.Get(e).(*gc.TileTemperature); ok && tileTemp != nil {
	// ...
}
// 新: HasComponent + GetX
if e.HasComponent(world.Components.TileTemperature) {
	tileTemp := world.Components.GetTileTemperature(e)
	// ...
}
```

### コード生成器

`internal/components/gen/main.go`（package main）を `//go:generate go run ./gen` で起動する。

- `internal/components` パッケージを `go/parser`（`parser.ParseFile`。非推奨の `ParseDir`/`ast.Package` は使わない）で解析する。
- `Components` 構造体の `*ecs.SliceComponent` フィールドを収集する。
- パッケージ内で宣言された型名も収集し、**型が実在するフィールドのみ**生成する（取り違えによる誤生成を防ぐ）。
- フィールド名 == 型名を基本とし、例外は override マップで補正する。
  - `DungeonState` → `Dungeon`
  - `VisualEffect` → `VisualEffects`
- 生成物 `internal/components/accessors_gen.go` をチェックインする（Go の慣習）。

`NullComponent`（マーカー）は Get 対象でないため生成しない（`HasComponent` で判定する）。

### 移行手順

パッケージ単位で以下を実施する（同一パッケージ内ファイルの競合を避ける）:

1. `world.Components.X.Get(E).(*gc.X)` → `world.Components.GetX(E)` に置換。
2. comma-ok（`a, ok := ...Get(E).(*gc.X)`）の2箇所は `HasComponent` + `GetX` に手で書き換える。
3. `goimports` で未使用 import を整理。
4. `go test ./<pkg>/` で確認。

全パッケージ移行後に `.golangci.yml` で `forcetypeassert` を有効化する。

## 採用しなかった方法

- **案A: ジェネリックラッパー型 `Component[T]`**。`Components.Name` を `Component[Name]` にして `Get` が直接 `*Name` を返す案。呼び出しは最も綺麗（アサーション削除のみ）で、埋め込みにより Join/AddComponent も無変更で動く。しかし `components.go` の初期化 reflect（`field.Type()==*ecs.SliceComponent` で分岐）と **save のシリアライズが具体型前提**であり、ジェネリック各インスタンスは別型のため type-switch できず、マーカーinterface経由への書き換えが必要。**save の roundtrip/golden を壊すリスクが高い**ため見送る。
- **案B: ジェネリック自由関数 `Get[T](c, e)`**。構造体・reflect・save は無変更だが、`Get[HP](Components.Name, e)` のように**コンポーネントと型の対応を縛れず**、安全性がむしろ現状より弱いため見送る。
- **`forcetypeassert` の先行有効化**。727件を一斉に `, ok` 化 or nolint するのは方針に反し、アーキテクチャ（型不変式が保証された ECS）にも合わない。**アクセサ移行の後**に有効化することで、生アサーションを置換済みにしてから禁止する。

## コメント

- 案C は「理想形（案A）」ではないが、save のシリアライズという壊してはならない部分を触らずに型安全を実現できる、リスク最小の現実解である。
- `GetX` のみ生成する。安全形 `TryX` は optional 取得が2箇所しかなく、YAGNI に従い生成しない。該当2箇所は `HasComponent` + `GetX` に書き換える。
- 生成メソッドはエクスポートされるため `unused` linter には引っかからない（未使用の `GetBook` 等があっても問題ない）。
- 移行の大半は機械置換で、手当てが要るのは comma-ok の2箇所のみ。
