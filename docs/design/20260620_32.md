# Steam ストアグラフィック制作計画

## 背景

Steam ストアのグラフィックアセットが reject された。現状の画像は original.png を各サイズにクロップしただけで、黒枠の残留・構図の崩壊・デバッグテキストの混入などの問題がある。本格的に作り直す。

## 設計

### テーマ

「遺跡の入口に立つ拾い屋」

- 鶴嘴を背負った探索者が、凍晶に覆われた遺跡の入口を見上げている
- 背後に橙色の焚き火の光、前方に青白い凍晶の光
- 火と氷の対比、孤独な探索者、未知への挑戦がゲームの核心を一枚で伝える
- 橙と青のコントラストにより、Small Capsule まで縮小してもシルエットで認識しやすい

### パーツベースの制作方針

各アセットはアスペクト比が大きく異なる（920x430 と 748x896 など）ため、1枚の絵をクロップする方法では構図が崩壊する。独立したパーツを用意し、サイズごとに組み合わせて配置する。

### 用意するパーツ

| パーツ | 内容 | 形式 |
|--------|------|------|
| ロゴ | 「RUINS」タイトル。橙色＋黒影 | 透過PNG |
| 探索者 | 鶴嘴を背負った後ろ姿シルエット | 透過PNG |
| 遺跡入口 | 凍晶に覆われたアーチ状の構造物 | 透過PNG |
| 焚き火 | 橙色の炎と光 | 透過PNG |
| 凍晶群 | 青白く発光する結晶 | 透過PNG |
| 背景（遺跡内壁面） | 暗い石壁＋凍晶のパノラマ | PNG |
| 背景（グラデーション） | 橙→青のグラデーション | PNG |

### 各アセットの構図

#### ストアアセット

**Header Capsule (920x430)** — 横長
- 探索者を左寄り、遺跡入口を右に配置
- ロゴは上部中央
- 橙→青のグラデーションが横方向に効く

**Small Capsule (462x174)** — 極端な横長
- ロゴが面積の大半を占める（120x45 まで縮小されるため）
- 背景は橙と青のグラデーション。探索者シルエットは小さく添える程度
- ロゴの読みやすさが最優先

**Main Capsule (1232x706)** — 大きめ横長
- Header Capsule と同系統の構図だが上下に余裕あり
- 探索者の全身が入る。焚き火と凍晶の光の対比をしっかり描ける
- ロゴは上部に大きく配置

**Vertical Capsule (748x896)** — 縦長
- 探索者を下部に配置。見上げる構図
- 上部にそびえる遺跡入口と凍晶の光
- ロゴは上部に配置。縦方向のスケール感を活かす

**Page Background (1438x810)** — 背景
- 探索者とロゴは不要。環境のみ
- 凍晶の遺跡内壁面を暗めに。ページコンテンツを邪魔しない
- 左右の端は暗くフェードアウト

#### ライブラリアセット

**Library Capsule (600x900)** — 縦長
- Vertical Capsule に近い構図
- 探索者を中央下部に。遺跡が上に広がる
- ロゴは上部 1/3 に配置

**Library Header (920x430)** — 横長
- Header Capsule と同じ画像を流用可能

**Library Hero (3840x1240)** — 超横長、テキストなし
- ロゴなし。背景のみ
- 凍晶の遺跡内部が左右に広がるパノラマ
- 中央 860x380 がセーフエリア。ここに遺跡入口を配置
- 左右の端は暗い洞窟壁面でフェードアウト

**Library Logo (1280x720)** — 透過PNG
- 透過背景に「RUINS」ロゴのみ
- Library Hero の青白い背景に対して読みやすいよう影をしっかりつける

### 制作順

| 順 | 作業 | 理由 |
|----|------|------|
| 1 | ロゴ（透過PNG） | 全アセットで使い回す。最初に確定させる |
| 2 | 探索者シルエット（透過PNG） | ロゴと並ぶ主要パーツ |
| 3 | グラデーション背景 | ロゴ＋探索者＋グラデーションで最低限のカプセルが成立する |
| 4 | 遺跡入口・凍晶群・焚き火 | 背景の情報量を上げる |
| 5 | 遺跡内壁面パノラマ | Library Hero と Page Background 用 |
| 6 | 各サイズへの組み合わせ・配置調整 | Photopea/GIMP でレイヤー管理 |

### ツール

- ロゴ: Textcraft / Font Meme で候補を作成し、Photopea で調整
- パーツ: Aseprite（ピクセルアートスタイルで統一する場合）
- 組み合わせ: Photopea / GIMP でサイズごとに .psd / .xcf を管理

## コメント

- Steam の共通ルール: 画像端まで隙間なくアートで埋める。タイトル以外のテキスト不可。詳細は docs/steam/README.md を参照
- パーツを独立して作ることで、将来のロゴ変更やアート更新にも対応しやすい

## 進捗

制作方針は本文のパーツベースから実装で乗り換わっている。実際は Stable Diffusion SD-turbo で凍晶ダンジョンのマスター画像を生成し、ImageMagick でアスペクト比ごとにクロップ・ピクセル化・暗化して SVG ロゴを影付き合成する方式になった。docs/steam/gen_master.py と docs/steam/crop_assets.sh がその実体である。探索者シルエットや焚き火など個別パーツの独立 PNG は作られていない。最終成果物のストア画像 9 種は全て docs/steam/generated に規定サイズで実在する。
