# systemでキャッシュをグローバル変数にするのをやめる

## 背景

並列実行のテストで、共有キャッシュがたまに競合するのを防ぐ。

## 設計

- 多くのSystemが、キャッシュを持っている。とくに描画系
- Systemは現状関数だが、構造体+メソッドにする
- 構造体フィールドにキャッシュを持たせて、キャッシュの使い道をわかりやすくする
- グローバルのかわりにどこかで持つ必要があるのだが、どこにするか
  - resources は、以前の最上位にある。worldを使えないので、ここでsystemは保存不可
  - world に保存することにした。保存はworld, 実装はsystemという形にする。最初にworld で Systemを初期化して、あとはそれをゲーム全体で使い回す
  - state

## コメント

- importの依存関係...
  - resources
    - world
      - states
        - systems
      - systems
- System interfaceを定義して、形式を統一した
  - 当初、System() のinterface methodに Draw() と Update() を定義していたが、どちらかしか処理がないSystemがあって、どちらを呼び出せばいいのかわかりにくいので Updater, Renderer interfaceに分けることにした

## 進捗

- [x] System を構造体+メソッド化する。`internal/systems/render_sprite.go` の `RenderSpriteSystem`、`camera_system.go` の `CameraSystem` など各 System が構造体で実装されている
- [x] キャッシュを構造体フィールドに保持する。`RenderSpriteSystem` が `spriteImageCache`、`coloredDarknessCache`、`darknessCacheImages` をフィールドに持ち、`NewRenderSpriteSystem` で初期化する
- [x] グローバル変数の可変キャッシュをやめる。`internal/systems/` に残るパッケージ変数は `wallShadowImage`、`moverShadowImage`、`frostTileImage` など一度だけ初期化する不変の描画画像のみで、可変キャッシュはフィールドへ移っている
- [x] System interface を定義して形式を統一する。`internal/world/world.go` に `Updater` と `Renderer` の interface を定義した
- [x] 単一 System interface を Updater と Renderer に分割する。`Updater` は `Update(world) error`、`Renderer` は `Draw(world, screen) error` を持ち、両方を実装する `HUDRenderingSystem`、`VisualEffectSystem` も存在する
- [x] System を world に保存する。`World` 構造体に `Updaters map[string]Updater` と `Renderers map[string]Renderer` を持たせ、`internal/systems/helper.go` の `InitializeSystems` で組み立て、`internal/maingame/game.go` の `world.Updaters, world.Renderers = gs.InitializeSystems(world)` で最初に初期化してゲーム全体で使い回す
- [~] 保存先を resources または state にする案。resources は world を使えないため保存不可、と本文で退けており、world に保存する結論を採用した
