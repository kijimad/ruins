# 滑らかなキャラクター移動

## 背景

現在、キャラクターの移動は `performMove()` で `GridElement` の座標が即座に変わり、描画もタイル座標をそのままピクセル座標に変換する。結果、キャラクターが瞬間移動して見える。

## 要件

- 移動時にキャラクターが滑らかに補間移動する
- カメラも補間に追従する
- 静止中のエンティティには影響しない
- 既存のターンベースのゲームロジックに影響しない

## 設計

`Animation` コンポーネントで移動元座標と進行度を保持する。移動先は `GridElement` が既に持っている。

```go
type Animation struct {
    FromX, FromY consts.Tile
    Progress     float64 // 0.0→1.0
}
```

### 処理の流れ

1. `performMove()` で `GridElement` を更新する直前に旧座標を記録し、`Animation` コンポーネントを付与する（Progress=0）
2. `MoveAnimationSystem` が毎フレーム Progress を加算する（1.0到達で `Animation` を除去）
3. `renderObjectLayer()` で `Animation` があればタイル座標間を線形補間してピクセル座標を算出する
4. `CameraSystem` でプレイヤーに `Animation` があれば補間座標をカメラ目標に使う

### 補間速度

1フレームあたりの加算量を定数で定義する。`0.2` なら5フレーム（60FPSで約83ms）で移動完了。

## 採用しなかった方法

- **GridElement に補間フィールドを追加する案**: GridElement はタイル座標の責務だけ持つべき。補間状態は移動中だけ必要な一時的な情報なので別コンポーネントが適切
- **Position コンポーネントを流用する案**: 既存の用途（非グリッドベースのスプライト配置）と衝突する
- **MoveLerp という専用コンポーネント名**: Animation で十分に意図が伝わる

## コメント

- 位置交換（`swapAllyIfNeeded`）で移動する隊員にも `Animation` を付与する
