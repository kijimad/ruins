# 隊マネジメントシステムへの転換

## 背景

従来の設計は1人のキャラクターを直接操作するローグライクだった。しかし既存のローグライクとの差別化が不足しており、ゲームを特徴づけるコアメカニクスがなかった。

RimWorldの間接操作とDarkest Dungeonの隊員育成・喪失サイクルを参考に、「リーダーは戦わず、隊員を育てて指示する」という構造に転換する。1人称視点のダンジョン探索なのに自分は戦わない、という組み合わせは既存のローグライクにない。

## 設計

### 概要

プレイヤーはリーダーとして隊員を雇用・育成・改造し、ダンジョンでは指示とアイテム支援で隊を導く。

### 変更点

| 項目 | 旧設計 | 新設計 |
|------|--------|--------|
| 操作 | 1人のキャラクターを直接操作 | リーダーを直接操作し、隊員にはAIと指示で行動させる |
| 戦闘 | プレイヤーが直接戦う | 隊員が戦い、リーダーはアイテム支援のみ |
| スキル永続性 | 毎ランリセット | 隊員に紐づいて永続。隊員の死亡で全喪失 |
| 職業 | ゲーム開始時に選択 | 廃止。隊員のランダム初期属性に置き換え |
| 本の入手 | ダンジョン発見 + 店で購入 | ダンジョン内発見のみ。店では購入不可 |
| リーダー死亡 | （単一キャラのため該当なし） | ラン失敗。通貨なし、バックパック消滅。ストレージと生存隊員は保持 |
| 隊員の死 | （該当なし） | 永久喪失。サイボーグ投資・スキル蓄積ごと失われる |
| 参考作品 | Elona, CDDA, Slay the Spire | RimWorld, Elona, Darkest Dungeon |

### 隊編成

- 2〜10人の隊員を雇用できる
- 軽量NPC（低コスト・使い捨て偵察）と重量NPC（高投資・精鋭）を組み合わせる
- 維持コスト（食料・凍晶バッテリー）が人数に比例するため、多ければ良いわけではない
- 隊員はハブの酒場でランダム候補から選んで雇用する

### 隊員の個性

- 名前・外見・初期属性がランダム生成される
- 性格や人間関係のシステムはない
- 能力面の差異と育成の歴史で愛着が生まれる設計

### 指示体系

位置ベースの命令（護衛・前衛・散開探索・待機・撤退）と戦闘行動の指示（攻撃・回避）を組み合わせる。行動はシンプルで予測可能にし、隊員の動きを見て「なぜそう動いたか」が常にわかる状態を目指す。

### HUD

- ミニマップで隊員・敵の位置を表示する
- 隊員HP一覧で状態を常時把握する
- 1人称視点で散開した隊員の状況をこの2要素で補う

### スキル永続化

`20260319_13.md` のスキル体系（36スキル・7カテゴリ・使用ベース成長・減衰曲線）はそのまま維持する。変更点は永続性モデルのみ:

- スキルは隊員に紐づいて永続する（旧: 毎ランリセット）
- 隊員が死亡するとその隊員のスキル蓄積は全て失われる
- 長く生き延びた隊員ほどスキルが高く、失ったときの損害が大きい

### 本の入手制限

スキルが永続になったため、本を店で購入可能にすると通貨で無限にスキルを伸ばせてしまう。本はダンジョン内でのみ発見できる形にしてバランスを保つ。

## コメント

### 採用しなかった案

| 案 | 不採用理由 |
|----|----------|
| 完全自動（RimWorld型） | 1人称視点と矛盾する。プレイヤーが暇になるリスクが高い |
| リーダー直接操作 + 仲間AI（従来RPG型） | 既にやっているゲームが多く差別化にならない |
| 隊員に性格・人間関係を持たせる | 実装コストが高く、「単なる駒」としての割り切りで十分な深さがある |
| スキル毎ランリセットの維持 | 隊員が永続するのにスキルがリセットされると、隊員の価値がサイボーグパーツだけになる |
| リーダー死亡でセーブ消滅 | 隊員の長期育成投資が一瞬で消えるのは罰が重すぎる |
| リーダー死亡でも隊員保持のみ | ラン失敗のペナルティが軽すぎてリーダーの位置取りが緊張感を欠く |
| 本を店でも購入可能 | スキル永続下では通貨→スキルの無限変換になりバランスが崩壊する |

### 設計の正典

この設計変更により `docs/plan/design.md` を全面改訂した。design.md が現在のゲーム設計の正典である。`20260319_13.md` のスキル体系（スキル一覧・効果式・成長曲線・成長速度）は引き続き有効だが、永続性モデルと本の入手方法は本ドキュメントの設計が優先する。
