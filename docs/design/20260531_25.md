# 敵AI行動戦略の多様化

## 背景

現状の敵AIは単一の行動パターンしか持たない。全ての敵が同じ `AIRoaming` の3状態（Waiting/Driving/Chasing）で動作し、徘徊時はランダム移動、追跡時は直線的に接近するだけである。

ローグライクとしての面白さを出すには、エンティティごとに異なる行動パターンを持たせる必要がある。加えて、エンティティ間の関係性が静的なFactionマーカーのみで表現されており、被ダメージによる態度変化や逃亡行動といった動的な反応ができない。

### 現状の構成

```
components/ai.go         ... AIRoaming（3状態）, AIVision, AIChasing
aiinput/processor.go     ... AI処理制御
aiinput/state_machine.go ... Waiting/Driving/Chasing の遷移
aiinput/action_planner.go ... 状態ごとのアクション決定
components/components.go ... FactionEnemy/FactionAlly/FactionNeutral（静的マーカー）
```

## 設計

2つの軸で拡張する。

### 軸1: Disposition — エンティティの動的な態度

現状のFactionは所属陣営を示す静的マーカーである。これとは別に、他者に対する態度を動的に変化させる `Disposition` コンポーネントを追加する。

```go
// components/ai.go

type DispositionType int

const (
    DispositionHostile  DispositionType = iota // 敵対: 視界内で攻撃する
    DispositionNeutral                         // 中立: 攻撃されると反撃する
    DispositionCowardly                        // 臆病: 攻撃されると逃亡する
    DispositionFleeing                         // 逃亡中: 距離を取る（ランタイム専用）
)

type Disposition struct {
    Default DispositionType // 初期態度。逃亡後にこの値に復帰する
    Current DispositionType // 現在の態度。被ダメージなどで変化する
}
```

データ定義では `disposition` フィールドの指定が必須である。値は `"hostile"`, `"neutral"`, `"cowardly"` のいずれか。`DispositionFleeing` はランタイム専用でデータからは指定しない。

#### Faction と Disposition の関係

| 概念 | Faction | Disposition |
|------|---------|-------------|
| 性質 | 静的な所属陣営 | 動的な態度 |
| 変化 | ゲーム中に変わらない | 被ダメージ・HP低下で変化する |
| 用途 | 敵味方の判定 | 行動パターンの決定 |
| 例 | 「この狼はモンスター陣営」 | 「この狼は今は中立だが、攻撃すると敵対する」 |

FactionEnemyでもDispositionNeutralなエンティティ（攻撃するまで襲ってこない敵）が表現できる。

#### 態度変化ルール

`Disposition.Default` で初期態度を設定する。ランタイムでは `Disposition.Current` が変化する。変動パターンは `Default` の値によって一意に決まり、データで個別に指定する必要はない。

| Default | 被ダメージ時 | HP低下時 | 逃亡終了時 |
|---|---|---|---|
| Hostile | 変化なし | 変化なし | Hostile に復帰 |
| Neutral | → Hostile | 変化なし | Neutral に復帰 |
| Cowardly | → Fleeing | → Fleeing | Cowardly に復帰 |

遷移の実装箇所:
- **被ダメージ・HP低下**: ダメージ処理側で `Disposition.Current` を書き換える
- **逃亡終了**: `aiinput/state_machine.go` の `updateFromFleeing` で `Current` を `Default` に復帰する
- **追跡終了**: `updateFromChasing` で移動状態に遷移する（態度は変化しない）

```
Neutral → 被ダメージ → Current=Hostile → 追跡開始
Cowardly → 被ダメージ → Current=Fleeing → 逃亡開始
Fleeing → プレイヤーを見失い逃亡時間終了 → Current=Default に復帰 → 通常行動に戻る
```

### 軸2: MovementPattern — 非戦闘時の移動パターン

現状の `AIRoamingDriving` はランダム移動のみである。エンティティごとに異なる移動パターンを持たせる。

```go
// components/ai.go

type MovementPattern string

const (
    MovementRandom      MovementPattern = "random"      // ランダム移動。現状と同じ
    MovementPatrol      MovementPattern = "patrol"       // 定点巡回。指定経路を往復する
    MovementWallHug     MovementPattern = "wallHug"      // 壁沿い移動。壁に沿って移動する
    MovementStationary  MovementPattern = "stationary"   // 固定。移動しない番兵タイプ
    MovementWander      MovementPattern = "wander"       // 徘徊。一定範囲をたまに動き回る。街の友好NPCなど
    MovementTerritorial MovementPattern = "territorial"  // 縄張り。スポーン地点から一定範囲内で移動する
    MovementSwarm       MovementPattern = "swarm"        // 群れ。同種族の仲間に寄る
)
```

#### MovementPattern の適用箇所

`aiinput/action_planner.go` の `planDrivingAction()` 内で分岐する。

```go
func (ap *DefaultActionPlanner) planDrivingAction(world w.World, aiEntity ecs.Entity, context *EntityContext) (activity.Behavior, activity.ActionParams) {
    switch context.MovementPattern {
    case gc.MovementStationary:
        // 常に待機
    case gc.MovementWander:
        // 低頻度でスポーン地点周辺をランダム移動
    case gc.MovementWallHug:
        // 壁に隣接する方向を優先
    case gc.MovementSwarm:
        // 最寄りのAIエンティティに接近
    default:
        // ランダム移動
    }
}
```

追跡（Chasing）時の移動は全パターンで共通のまま維持する。MovementPatternは非戦闘時の巡回パターンだけを制御する。

### 組み合わせの例

| 敵タイプ | Faction | Disposition | MovementPattern | 説明 |
|---|---|---|---|---|
| ゴブリン | Enemy | Hostile | Random | 見つけたら追いかける標準的な敵 |
| スケルトン | Enemy | Hostile | Patrol | 通路を巡回し、発見で追跡に切り替わる |
| 番兵 | Enemy | Hostile | Stationary | 動かないが近づくと攻撃する |
| コウモリ | Enemy | Cowardly | Random | ランダム移動、接近すると逃亡する |
| 商人NPC | Neutral | Neutral | Stationary | 攻撃すると逃亡する |
| 街の住民 | Neutral | Neutral | Wander | 店の周辺をうろうろする。攻撃すると敵対する |
| 狼 | Enemy | Aggressive | Territorial | 縄張り内でのみ追跡する |
| スライム群 | Enemy | Hostile | Swarm | 仲間同士で固まって追跡する |
| 大蛇 | Enemy | Neutral | WallHug | 壁沿いに移動、攻撃されると反撃する |

### 変更対象ファイル

```
components/ai.go           ← Disposition, MovementPattern を追加する
aiinput/state_machine.go   ← Disposition に基づく状態遷移を追加する
aiinput/action_planner.go  ← MovementPattern に基づく移動決定に分岐を追加する
aiinput/processor.go       ← 被ダメージ時の態度変化処理を追加する
worldhelper/spawn.go       ← 敵生成時に Disposition/MovementPattern を設定する
raw/raw.go                 ← 敵データに Disposition/MovementPattern フィールドを追加する
```

### 採用しなかった案

**エンティティ間ペアの関係テーブル**: CDDAでは各エンティティペアごとに関係値を保持しているが、現状のECSアーキテクチャではオーバーキルである。Faction + Disposition の2軸で十分な表現力がある。将来的に必要になった場合にのみ導入する。

**BehaviorTree**: 汎用的で拡張性が高いが、現状のFSMベースの設計に対して複雑さが大きい。FSMにMovementPatternパラメータを追加する方がシンプルで既存コードとの親和性が高い。

**Faction間の関係マトリクス**: 肉食動物が草食動物を襲うといった陣営間の関係を表現できるが、まずはDispositionで個体レベルの態度変化を実装し、必要になった時点で導入する。

## コメント

- Dispositionから先に実装する。MovementPatternはDispositionが安定してから追加する
- DispositionのDefault/Current分離により、逃亡後に元の態度に復帰する動作が自然に実現できる
- MovementPatternは非戦闘時の移動のみを制御し、追跡ロジックは共通のままにすることで複雑さを抑える
- 既存の3状態FSM（Waiting/Driving/Chasing）はそのまま維持する。Dispositionは「どの状態に遷移するか」、MovementPatternは「Driving状態でどう動くか」を制御する
- 態度の変動パターンはDefaultの値から一意に決まる。データでトリガールールを個別指定する方式も検討したが、現状のパターン数ではYAGNIに反するため採用しない
- WanderとTerritorialの違い: Wanderは移動頻度が低く街のNPC向け（大半のターンは待機）、Territorialは敵向けで常に動き回るが範囲制限がある
