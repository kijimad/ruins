# 隊員アイテム運搬システム

## 背景

`20260628_41.md` で隊員のアイテム拾得（PolicyPickup）と転送（PolicyDistribute）を実装した。現状の転送先はリーダー固定であり、運搬の戦略性が乏しい。

RimWorldのように運搬を重要な要素にするため、隊員間のアイテム受け渡しを柔軟にする。具体的には:

- 各隊員が一次ストレージ（自分のバックパック）を持つ
- 運搬役の隊員を二次ストレージとして指定できる
- 一次ストレージがいっぱいになったとき、または換金用アイテムは二次ストレージに自動転送する
- 運搬役自身が他の隊員のところに行ってアイテムを受け取れる

## 要件

1. 隊員ごとに転送先を指定できる（リーダー or 特定の隊員）
2. 積載量の閾値超過で自動転送をトリガーする
3. アイテム種別（素材・換金品など）で転送対象を判別する
4. 運搬専用の隊員が他の隊員からアイテムを回収できる

## 設計

### 転送先の指定

`SquadPolicy` に転送先エンティティを追加する。nilならリーダーにフォールバックする。

```go
// internal/components/squad.go

type SquadPolicy struct {
    Position     PositionPolicy
    Combat       CombatPolicy
    ItemPickup   ItemPickupPolicy
    ItemHandling ItemHandlingPolicy
    TransferTo   *ecs.Entity  // 転送先。nilならリーダー
}
```

転送先の解決ロジック:

```
TransferTo が設定済み && 対象が生存 && 対象がActive → その隊員
それ以外 → リーダー
```

### 転送トリガー

2つの条件で転送をトリガーする:

1. **積載量超過**: 現在重量がMaxの80%を超えたとき
2. **アイテム種別**: 素材（Material）は常に転送対象とする

```go
// internal/aiinput/squad_processor.go

// transferThresholdPercent は積載量がこの割合を超えたら転送をトリガーする
const transferThresholdPercent = 80

// shouldTransferItem は特定のアイテムを転送すべきかを判定する
func shouldTransferItem(world w.World, item ecs.Entity, owner ecs.Entity) bool {
    // 素材は常に転送する
    if item.HasComponent(world.Components.Material) {
        return true
    }

    // 積載量超過なら全アイテムを転送対象にする
    wcComp := world.Components.WeightCapacity.Get(owner)
    if wcComp == nil {
        return false
    }
    wc := wcComp.(*gc.WeightCapacity)
    if wc.Max > 0 && wc.Current*100/wc.Max > transferThresholdPercent {
        return true
    }

    return false
}
```

### planItemHandlingAction の変更

現在の実装はリーダーへ無条件転送する。これを以下に変更する:

- PolicyDistribute: 転送条件を満たすアイテムのみ、TransferTo先に転送する
- TransferActivityに転送先エンティティを渡す仕組みを追加する

```go
func (sp *SquadProcessor) planItemHandlingAction(...) {
    if ctx.Policy.ItemHandling != gc.PolicyDistribute {
        return nil, activity.ActionParams{}, false
    }

    // 転送先を解決する
    target := resolveTransferTarget(world, entity, ctx)
    targetGrid := world.Components.GridElement.Get(target).(*gc.GridElement)

    // 転送先と隣接しているときだけアイテムを渡す
    dist := chebyshevDistance(ctx.Grid, targetGrid)
    if dist > 1 {
        return nil, activity.ActionParams{}, false
    }

    // 転送条件を満たすアイテムを検索する
    item := findTransferableItem(world, entity)
    if item == nil {
        return nil, activity.ActionParams{}, false
    }

    // 転送先情報をパラメータに含める
    ...
}
```

### 運搬役の行動パターン

運搬専用隊員は既存の `PositionPolicy` に新しい値を追加するのではなく、`ItemHandling` に新しいポリシーを追加する。

```go
const (
    PolicyKeep       ItemHandlingPolicy = iota // 保持。アイテムを持ち続ける
    PolicyDistribute                           // 分配。転送先にアイテムを渡す
    PolicyCollect                              // 回収。他の隊員からアイテムを受け取りに行く
)
```

PolicyCollect の行動:

```
1. 同じリーダー配下の隊員でバックパックにアイテムがある隊員を探す
2. その隊員に向かって移動する（位置ポリシーより優先）
3. 隣接したら相手のバックパックからアイテムを自分のバックパックに移す
4. 自分がいっぱいなら回収をやめてリーダーの近くに戻る
```

### TransferActivity の変更

現在はリーダー固定だが、任意のエンティティを転送先として受け取れるようにする。`Activity.Destination` を「転送先のGridElement」として流用するか、新しいフィールドを追加するか。

案A: Activity.Target を「転送対象アイテム」、新フィールド Recipient を「受取人」にする。

```go
type Activity struct {
    BehaviorName BehaviorName
    State        ActivityState
    TurnsTotal   int
    TurnsLeft    int
    Target       *ecs.Entity   // 操作対象（転送するアイテム）
    Recipient    *ecs.Entity   // 受取人（転送先エンティティ）
    Destination  *GridElement
    CancelReason string
}
```

案B: SquadMemberのLeaderを辿る現状の方式を維持し、TransferTo フィールドで上書きする。Activity側は変更しない。

→ 案Bを採用する。Activityの構造体変更は影響範囲が広く、TransferActivityの内部で転送先を解決すれば十分。

```go
// TransferActivity.performTransfer 内
func (ta *TransferActivity) resolveRecipient(world w.World, actor ecs.Entity) ecs.Entity {
    policy := query.SquadPolicy(world, actor)
    if policy.TransferTo != nil {
        target := *policy.TransferTo
        // 生存かつActiveな隊員であることを確認する
        if !target.HasComponent(world.Components.Dead) {
            return target
        }
    }
    // フォールバック: リーダーに転送する
    return query.SquadLeader(world, actor)
}
```

### 重量システムとの連携

既存の仕組み:

- `WeightCapacity{Max, Current}` — エンティティの積載能力
- `Weight{Kg}` — アイテムの重さ
- `WeightDirty` — 重量再計算マーカー
- `SkillWeightBearing` — 荷重スキル。レベルに応じてMaxが増加する
- `MoveToBackpack` は `WeightDirty` を自動付与する

新規で追加するもの:

- `shouldTransferItem` の中で `WeightCapacity` を参照して閾値判定するのみ
- 重量計算ロジック自体は変更不要

## コメント

### 採用しなかった案

| 案 | 不採用理由 |
|----|----------|
| Activity構造体にRecipientフィールドを追加する | Activityは全アクティビティ共通の構造体であり、転送専用フィールドを追加すると他のアクティビティに不要な複雑さが生じる。SquadPolicyのTransferToで解決すれば十分 |
| ジョブキュー方式 | RimWorldのようにジョブキューを管理する方式は柔軟だが、現時点では隊員間の直接受け渡しで十分。ジョブキューは必要になってから導入する |
| アイテムフィルタ（拾うアイテムの条件設定） | 現時点では素材を自動転送するだけで十分。細かいフィルタ条件はUIの複雑さに対して効果が薄い |
| 位置ポリシーに運搬役を追加する | 運搬は「どこにいるか」ではなく「何をするか」の問題なので、ItemHandlingPolicyに追加するのが適切 |
