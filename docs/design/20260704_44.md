# オートセーブと手動セーブ

## 背景

現在のセーブシステムは3つの固定スロット（slot1〜slot3）で、以下の問題がある:

1. **識別性の低さ**: 「スロット1」「スロット2」では、どのセーブデータがどの状態か区別しにくい
2. **手動セーブのみ**: オートセーブがないため、セーブし忘れてプログレスを失う
3. **セーブのタイミング**: 拠点に戻った直後は自然なセーブポイントだが、わざわざメニューを開く必要がある

## 要件

1. 拠点に帰還したとき（精算画面→町遷移時）にオートセーブを実行する
2. オートセーブのファイル名は日時ベース（例: `auto_20260704_1830`）
3. 手動セーブは固定4スロット。名前入力は不要
4. スロットには主人公名とタイムスタンプを表示する
5. ロード画面では手動スロットとオートセーブの両方を表示する
6. WASM環境でも動作する

## 設計

### セーブ名の体系

| 種別 | ファイル名の例 | 生成方法 |
|------|--------------|---------|
| オートセーブ | `auto_20260704_1830.json` | `auto_` + 日時で自動生成 |
| 手動セーブ | `slot1.json` | 固定スロット名 |

### セーブメニューUI

**セーブ画面（手動スロットのみ）:**

```
┌──────────────────────────────┐
│  スロット1  Ash  07/04 18:30 │
│  スロット2  ---              │
│  スロット3  ---              │
│  スロット4  ---              │
│  戻る                        │
└──────────────────────────────┘
```

- 選択すると即座に上書き保存する
- 主人公名はセーブデータから取得する。未使用スロットは `---` を表示する

**ロード画面（手動4スロット + オート4スロット）:**

```
┌──────────────────────────────────────────┐
│ ── 手動セーブ ──                          │
│  Ash  07/04 18:30                        │
│  ---                                     │
│  ---                                     │
│  ---                                     │
│ ── オートセーブ ──                        │
│  Ash  07/04 18:30                        │
│  Ash  07/04 17:00                        │
│  Ash  07/04 15:00                        │
│  Ash  07/04 13:00                        │
│  戻る                                    │
└──────────────────────────────────────────┘
```

- セクション見出しは装飾行でカーソルは止まらない
- 手動スロットは常に4つ表示する。データがないスロットは選択不可にする
- オートセーブは新しい順に最大4件を表示する

### SerializationManager の拡張

既存の `SaveWorld(world, slotName)` / `LoadWorld(world, slotName)` はそのまま使える。追加するのはセーブ一覧の取得と削除:

```go
// internal/save/manager.go

type Info struct {
    Name      string
    Timestamp time.Time
    IsAuto    bool
}

func (sm *SerializationManager) ListSaves() ([]Info, error)
func (sm *SerializationManager) DeleteSave(name string) error
```

`IsAuto` は `auto_` プレフィックスで判定する。

### オートセーブのトリガー

ダンジョン選択確定時（DungeonSelectState.handleSelection、キャンセル判定後・遷移前）にオートセーブする。オートセーブ失敗時はエラーを返してダンジョン遷移を中止する。

### オートセーブの上限

最大4件を保持し、超過時は最も古いオートセーブを自動削除する。

### 主人公名の取得

セーブデータからプレイヤー名を取得するために `GetSavePlayerName(slotName)` を追加する。タイムスタンプと同様に部分的なデシリアライズで取得する。

## 採用しなかった方法

| 方法 | 不採用理由 |
|-----|----------|
| ユーザーがセーブに名前を付けられる | 主人公名+タイムスタンプで十分識別できる。テキスト入力UIが不要になりシンプルになる |
| セーブの削除機能 | スロット制のため上書きで十分。オートセーブは上限管理で自動削除される |
| ダンジョン内でもオートセーブ | 地形・敵・アイテムはセーブ対象外で毎回再生成する設計のため、ダンジョン内セーブは設計変更が大きすぎる |
| 定期的な時間ベースオートセーブ | 拠点のみのセーブ設計と矛盾する。セーブ可能な状態の判定ロジックが必要になり複雑化する |
