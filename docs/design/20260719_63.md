# シームレスワールド Phase 7: 永続ステージと往復

## 背景

[20260717_60.md](./20260717_60.md) のシームレスワールド計画の Phase 7。オーバーワールドの帯・ダンジョンの階・遺跡を、**離れても保持され、階段で行き来できる永続ステージ**として統一する。1つのステージは ECS の1つの `Level`、すなわちタイル空間に対応する。

当初は「ECS world は同時に1つの `Level` しか持てないので、離れるステージをスナップショット退避するしかない」と捉えていた。しかし各ステージのエンティティは互いに独立で、**ステージをまたぐエンティティ参照が存在しない**。ならば全ステージのエンティティを同一 world に共存させ、現在ステージのぶんだけを稼働させて、システム側で絞り込めばよい。退避も serde 往復も要らない。往復はマーカーの付け替えに還元される。この1機構で、遺跡進入も階層移動もオーバーワールド往復もすべて説明できる。遺跡は「隣のステージがオーバーワールド帯」という特殊ケースにすぎない。

現状のダンジョンは一方向で永続でない。`WarpNext` は下るたびに新フロアを生成し前の階を保存しない。`WarpEscape` は街へ出る。上り階段も階の保持もない。オーバーワールド帯は存在するが、遺跡は未実装。本 Phase でこれらを永続ステージの往復モデルへ寄せる。

過去に B2 として `OverworldState.OnPause` で帯退避・`OnResume` で復元を実装し revert した。理由は `internal/states/overworld.go` 207-215 のコメントに残る。`OnPause/OnResume` は射撃・観察・メニュー等のオーバーレイ `TransPush` でも呼ばれ、そこでレベルをいじると画面が黒くなり隊員が再配置される。汎用フックはオーバーレイとレベル遷移を区別できない。本設計は退避そのものを廃し全ステージ共存にするため、オーバーレイは現在ステージを一切触らず本質的に安全になる。マーカー付け替えはレベル遷移専用アクションだけで行う。

## 要件

- すべてのステージは、離れても保持され、戻ると寸分違わず同じ状態になる。エンティティは world を離れないので、忠実度は構造的に保証される。掘った跡・拾った跡・動的エンティティも同一エンティティのまま残る。
- 階段で上下の階を行き来できる。訪問済みの階は稼働へ戻すだけ、未訪問の階は決定的に生成する。
- オーバーレイ `TransPush` は従来どおり現ステージを残す。マーカーを触らない。レベル遷移とオーバーレイを取り違えない。
- 前線・昼夜は単一世界時計 `GameTime.TotalTurns` から導出する。レベル移動中も End フェーズで時計は進むので、戻ったときは経過ターンを反映して前線が進み昼夜も移る。地形の完全保持とは別の軸。時計は巻き戻さない。
- 遺跡入口の配置とステージ生成はチャンク seed や深度から決定的。
- 共存するステージのメモリは1回の潜行の範囲で有界。街や地上へ完全離脱すると、そのステージのエンティティを一括除去して破棄する。

## 設計

### 全体像 — 全ステージ共存 + 稼働マーカーで絞り込み

全ステージのエンティティを同一 world に置き、現在ステージだけを稼働させる。各エンティティは所属ステージを表す `StageMember` と、非稼働を表す `Suspended` マーカーを持つ。ステージ跨ぎのシステムは `Without(Suspended)` で現在ステージだけを処理する。降りる・上る・遺跡へ入る・出るは、すべて「対象ステージを稼働させ、現ステージを Suspended にする」1操作へ還元される。

```
[同一 world に全ステージが共存]

  Player / Squad / シングルトン … StageMember を持たず Suspended も付かない。常に稼働
  ステージA のエンティティ … StageMember{A}。現在ステージなら Suspended なし
  ステージB のエンティティ … StageMember{B}。退避中なら Suspended あり
     │ 階段/入口に触れる
     ▼ swapTo(targetKey)
  ① 現ステージ StageMember{current} の全エンティティに Suspended を付ける
  ② target が既存なら Suspended を外す、なければ決定的生成して StageMember{target} を付ける
  ③ 現在ステージ指標を target にする
  ④ Player を入口/帰還位置へ配置する
  ⑤ 前線など時間派生を現在ターンで再導出する
```

- 各ステージは離れても world に残る。戻ると同一エンティティがそのまま稼働する。掘った跡も敵もアイテムも、コピーでなく現物が残る。
- 一度訪れた階へ**下り直す**ときも、既存エンティティを稼働へ戻すだけ。再生成しない。これが「行き来しても保持」。
- 共存は1回の潜行スコープ。街や地上へ完全離脱すると一括除去する。ダンジョンは深さで有界、オーバーワールドは帯シフトが元々チャンクを捨てるので無限には積まない。

### 1. ステージの同定とマーカー

- **`StageMember{Key StageKey}`**: 所属ステージの identity タグ。全ステージエンティティに付ける。swap の対象選択・完全離脱の一括除去・訪問済み判定に使う。値走査になるので swap 時だけ回し、毎フレームは回さない。
- **`Suspended`**: フィールドを持たないマーカー。現在ステージ以外のエンティティに付く。毎フレームのシステム絞り込みは `Without(Suspended)` で行う。ark のクエリはコンポーネントの有無で絞るので、これはアーキタイプ絞り込みになり安価。
- **Player・SquadMember・共有シングルトンは `StageMember` を持たず、`Suspended` も付かない**。常に稼働し、ステージをまたいで生き続ける。`GameTime` も生きたシングルトンのまま進む。
- なぜ2つに分けるか。`Key` は「どのステージか」を表し管理操作で使う。`Suspended` は「今このフレームで動くか」を表し描画・更新で使う。用途と頻度が違うので別コンポーネントにする。値フィールド1つで両方を兼ねると、毎フレームの絞り込みが値走査になり ark の安価なアーキタイプ絞り込みを使えない。

### 2. ステージ遷移操作 swap

`swapTo(targetKey, generate)` を1つの正典操作にする。

1. 現在ステージ `StageMember{current}` の全エンティティに `Suspended` を付ける。
2. `StageMember{target}` のエンティティが1つでもあれば訪問済み。その全てから `Suspended` を外す。なければ `generate` で決定的生成し、生成物に `StageMember{target}` を付ける。`Suspended` は付けない。
3. シングルトンの現在ステージ指標を `target` にする。
4. Player を入口または帰還位置へ配置する。
5. 前線など時間派生を現在の `TotalTurns` で再導出する。

下り・上り・遺跡進入・脱出は、target とプレイヤー配置が違うだけで、すべてこの swap を呼ぶ。serde も除去もない。往復は同一エンティティがそのまま残るので忠実度は100%。途中失敗しても、マーカーの付け外しは純粋な ECS 操作で、世界が消える壊れ方をしない。

`Suspended` の付与は ark ではアーキタイプ移動、すなわちコンポーネントデータのコピーを伴う。serde 往復よりは桁違いに軽いが無料ではない。帯のような大きいステージを一括 suspend すると遷移時に小さな hitch が出うる。ステージサイズに比例するコストとして見込む。

### 3. 専用経路と汎用フックの区別

マーカー付け替えを汎用 `OnPause` でなく、レベル遷移専用アクションで行う。これが B2 revert の教訓。

- **レベル種が同じ**、ダンジョン階↔階や遺跡階↔階は、Update ロジックが同じなので同一 State 内で swap する。状態遷移を伴わない。
- **レベル種が変わる**、オーバーワールド↔遺跡は、Update が違う。オーバーワールドは前線・帯シフトを回すが遺跡は回さない。そこで State を切り替える。`TransPush`/`TransPop` に swap を載せる。
- **オーバーレイ `TransPush` はマーカーを一切触らない**。現ステージは `Suspended` が付かないまま稼働を続けるので画面はそのまま。共存方式では退避が無いので、オーバーレイは本質的に安全になる。
- `OnResume` はレベル退避フラグが立つときだけ、現ステージの un-suspend と帰還配置と前線再導出を行う。オーバーレイ復帰では何もしない。フラグでレベル遷移とオーバーレイを区別する。`OnPause` は触らない。退避は専用アクションで push の前に済ませてある。

```go
func (st *OverworldState) OnResume(world w.World) {
    if !st.levelSuspended {
        return // オーバーレイ復帰。現ステージはそのまま稼働している
    }
    st.levelSuspended = false
    st.resumeStage(world)          // 帯エンティティの Suspended を外す。復元でなく再稼働
    st.movePlayerToReturn(world)
    st.updateFront(world)          // 前線は現在ターンで再導出。巻き戻さない
}
```

### 4. 階段と遷移イベント

| 操作 | イベント | 遷移 | 備考 |
| --- | --- | --- | --- |
| 下り階段（同種レベル） | `WarpDescend` | 同一 State 内 swap | 深度+1。訪問済みなら稼働へ戻す |
| 上り階段（同種レベル） | `WarpAscend` | 同一 State 内 swap | 深度-1。稼働へ戻す |
| 遺跡進入（種が変わる） | `WarpRuinEnter{定義名}` | `TransPush` + swap | オーバーワールド→遺跡 |
| 遺跡脱出（種が変わる） | `WarpRuinExit` | `TransPop` + swap | 遺跡→オーバーワールド |
| 完全離脱 | `WarpEscape` | 街/地上へ。共存ステージを一括除去 | ダンジョン/遺跡ごと離脱 |

既存の `WarpNext`/`WarpEscape` は、`WarpNext` を「訪問済みなら稼働へ戻す `WarpDescend`」へ拡張し、`WarpEscape` は完全離脱として残す。遺跡地上の脱出は街でなくオーバーワールドへ戻るので `WarpRuinExit` に分け、`WarpEscape` の街帰還を壊さない。階段プロップは既存の `spawnPortals`（`internal/mapspawner/spawner.go` 219-237）に倣い、`raw` の prop トリガを拡張して生成する。

### 5. ステージの種類

| ステージ | StageKey | 生成器 | State 型 | 上下の意味 |
| --- | --- | --- | --- | --- |
| オーバーワールド帯 | 単一 | `overworld.ChunkGen`（帯 K チャンク） | `OverworldState` | 遺跡入口で潜る。地上が最上位 |
| ダンジョン階 | 深度 | `mapplanner` + `mapspawner` | `DungeonState` | 階段で上下 |
| 遺跡階 | 遺跡 id + 深度 | `mapplanner` + `mapspawner` | `DungeonState`（遺跡定義） | 階段で上下。最上階の上りで地上へ |

オーバーワールド帯だけは通常の永続ステージと違い、内部で帯シフトによる東西ストリーミングを持つ。潜行中は帯エンティティが `Suspended` になり、帯シフトのシステムも `Without(Suspended)` で対象外になるのでシフトが止まる。探索中に入口チャンクが破棄されることはない。

### 6. 遺跡の配置

オーバーワールドのチャンク生成にまばらに遺跡入口を置く。`internal/mapplanner/overworld_field.go` の生成後、チャンク seed から決定的に、確率的にごく少数の遺跡入口プロップを配置する。各入口は遺跡定義名を持つ。決定的なので、離れて破棄されたチャンクへ戻っても同じ入口が再生成される。破棄ラインはプレイヤーが規定チャンク分離れたときだけ発生し前線とは無関係、という Phase 5 の性質は保たれる。

### 7. 時計と時間派生の分離

`GameTime.TotalTurns` は単一の世界時計で、書き込みは `internal/components/game_time.go` の `TotalTurns++` 1点のみ。End フェーズから呼ばれる。レベル移動でリセットしない。潜っている間も進む。前線位置・昼夜は保存されずこの時計から導出されるので、ステージ再稼働後に再導出すれば経過が自然に反映される。潜行中に寒波が迫るのは意図した緊張。地形は完全保持・時間は前進、を両立させる。

エンティティ側が絶対ターンをキャッシュしている場合、たとえば「ターン N に湧いた」「ターン M にクールダウン終了」といった値は、退避中に時計が進んでも古い値のまま残る。これらは復元操作を経ないので巻き戻りはしないが、時間派生として再導出すべき状態が前線・昼夜以外にないかは実装時に棚卸しする。

### 8. メモリとセーブ境界

- 共存するのは訪問済みステージのエンティティ。潜行は深さで有界、帯は自己破棄なので無限には積まない。ただしオーバーワールド帯は K チャンク分あり潜行中ずっと live で残るため「有界」であって「小さい」ではない。compact な []byte でなく live エンティティで持つぶん、スナップショット方式より常駐は重い。
- 完全離脱 `WarpEscape` で、そのステージ群の `StageMember` を持つエンティティを一括 `RemoveEntity` する。深さで有界。
- **セーブは全 world serde がそのまま全ステージを保存する**。`StageMember` と `Suspended` を serde 対象に含めるので、ロード後も稼働/非稼働の別が復元される。潜行中セーブも特別扱いなく動く。スナップショット方式が必要としていた 7a/7b の段階分けは不要になり、永続化がタダで付いてくる。代わりにセーブファイルは共存ステージ分だけ大きくなる。
- `ExploredTiles` は現状どおり `json:"-"` で保存しない。入り直しでリセットする方針とも整合するので、探索履歴に追加のセーブ処理は要らない。ロードや入り直しで空に戻る。

### 9. 座標空間の衝突とリークの防止

共存では全ステージのエンティティが同一 world にいるため、2つの問題を分けて塞ぐ。既存コードは「world = 現ステージのみ」を暗黙に前提しているので、その前提を明示的に張り直す作業になる。

**(a) 座標空間の衝突（最優先）**。2階と3階はどちらもタイル `(5,5)` を持つ。座標キーのシングルトン構造が両ステージ分を混在させると、同じキーに複数ステージのエンティティ・タイルが載って衝突する。対象は `SpatialIndex.Characters`/`BlockPass`（`spatial_index.go`）、`Dungeon.VisibleTiles`/`LightSourceCache`/`ExploredTiles`（`dungeon.go`）。いずれも `GridElement` キーの単一 map。ここは静かなリークでなく**キーの上書き**というハードな壊れ方をするので最優先。

- **派生構造は現ステージだけで再構築する**。`SpatialIndex` は invalidate 時に、`VisibleTiles`/`LightSourceCache` は毎フレーム、`Without(Suspended)` を効かせて現ステージのエンティティ・タイルだけから作り直す。suspended は索引に入らないので座標が重ならない。swap 時に spatial index を invalidate して新ステージで再構築する。
- **蓄積状態は入り直しでリセットする**。`ExploredTiles` は探索履歴で、`vision.go` 130 が `= true` を積むだけの蓄積値。現在のエンティティから再構築できないので、単一 map のままだと階をまたいで探索済みが混ざる。ただし**入り直しで探索履歴をリセットしてよい**という方針なので、ステージ別に持つ必要はない。単一 map のまま、swap でステージへ入るたびに clear する。これで map には現ステージ分の座標しか載らず衝突しない。履歴を捨てる代わりに実装が単純になる。`VisibleTiles` は毎フレーム置換の派生、`ExploredTiles` は蓄積で性質は逆だが、後者はリセットで単一 map に収める。

**(b) エンティティのリーク**。座標を介さないシステムも現ステージだけを対象にする。全エンティティ走査のクエリ、エンティティ数の集計、座標の直接検索は、`Without(Suspended)` を付けないと非現ステージを巻き込む。2階のモンスターが3階で行動する類で、失敗が静か。

- ステージ跨ぎのシステムを列挙し、絞り込みを1つのクエリ規約へ集約する。描画・更新・視界・カメラ・衝突・AI・ターン処理・湧きが対象。ark にグローバルな choke point は無いので、規約と回帰テストで担保する。将来のシステム追加でも同じ規律が要る。
- 「`Suspended` なエンティティは現ステージの描画・更新・視界・衝突・AI・タイル索引に影響しない」をテストで固定する。ローカルの ECS テストで完結し、外部 serde の能力に賭けない。
- Player・シングルトンは `Suspended` を持たないので、絞り込みを付け忘れても常に処理される。これは意図どおりで、Player はどのステージでも動く。

## 採用しなかった方法

- **スナップショット退避 + serde 往復（本 Phase の旧設計）**: 離れるステージを serde でメモリ退避し、戻ると Reset せずライブ world へマージ復元する案。だが現行 serde は `world.ECS.Reset()` + 全復元を前提とし（`internal/save/manager.go` 106-111）、Player を残したままレベル部分集合をマージ復元する能力が未確認だった。加えて往復ごとの serde コスト・swap の非原子性・7b 永続化の複雑さを抱える。共存方式はこれらを構造的に消すので採らない。境界越え参照が無いという不変条件のもとでは、退避せず共存させるほうが単純で忠実。
- **決定的再生成のみで復元する**: 深度や seed から再生成すれば地形は一致し保持が軽い。だが掘った跡・拾った跡・動的エンティティを失う。「戻ると寸分違わず同じ」を満たせない。共存で現物を残す。
- **フィールド値 `StageMember{Key}` だけで毎フレーム絞る**: `Suspended` マーカーを持たず、各システムが `Key == current` を毎回判定する案。ark のクエリは値でなくコンポーネント有無で絞るため、全ステージのエンティティを毎フレーム走査して現ステージ以外をスキップすることになり無駄が出る。稼働の有無はアーキタイプ絞り込みが効くマーカーで持つ。
- **遺跡入口を `InteractionDungeonGate` で実現**: ダンジョン選択メニューを開く挙動。フィールドの階段は特定遺跡へ直接入るので直接遷移の新相互作用にする。
- **`WarpEscape` を流用して遺跡地上の脱出にする**: `WarpEscape` は街へ帰り、完全離脱で共存ステージを破棄する。遺跡地上はオーバーワールドへ `TransPop` で戻り共存を保つので、別イベント `WarpRuinExit` に分ける。
- **ステージごとに別 ECS world を生成**: エンジンは単一 world 前提。多 world 化は影響が大きすぎる。同一 world に共存させマーカーで絞る。

## コメント

- 共存 + 稼働マーカーは遺跡専用でなく汎用の「ステージ切り替え」。遺跡進入・階層移動・オーバーワールド往復を1機構で説明できる。遺跡は隣のステージがオーバーワールドという特殊ケース。
- 「境界越え参照が存在しない、各ステージは独立」という不変条件が本設計の要石。これを破る変更、たとえば召喚主参照やステージ跨ぎの掴みが入ると、共存の前提が崩れる。不変条件をコメントとテストで固定し、破る変更を検知できるようにする。
- マーカー付け替えを「専用アクションで実行、`OnResume` のフラグ分岐で再稼働」に寄せるのが肝。汎用フックのままオーバーレイとレベル遷移を状態で区別する。退避が無いぶん、旧設計より汎用フックの危険が小さい。
- 「戻ると寸分違わず同じ」は共存で現物を残すことで、「行き来しても保持」はステージキーで再稼働することで満たす。地形の完全保持と、前線・昼夜が経過ターンを反映して進むことは別の軸。時計は単一で巻き戻さない。
- 本質コストは2つ。最も鋭いのは座標空間の衝突で、`SpatialIndex` や視界マップなど `GridElement` キーのシングルトンが単一ステージ占有を構造レベルで前提していること。既存コード全体が「world = 現ステージ」を暗黙に敷いているので、その前提を張り直す。次がエンティティのリークで、`Without(Suspended)` の付け忘れ。どちらもローカル ECS テストで固定でき、外部 serde の能力に賭けない点は旧設計より確実だが、「フィルタを足すだけ」の軽い話ではなく座標構造の作り替えを含むと見積もる。
- 座標構造は「派生は再構築、蓄積はリセット」で分ける。`SpatialIndex`/`VisibleTiles`/`LightSourceCache` は現ステージから作り直せる。`ExploredTiles` は探索履歴の蓄積で作り直せないが、入り直しでリセットしてよい方針なので、単一 map のまま swap で clear すれば足りる。履歴の永続を諦めることで、ステージ別保持を避け実装を単純に保つ。取り違えて単一 map をリセットせず使うと、探索済みが階をまたいで混ざる。
