# シームレスワールド Phase 7: 永続レベルと往復

## 背景

[20260717_60.md](./20260717_60.md) のシームレスワールド計画の Phase 7。オーバーワールドの帯・ダンジョンの階・遺跡を、**離れても保持され、階段で行き来できる永続レベル**として統一する。往復は既存のポータル遷移と serde 機構を再利用する。

難所は単一 world の制約にある。ECS world は同時に1つの `Level`、すなわちタイル空間しか持てない。あるレベルを保ったまま別レベルを見るには、離れるレベルを**スナップショット退避**するしかない。この1つの機構で、遺跡進入も階層移動もオーバーワールド往復もすべて説明できる。遺跡は「隣のレベルがオーバーワールド帯」という特殊ケースにすぎない。

現状のダンジョンは一方向で永続でない。`WarpNext` は下るたびに新フロアを生成し前の階を保存しない。`WarpEscape` は街へ出る。上り階段も階の保持もない。オーバーワールド帯は存在するが、遺跡は未実装。本 Phase でこれらを永続レベルの往復モデルへ寄せる。

過去に B2 として `OverworldState.OnPause` で帯退避・`OnResume` で復元を実装し revert した。理由は `internal/states/overworld.go` 208-216 のコメントに残る。`OnPause/OnResume` は射撃・観察・メニュー等のオーバーレイ `TransPush` でも呼ばれ、そこでレベルを退避すると画面が黒くなり隊員が再配置される。汎用フックはオーバーレイとレベル遷移を区別できない。本設計は退避を「レベル遷移専用の経路」で行う。

## 要件

- オーバーワールド帯・ダンジョン階・遺跡階のすべてのレベルは、離れても保持され、戻ると寸分違わず同じ状態になる。地形だけでなく、掘った跡・拾った跡・動的エンティティも含めて完全保持する。
- 階段で上下の階を行き来できる。一度訪れた階へ入り直すと復元し、未訪問の階は決定的に生成する。
- オーバーレイ `TransPush` は従来どおり現レベルを残す。レベル遷移とオーバーレイを取り違えない。
- 前線・昼夜は単一世界時計 `GameTime.TotalTurns` から導出する。レベル移動中も End フェーズで時計は進むので、復元後は経過ターンを反映して前線が進み昼夜も移る。地形の完全保持とは別の軸。時計は巻き戻さない。
- 遺跡入口の配置とレベル生成はチャンク seed や深度から決定的。
- スナップショットのメモリは1回の潜行の範囲で有界。街や地上へ完全離脱すると破棄する。

## 設計

### 全体像 — キー付きスナップショットストア

レベルの保持と復元を、キーで識別するスナップショットストア1つに集約する。降りる・上る・遺跡へ入る・出るは、すべて「対象レベルへ swap する」1操作へ還元される。

```
[キー付きレベルストア]  map[LevelKey][]byte  （退避中のレベルだけを保持）

  現レベル（world に載っている唯一の Level）
     │ 階段/入口に触れる
     ▼ swapTo(targetKey)
  ① 現レベルを LevelKey で snapshot してストアへ
  ② world から現レベルのエンティティを除去
  ③ ストアに target があれば復元、なければ決定的生成
  ④ プレイヤーを入口/帰還位置へ配置
  ⑤ 前線など時間派生を現在ターンで再導出
```

- 各レベルは離れるとき丸ごとスナップショット。戻ると寸分違わず復元。掘った跡も敵もアイテムも保持。
- 一度訪れた階へ**下り直す**ときも、ストアにあれば再生成でなく復元する。これが「行き来しても保持」。
- ストアは1回の潜行スコープ。街や地上へ完全離脱すると破棄する。ダンジョンは深さで有界、オーバーワールドは帯シフトが元々チャンクを捨てるので無限には積まない。

### 1. スナップショット機構

離れるレベルを serde でメモリへ保存し、復元は逆操作で行う。往復は save/load と同じ round-trip になり、完全一致が構造的に保証される。

- **保存対象**: 現レベルのタイル・動的エンティティ・`Level`。Player/SquadMember と共有シングルトンは除く。除外集合は `DungeonState.OnStop` と同じ考え方。プレイヤーはスタックを通じて生存し続けるので重複生成しない。
- **`GameTime` は保存も巻き戻しもしない**。スナップショットに含めて復元すると時計がロールバックし、潜っている間の経過が消えて前線が戻る。時計は生きたシングルトンのまま進め、復元後に前線位置を現在ターンから再導出する。地形の完全保持と時間の前進を両立させる肝。
- **LevelKey で識別**: オーバーワールド帯は単一キー、ダンジョン階は深度、遺跡階は遺跡 id と深度の組。
- ストアは singleton に持つ。例えば `Dungeon.LevelStore map[LevelKey][]byte`。7b で serde 対応する。

### 2. レベル遷移操作 swap

`swapTo(targetKey, generate)` を1つの正典操作にする。

1. 現レベルを現在の LevelKey で snapshot してストアへ入れる。
2. world から現レベルのエンティティを除去する。次のレベルが同じ world に載っても衝突しないようにする。
3. `store[targetKey]` があれば復元、なければ `generate` で決定的生成する。
4. プレイヤーを入口または帰還位置へ配置する。
5. 前線など時間派生を現在の `TotalTurns` で再導出する。

下り・上り・遺跡進入・脱出は、target とプレイヤー配置が違うだけで、すべてこの swap を呼ぶ。

### 3. 専用経路と汎用フックの区別

退避を汎用 `OnPause` でなく、レベル遷移専用アクションで行う。これが B2 revert の教訓。

- **レベル種が同じ**、ダンジョン階↔階や遺跡階↔階は、Update ロジックが同じなので同一 State 内で swap する。状態遷移を伴わない。
- **レベル種が変わる**、オーバーワールド↔遺跡は、Update が違う。オーバーワールドは前線・帯シフトを回すが遺跡は回さない。そこで State を切り替える。`TransPush`/`TransPop` に swap を載せる。
- `OnResume` はレベル退避フラグが立つときだけ復元する。オーバーレイ復帰では何もしない。フラグでレベル遷移とオーバーレイを区別する。`OnPause` は触らない。退避は専用アクションで push の前に済ませてある。

```go
func (st *OverworldState) OnResume(world w.World) {
    if !st.levelSuspended {
        return // オーバーレイ復帰。現レベルはそのまま残っている
    }
    st.levelSuspended = false
    st.restoreLevel(world)         // スナップショットから完全復元
    st.movePlayerToReturn(world)
    st.updateFront(world)          // 前線は現在ターンで再導出。巻き戻さない
}
```

### 4. 階段と遷移イベント

| 操作 | イベント | 遷移 | 備考 |
| --- | --- | --- | --- |
| 下り階段（同種レベル） | `WarpDescend` | 同一 State 内 swap | 深度+1。訪問済みなら復元 |
| 上り階段（同種レベル） | `WarpAscend` | 同一 State 内 swap | 深度-1。復元 |
| 遺跡進入（種が変わる） | `WarpRuinEnter{定義名}` | `TransPush` + swap | オーバーワールド→遺跡 |
| 遺跡脱出（種が変わる） | `WarpRuinExit` | `TransPop` + swap | 遺跡→オーバーワールド |
| 完全離脱 | `WarpEscape` | 街/地上へ。ストア破棄 | ダンジョン/遺跡ごと離脱 |

既存の `WarpNext`/`WarpEscape` は、`WarpNext` を「訪問済みなら復元する `WarpDescend`」へ拡張し、`WarpEscape` は完全離脱として残す。遺跡地上の脱出は街でなくオーバーワールドへ戻るので `WarpRuinExit` に分け、`WarpEscape` の街帰還を壊さない。階段プロップは既存の `spawnPortals`（`internal/mapspawner/spawner.go` 219-237）に倣い、`raw` の prop トリガを拡張して生成する。

### 5. レベルの種類

| レベル | LevelKey | 生成器 | State 型 | 上下の意味 |
| --- | --- | --- | --- | --- |
| オーバーワールド帯 | 単一 | `overworld.ChunkGen`（帯 K チャンク） | `OverworldState` | 遺跡入口で潜る。地上が最上位 |
| ダンジョン階 | 深度 | `mapplanner` + `mapspawner` | `DungeonState` | 階段で上下 |
| 遺跡階 | 遺跡 id + 深度 | `mapplanner` + `mapspawner` | `DungeonState`（遺跡定義） | 階段で上下。最上階の上りで地上へ |

オーバーワールド帯だけは通常の永続レベルと違い、内部で帯シフトによる東西ストリーミングを持つ。潜行中は帯が退避されシフトが止まるので、探索中に入口チャンクが破棄されることはない。

### 6. 遺跡の配置

オーバーワールドのチャンク生成にまばらに遺跡入口を置く。`internal/mapplanner/overworld_field.go` の生成後、チャンク seed から決定的に、確率的にごく少数の遺跡入口プロップを配置する。各入口は遺跡定義名を持つ。決定的なので、離れて破棄されたチャンクへ戻っても同じ入口が再生成される。破棄ラインはプレイヤーが規定チャンク分離れたときだけ発生し前線とは無関係、という Phase 5 の性質は保たれる。

### 7. 時計と時間派生の分離

`GameTime.TotalTurns` は単一の世界時計で、書き込みは `turn_system.go` の End フェーズ1点のみ。レベル移動でリセットしない。潜っている間も進む。前線位置・昼夜は保存されずこの時計から導出されるので、レベル復元後に再導出すれば経過が自然に反映される。潜行中に寒波が迫るのは意図した緊張。地形は完全保持・時間は前進、を両立させる。

### 8. メモリとセーブ境界

- ストアは潜行スコープ。`WarpEscape` の完全離脱で破棄する。深さで有界。
- 潜行中にセーブすると、退避中のレベル群はストアに、スタック構造は State インスタンスにある。これらがセーブに乗るかが境界。段階を切る。
  - **7a**: 往復機構を実装する。潜行中はセーブを無効化する、またはロードは地上/街から再開する。ストアの永続化はしない。
  - **7b**: ストアとスタックをセーブ・ロードに載せる。`LevelStore` は既に serde の産物なので永続化と相性がよい。ロード時のステート復元経路 `newResumeStateFactory` を拡張する。7a 完了後に別途設計する。

## 採用しなかった方法

- **汎用 `OnPause/OnResume` でレベル退避（B2 の旧実装）**: オーバーレイとレベル遷移を区別できず画面が黒くなる。revert 済み。専用アクション + フラグで区別する本設計に置換する。
- **決定的再生成のみで復元する**: 深度や seed から再生成すれば地形は一致しスナップショットより軽い。だが掘った跡・拾った跡・動的エンティティを失う。「戻ると寸分違わず同じ」を満たせない。完全保持のスナップショットを採る。
- **スタックのみ、キーなしのスナップショット**: 1つ前のレベルしか戻れず、行き来して入り直すと再生成になる。「行き来しても保持」を満たさない。深度/遺跡 id をキーにしたストアで revisit 復元を成立させる。
- **遺跡入口を `InteractionDungeonGate` で実現**: ダンジョン選択メニューを開く挙動。フィールドの階段は特定遺跡へ直接入るので直接遷移の新相互作用にする。
- **`WarpEscape` を流用して遺跡地上の脱出にする**: `WarpEscape` は街へ帰り、完全離脱でストアを破棄する。遺跡地上はオーバーワールドへ `TransPop` で戻りストアは保つので、別イベント `WarpRuinExit` に分ける。
- **レベルごとに別 ECS world を生成**: エンジンは単一 world 前提。多 world 化は影響が大きすぎる。同一 world にスナップショット退避で共存させる。

## コメント

- スナップショット退避・復元は遺跡専用でなく汎用の「レベル一時退避」。遺跡進入・階層移動・オーバーワールド往復を1機構で説明できる。遺跡は隣のレベルがオーバーワールドという特殊ケース。
- 退避を「専用アクションで実行、復元は `OnResume` のフラグ分岐」に寄せるのが肝。汎用フックのままオーバーレイとレベル遷移を状態で区別する。
- 「戻ると寸分違わず同じ」はスナップショット完全保持で、「行き来しても保持」はキー付きストアで満たす。地形の完全保持と、前線・昼夜が経過ターンを反映して進むことは別の軸。時計は単一で巻き戻さない。
- 7a は往復機構に絞り、潜行中セーブは無効化か地上再開で回避する。永続化の 7b は別設計。段階を切って耐荷重を確認しながら進める。
