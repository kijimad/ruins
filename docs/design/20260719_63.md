# シームレスワールド Phase 7: 往復可能な階段の遺跡

## 背景

[20260717_60.md](./20260717_60.md) のシームレスワールド計画の Phase 7。オーバーワールドの帯 `worldstream.Band` の上に「遺跡」を配置し、階段で潜って戻れるようにする。遺跡内部は既存の通常ダンジョン機構 `DungeonState` をそのまま使い、往復は既存のポータル遷移を再利用する。

難所は帯と遺跡の共存にある。遺跡は同じ ECS world に別マップを生成するため、帯タイルと座標が衝突する。したがって遺跡進入時に帯を退避し、帰還時に復元する必要がある。

過去に B2 として `OverworldState.OnPause` で帯退避・`OnResume` で再生成復元を実装し、revert した。理由は `internal/states/overworld.go` 208-216 のコメントに残る。`OnPause/OnResume` は射撃・観察・ダンジョンメニュー等のオーバーレイ `TransPush` でも呼ばれ、そこで帯を退避すると画面が黒くなり復帰時に隊員が再配置されてしまう。汎用フックはオーバーレイと遺跡進入を区別できない。

Phase 7 はこの制約を踏まえ、退避を「遺跡進入専用の経路」で行う。

## 要件

- オーバーワールド上の階段に触れると遺跡へ潜る。遺跡の階段で元のオーバーワールドの元いた位置へ戻れる。往復可能。
- 遺跡内部は通常ダンジョンと同じ生成・戦闘・階層昇降を使う。遺跡が複数階層でもよい。
- 遺跡往復でオーバーワールドの帯地形が変わらない。帯は決定的なので eastIndex から再生成すれば同一地形へ戻る。
- オーバーレイ `TransPush` は従来どおり帯を残す。遺跡進入とオーバーレイを取り違えない。
- 遺跡の配置はチャンク seed から決定的。同じ場所へ戻れば同じ遺跡入口が再生成される。
- 帯上の動的エンティティ、敵やアイテムは遺跡往復で失われることを許容する。現状の帯は無人。60.md §363 で許容済み。

## 設計

### 全体像

遺跡は `DungeonState` の一種として `TransPush` でオーバーワールドの上に載せる。`TransPush` なのでオーバーワールドはスタックに保持され、遺跡の階段で `TransPop` すれば元の状態へ戻る。往復はこのスタック push/pop で自然に成立する。

帯退避は push の前に専用アクションで行い、復元は pop 後の `OnResume` で「退避フラグが立っているときだけ」行う。フラグで遺跡進入とオーバーレイを区別する。

```
オーバーワールド（帯を描画）
  │ 階段に触れる → WarpRuinEnter イベント
  │   [専用アクション] 帰還位置を記録し帯タイルを消す。bandSuspended=true
  ▼ TransPush（OnPause 呼ばれるが何もしない）
遺跡 DungeonState（別マップを生成・描画）
  │ 内部の WarpNext で深層へ、WarpRuinExit で地上へ
  │ 上り階段に触れる → WarpRuinExit イベント
  ▼ TransPop
オーバーワールド OnResume
      bandSuspended なら帯を eastIndex から再生成し帰還位置へ配置。フラグを倒す
      bandSuspended でない（オーバーレイ復帰）なら何もしない
```

### 1. 進入経路と専用アクション

新しい相互作用 `InteractionRuinEnter` と新イベント `WarpRuinEnter` を足す。既存の `InteractionDungeonGate` はダンジョン選択メニューを開くので使わない。階段は選択なしで特定の遺跡へ直接入るため直接遷移にする。

`OverworldState` はイベント処理を `DungeonState.handleStateChangeRequest`（`internal/states/dungeon.go` 525-536）から継承する。ここに `WarpRuinEnter` を足すのではなく、`OverworldState` 側でオーバーライドして遺跡進入を横取りする。オーバーワールド固有の帯退避が絡むため、遺跡進入はオーバーワールドの責務に閉じる。

専用アクションの中身、`OverworldState.enterRuin`:
1. プレイヤーの現在ローカル座標を記録する。帰還位置。
2. 帯タイルを world から削除する。`worldstream.RemoveEntitiesInXRange` 相当、または帯全域。
3. `st.bandSuspended = true`、`st.returnLocalX/Y = ...` を保持する。
4. 遺跡の `DungeonState` を `TransPush` で返す。遺跡定義名は階段エンティティが持つ。

`OnPause` は従来どおり no-op のまま。退避は enterRuin で push の前に済ませてあるので、オーバーレイの `OnPause` は帯に触れない。

### 2. 帯退避と復元

`OverworldState` に退避状態を持たせる。

```go
type OverworldState struct {
    *DungeonState
    // ...既存フィールド
    bandSuspended bool       // 遺跡へ潜って帯を退避中か
    returnLocalX  consts.Tile // 帰還時のプレイヤーローカル X
    returnLocalY  consts.Tile
}
```

`OnResume` をオーバーライドする。

```go
func (st *OverworldState) OnResume(world w.World) {
    if !st.bandSuspended {
        return // オーバーレイ復帰。帯はそのまま残っている
    }
    st.bandSuspended = false
    // 帯を eastIndex から決定的に再生成し、帰還位置へプレイヤーを戻す
    st.regenerateBand(world)
    st.movePlayerTo(world, st.returnLocalX, st.returnLocalY)
}
```

帯は保存でなく再生成する。`SeamlessBand.EastIndex` はセーブ対象で `restoreFromSave` 相当の経路があるので、eastIndex から K チャンクを再生成すれば同一地形へ戻る。動的エンティティは失うが許容範囲。

### 3. 往復可能な階段

- 下り階段、オーバーワールド → 遺跡: 新規 `InteractionRuinEnter` + `WarpRuinEnter{DefinitionName}`。上記 enterRuin で `TransPush`。
- 上り階段、遺跡 → オーバーワールド: 新規 `InteractionRuinExit` + `WarpRuinExit` → `TransPop`。
- 遺跡内の深層移動: 既存の `InteractionPortalNext` + `WarpNext` をそのまま使う。遺跡が複数階層なら深く潜れる。

既存の `WarpEscape`、街への帰還は変更しない。遺跡地上の脱出は街でなくオーバーワールドへ戻るので、`TransPop` へ解決する別イベント `WarpRuinExit` を新設して分ける。遺跡の `DungeonState` が `WarpRuinExit` を受けて `TransPop` を返す。

遺跡内の階段プロップは既存の `spawnPortals`（`internal/mapspawner/spawner.go` 219-237）の仕組みに倣い、`raw` の prop トリガに `WarpRuinExitTrigger` を足して生成する。

### 4. 遺跡の配置

オーバーワールドのチャンク生成にまばらに遺跡入口を置く。`internal/mapplanner/overworld_field.go` の生成後、チャンク seed から決定的に、確率的にごく少数の遺跡入口プロップを配置する。各入口は遺跡定義名を持つ。

決定的なので、離れて破棄されたチャンクへ戻っても同じ入口が再生成される。破棄ラインはプレイヤーが規定チャンク分離れたときだけ発生し前線とは無関係、という Phase 5 の性質は保たれる。遺跡探索中は帯を退避しシフトが止まるので、探索中に入口チャンクが破棄されることはない。

### 5. セーブ境界

遺跡内でセーブすると、world には遺跡の `Level` が載り、退避中のオーバーワールドはスタック上の `OverworldState` インスタンスにある。このインスタンスの `bandSuspended`・`returnLocalX/Y` とスタック構造はセーブ対象でない。したがって遺跡内セーブ・ロードは帰還文脈を失う。

Phase 7 では踏み込まず、段階を切る。

- 7a: 往復機構を実装する。遺跡内ではセーブを無効化する、またはロードはオーバーワールドから再開する。帰還文脈の永続化はしない。
- 7b: 遺跡セッションをセーブ・ロードに載せる。`SeamlessBand` に `InRuin bool`・`RuinReturnX/Y` を足し、ロード時のステート復元経路を拡張する。既存のロード経路 `newResumeStateFactory` と整合させる。7a 完了後に別途設計する。

## 採用しなかった方法

- **汎用 `OnPause/OnResume` で帯退避（B2 の旧実装）**: オーバーレイと遺跡進入を区別できず画面が黒くなる。revert 済み。専用アクション + フラグで区別する本設計に置換する。
- **帯を保存して復元（serde でメモリスナップショット）**: 帯は決定的なので再生成で足りる。スナップショットは実装とセーブサイズが重い。動的エンティティ保持が要件になった時点で再検討する。
- **遺跡入口を `InteractionDungeonGate` で実現**: ダンジョン選択メニューを開く挙動。フィールドの階段は特定遺跡へ直接入るので直接遷移の新相互作用にする。
- **`WarpEscape` を流用して遺跡地上の脱出にする**: `WarpEscape` は街へ帰る。遺跡はオーバーワールドへ `TransPop` するので別イベント `WarpRuinExit` に分ける。街ダンジョンの脱出を壊さない。
- **遺跡を別 ECS world に生成**: エンジンは単一 world 前提。多 world 化は影響が大きすぎる。同一 world に帯退避で共存させる。

## コメント

- 帯退避を「進入アクションで実行、復元は `OnResume` のフラグ分岐」に寄せるのが肝。汎用フックのままオーバーレイと遺跡を型でなく状態で区別する。
- 7a は往復機構に絞り、遺跡内セーブは無効化か地上再開で回避する。永続化の 7b は別設計。段階を切って耐荷重を確認しながら進める。
