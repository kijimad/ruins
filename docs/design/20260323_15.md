# ダンジョン脱出時の自動売却

## 背景

設計書 `docs/plan/design.md` のランサイクルに基づく。ラン終了時に装備・アイテムは全て自動売却され、通貨に変換される。プレイヤーは初期装備に戻った状態でハブに再スポーンする。

```
クリア時: 所持アイテム全売却 + ボーナス通貨
撤退時:   所持アイテム全売却 + 到達階層に応じた通貨の一部
死亡時:   所持アイテム消滅、通貨獲得なし
```

今回は撤退時の自動売却を実装する。クリア時のボーナス通貨と死亡時の処理は別途実装する。

要件:
- ラン終了時にバックパック内アイテムと装備中アイテムを全て売却する
- 売却結果の一覧（アイテム名・個数・価格）と合計金額を確認画面で表示する
- プレイヤーは確認して「OK」を押すだけでよい
- 交渉スキルによる売値倍率を適用する
- 売却後、選択済みの職業で装備・スキル・能力値を再適用する

## 設計

### フロー

```
ポータル（帰還）踏む
  → WarpEscapeEvent
  → FadeoutAnimation
  → AutoSellState.OnStart（装備解除 → 全アイテム売却 → 職業を再適用）
  → 売却結果画面を表示
  → OK押下
  → TownState（職業の初期装備・スキルに戻った状態）
```

### AutoSellState (`internal/states/auto_sell.go`)

ラン終了時の精算画面。OnStartで売却とリセットを実行し、結果を表示する。

```go
type AutoSellState struct {
    es.BaseState[w.World]
    widget *ebitenui.UI
    result AutoSellResult
}
```

- `OnStart`: `worldhelper.EndRun` を呼び、結果を保持する
- `Update`: 上下キーでページ送り、Enter で `TransReplace → TownState`
- `Draw`: 売却結果リストと合計金額を表示

アイテム一覧はページネーションする。1ページあたりの表示件数は画面サイズから算出する。

UI構成:
```
┌─────────────────────────┐
│        帰還報告          │
├─────────────────────────┤
│ 回復薬        x3    150 │
│ 木刀          x1     50 │
│ 西洋鎧        x1    200 │
│ 鉄鉱石        x5    250 │
│ ...                     │
│                 (1 / 3) │
├─────────────────────────┤
│              合計   650  │
│                         │
│  ↑↓ ページ  Enter 続ける │
└─────────────────────────┘
```

合計金額はフッターに固定表示する。売却対象がない場合は「収穫なし」と表示する。

### EndRun (`worldhelper/run.go`)

ラン終了時の一括処理を行う。

```go
// SoldItem は売却されたアイテム1件の情報
type SoldItem struct {
    Name  string
    Count int
    Price int
}

// AutoSellResult は自動売却の結果
type AutoSellResult struct {
    Items []SoldItem
    Total int
}

// EndRun はラン終了時の精算処理を行う。
// 全装備を解除し、全所持アイテムを売却し、職業の初期状態を再適用する。
func EndRun(world w.World, playerEntity ecs.Entity) AutoSellResult
```

処理:
1. 装備中アイテムを全てバックパックに移動する（`MoveToBackpack`）
2. バックパック内アイテムを全て列挙する
3. 各アイテムの売却価格を計算する（`CalculateSellPrice` + `CharModifiers.SellPrice` 適用）
4. アイテムを削除し、通貨を加算する
5. プレイヤーの `ProfessionID` から職業データを取得し、`applyProfession` で再適用する（能力値・スキル・初期装備・初期アイテム）

Value コンポーネントがないアイテム（価値0）も削除する。ラン終了後にアイテムが残ってはならない。

### Profession コンポーネント (`internal/components/components.go`)

プレイヤーが選択した職業IDを保持する。

```go
type Profession struct {
    ID string // raw.Profession.ID に対応する
}
```

職業選択時にプレイヤーに付与する。`EndRun` ではこのIDで `raw.Master.GetProfession` を呼び、職業データを取得して再適用する。

### applyProfession の共通化

現在 `CharacterJobState.applyProfession` はステートのメソッドだが、`EndRun` からも呼ぶ必要がある。`worldhelper` パッケージにトップレベル関数として移動する。

```go
// ApplyProfession はプレイヤーエンティティに職業の属性値・スキル・装備を適用する
func ApplyProfession(world w.World, player ecs.Entity, prof raw.Profession)
```

### DungeonState の変更 (`states/dungeon.go`)

`handleStateEvent` の `WarpEscapeEvent` 分岐を変更する。

```go
case resources.WarpEscapeEvent:
    return es.Transition[w.World]{Type: es.TransPush, NewStateFuncs: []es.StateFactory[w.World]{
        NewFadeoutAnimationState(NewAutoSellState()),
    }}, nil
```

## コメント

### 採用した方法
- **装備品も含めて全売却**: 設計書の「装備・アイテムは自動売却され初期装備に戻る」に従う。ラン内完結の設計なので、持ち帰りの概念がない
- **装備解除→売却の2段階**: 装備中アイテムは `ItemLocationEquipped` なので、先にバックパックに移動してから一括売却する。売却ロジックが `ItemLocationInPlayerBackpack` のJoinだけで済む
- **EndRunをworldhelper/run.goに分離**: 売却だけでなくスキルリセットも含むラン終了処理。shop.goの売却ロジックとは責務が異なる
- **確認画面を挟む**: 何をいくらで換金したかが一目でわかる。ランの区切りとしての演出になる
- **OnStartで売却を実行**: 画面表示前に処理を完了させる。表示は結果の参照のみ

### 採用しなかった方法
- **装備品だけ残す**: 設計書の「毎ランゼロからビルドを組み立てる」に反する。装備を持ち越せると通貨で買う意味が薄れる
- **shop.goに売却ロジックを追加**: EndRunは売却+リセットの複合処理。shop.goは個別アイテムの売買が責務
- **DungeonState.OnStopで売却**: OnStopはフィールドエンティティの削除が責務。ラン精算とは別の関心事
- **ログ表示のみ**: 見落とす可能性がある。帰還の区切りとして画面を1枚挟むほうが体験がよい
