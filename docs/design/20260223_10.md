# 気温・体温システム設計

## 概要

気温・体温システムを実装する。
6部位（胴体・頭・腕・手・脚・足）の体温追跡、収束温度モデル、タイル単位の気温、装備の保温効果を含む。

## 参考: CDDA の仕組み

- **収束温度モデル**: `temp_cur`（現在）が`temp_conv`（目標）に徐々に近づく
- **温度定数**: FREEZING=28°C, COLD=34°C, NORM=37°C, HOT=40°C, SCORCHING=46°C
- **部位別ペナルティ**: 頭→INT, 手→DEX, 胴体→STR
- **凍傷**: 末端部位（手・足）のみ、タイマー制

## 温度関連要素

### 外的状況（環境気温を決定）

| 要素 | 説明 | 値の例 |
|------|------|--------|
| 基本気温 | ダンジョンの温度範囲からランダム決定、階層移動で変動 | 9°C |
| 天候 | 晴れ、曇、雨、雪など | -3°C |
| 時間帯 | 朝、昼、夜、深夜 | -3°C |
| タイル補正 | 屋内、直射日光、熱源など | +3°C |
| 風 | 風速による体感温度低下（ウィンドチル） | -5°C |

```
環境気温 = 基本気温 + 天候補正 + 時間帯補正 + タイル補正 + 風補正
例: 9 + (-3) + (-3) + 3 + (-5) = 1°C
```

### 風

風速が高いほど体感温度が下がる。

| 風速 | 補正 | 説明 |
|------|------|------|
| 無風 | 0°C | 風なし |
| 微風 | -2°C | 葉が揺れる程度 |
| 弱風 | -5°C | 旗がなびく |
| 強風 | -10°C | 歩きにくい |
| 暴風 | -15°C | 立っているのが困難 |

**風の遮蔽:**

| 状況 | 風の影響 |
|------|----------|
| 屋内 | 無効 |
| 壁際 | 50%軽減 |
| 森の中 | 30%軽減 |
| 屋外 | 100% |

風は天候と連動する。嵐のときは強風、晴れのときは無風〜微風など。

### 内的状況（体温への影響）

| 要素 | 説明 | 効果 |
|------|------|------|
| 装備の保温値 | 部位ごとの保温効果 | 手+5, 胴+10 など |
| 装備の防風値 | 風の影響を軽減 | 風補正を30%軽減など |
| 濡れ状態 | 雨、水辺で濡れると寒さが加速 | 保温効果-50% |
| 活動状態 | 走る、戦闘で体温上昇 | 収束温度+5〜10 |
| 体質/スキル | 寒冷耐性、暑熱耐性など | 影響軽減 |
| 空腹 | 飢餓状態で体温維持が困難 | 収束速度+50% |
| 食事 | 温かい食事で一時的に体温上昇 | 収束温度+10（一時的） |

### 体温レベル（内部値 0-100）

| レベル | 範囲 | 状態 |
|--------|------|------|
| 凍結 | 0-10 | 深刻なペナルティ、凍傷リスク |
| 非常に寒い | 11-25 | 大きなペナルティ |
| 寒い | 26-40 | 軽度ペナルティ |
| 正常 | 41-60 | ペナルティなし |
| 暑い | 61-75 | 軽度ペナルティ |
| 非常に暑い | 76-90 | 大きなペナルティ |
| 灼熱 | 91-100 | 深刻なペナルティ |

// 6部位: 胴体・頭・腕・手・脚・足
type BodyPart int

const (
    BodyPartTorso BodyPart = iota // 胴体
    BodyPartHead                   // 頭
    BodyPartArms                   // 腕
    BodyPartHands                  // 手
    BodyPartLegs                   // 脚
    BodyPartFeet                   // 足
    BodyPartCount                  // 部位数 = 6
)

// 温度定数（内部値、0-100スケール）
const (
    TempFreezing  = 10  // 凍結
    TempVeryCold  = 25  // 非常に寒い
    TempCold      = 40  // 寒い
    TempNormal    = 50  // 正常
    TempHot       = 60  // 暑い
    TempVeryHot   = 75  // 非常に暑い
    TempScorching = 90  // 灼熱
)

type BodyTemperature struct {
    // 部位ごとの温度（0-100、50が正常）
    Current   [BodyPartCount]int // 現在の体温
    Convergent [BodyPartCount]int // 収束先の体温

    // 凍傷タイマー（手・足のみ使用）
    FrostbiteTimer [BodyPartCount]int
}

type TempLevel int

const (
    TempLevelFreezing  TempLevel = iota // 凍結 (0-10)
    TempLevelVeryCold                    // 非常に寒い (11-25)
    TempLevelCold                        // 寒い (26-40)
    TempLevelNormal                      // 正常 (41-60)
    TempLevelHot                         // 暑い (61-75)
    TempLevelVeryHot                     // 非常に暑い (76-90)
    TempLevelScorching                   // 灼熱 (91-100)
)

func (bt *BodyTemperature) GetLevel(part BodyPart) TempLevel
func (bt *BodyTemperature) GetPenalty(part BodyPart) int
func (bt *BodyTemperature) Update() // current を convergent に近づける

## 5. 温度計算式（CDDA参考）

### 環境気温（外部）

```
環境気温(°C) = BaseTemperature
             + (Depth * TemperaturePerFloor)
             + TileModifier
             + TimeOfDayModifier
```

### 収束温度（部位ごと）

```
// 快適温度 = 20°C
// 環境との差分を内部値(0-100)に変換
収束温度 = 50 + (環境気温 - 20) * 2 + 装備保温値

// 例: 環境0°C、保温0 → 収束温度 = 50 + (-20)*2 + 0 = 10（凍結）
// 例: 環境0°C、保温20 → 収束温度 = 50 + (-20)*2 + 20 = 30（寒い）
// 例: 環境20°C、保温0 → 収束温度 = 50（正常）
```

### 温度収束（CDDA式）

```go
// 半減期約30ターンで収束
convergenceRate := 0.03 // 1 - exp(-0.03)
delta := convergent - current
current += delta * convergenceRate
```

### 凍傷タイマー（手・足のみ）

```go
// 温度レベルに応じてタイマー増減
if level <= TempLevelVeryCold {
    frostbiteTimer += 3  // 危険
} else if level <= TempLevelCold {
    frostbiteTimer += 1  // リスク
} else {
    frostbiteTimer -= 2  // 回復
}
// タイマー100で凍傷発症
```

## 7. UI設計

### 設計思想: Rimworld的アプローチ

- すべての要素に数値がある（内部的にシミュレーション）
- 常時表示は最小限にして没入感を保つ
- 詳細メニューで「なぜこうなったか」を全て説明できる
- プレイヤーは結果を見て、必要なら原因を調べられる

### 7.1 ゲームログ（状態変化時）

体温レベルが変化したときにログに出力する。

```
「手が冷たくなってきた」
「足の感覚が戻ってきた」
「胴体が凍えてきた！」

**ログテキスト:**

| 正常→暑い | 「{部位}が熱くなってきた」 |

### 7.2 ステータス異常（左下、飢餓と同列）

深刻な状態のみ、既存のステータス異常表示に追加する。

```

**表示条件:**

| 状態 | 表示 |

| 凍傷発症 | 「凍傷:{部位}」 |

軽度〜中度の寒さ/暑さはステータス異常には表示しない。

### 7.3 データ構造

```go
// 環境気温の内訳を保持
type TemperatureBreakdown struct {
    BaseTemp  int    // ダンジョン基本気温
    FloorMod  int    // 階層補正
    TileMod   int    // タイル補正
    TimeMod   int    // 時間帯補正
    Total     int    // 合計環境気温
}

// 部位ごとの詳細情報
type BodyPartDetail struct {
    Part           BodyPart
    Current        int       // 現在値
    Convergent     int       // 収束先
    Level          TempLevel // 現在レベル
    WarmthFromGear int       // 装備からの保温
    FrostbiteTimer int       // 凍傷タイマー（手・足のみ）
    TurnsToChange  int       // 次のレベルまでのターン数（概算）
    Penalties      []string  // 適用中のペナルティ
}
```

### 7.4 表示の優先度

常時表示で複数の問題がある場合の優先順位:

```
1. 凍傷/灼熱発症中（最優先で警告）
2. 凍結/灼熱状態の部位
3. 非常に寒い/暑い状態の部位
4. 寒い/暑い状態の部位
5. 正常（表示しない）
```

同じ優先度の場合は、重要度順: 胴体 > 頭 > 手 > 足 > 腕 > 脚

## 11. 検証方法

1. `make test` - 全テスト通過
2. `make check` - lint通過

   - BodyTemperature の収束計算
   - 環境気温の計算

4. 手動テスト:
   - 異なるダンジョンで気温が変わることを確認

   - 装備で保温効果があることを確認
   - 体温レベルでペナルティが適用されることを確認
