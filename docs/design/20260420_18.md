# ボス戦の設計方針

## 背景

ローグライクのボス戦は設計が難しい。ビルドがランダムなため特定戦術前提のボスが作れず、死＝最初からなので理不尽に感じさせてはいけない。繰り返し戦うため飽きやすい。

消耗戦型を採用する。

## 設計

### 基本方針

- 通常戦闘の延長でボス戦を設計する。専用のボス戦システムは作らない
- 通常戦闘の選択肢が多ければ、自然とボス戦もさまざまなアプローチで楽しめる
- ボスは通常戦闘の深さの「試験」として機能する

### 消耗戦型

ボス戦の勝敗は「ボス部屋に入る前」にほぼ決まっている。ダンジョン攻略中にどれだけリソースを温存し、有効な手段を集められたかが問われる。

- ボスはユニークNPCとして固有のステータス・行動パターンを持つが、戦闘ルールは通常と同じ
- 弱点をつけば比較的楽に倒せる。弱点なしでもゴリ押しで倒せるが、リソースが厳しい
- 弱点をつける手段はダンジョン内で集める。ボス戦前の準備がカギとなる

### 弱点システム

ボス専用アイテムは作らない。汎用アイテムがボスに対して特に効く設計にする。

- 火属性アイテムは通常敵にも使えるが、凍晶化したボスには特に効く
- 体温回復アイテムは普段から使うが、極寒のボス部屋では必須になる
- 状態異常が通るボスには積極的に使える

汎用的な手段の組み合わせで攻略する。Elonaの「耐性装備・壁魔法・状態異常」の設計を参考にする。

### ボス一覧（16.mdより）

| ボス | ダンジョン | 特徴 |
|------|-----------|------|
| 凍晶化した猟師 | 廃坑 | まだ人の形を留めている |
| 凍晶化した学者 | 研究所 | 凍晶と融合して巨大化している |
| 最初の拾い屋 | 深淵 | 人の面影はない |

### ボスフロア

各ダンジョンの最終階層をボスフロアとする。固定のフロア構成で、帰還用の小部屋とボス部屋の大部屋で構成される。

- ボスフロアは固定マップである。ランダム生成しない
- 帰還ワープの小部屋とボス戦の大部屋で構成される
- 大部屋は闘技場型。入ったら途中で帰還できない
- 大部屋は広く、障害物として柱がいくつかある
- 環境は寒い。体温管理が戦闘中も求められる
- ボス単体の場合と、お供の雑魚がいる場合がある。お供のほうが厄介なパターンもある
- ボスを倒したらそのダンジョンのクリアフラグが立つ
- ボスを倒したら小部屋の帰還ワープで街に帰る
- 撃破後の演出はない
- 事前の警告はない。ステージ構成で危険を伝える

### ボスの報酬と再戦

- ボスは比較的レアなアイテムをランダムドロップする
- ボスは復活する。クリア済みダンジョンに再度潜ってボスと再戦できる

### クリアフラグの永続化

ダンジョンクリアフラグは冒険をまたいで残る永続データである。`Dungeon` 構造体は冒険ごとにリセットされるため、別の構造体で管理する。

- `GameProgress` 構造体を `resources` パッケージに新設する
- ダンジョンクリアフラグ、NPCイベントフラグなどゲーム進行に関する永続データを集約する
- セーブ/ロード対象にする

### 実装方針

| 項目 | 方針 |
|------|------|
| ボスフロアのマップ | テンプレートシステムで固定マップを定義する |
| 最終階層の判定 | `Definition` に `BossPlannerType` を追加し、`Depth == TotalFloors` で切り替える |
| 帰還不可 | `Door` に `Locked` フラグを追加。ボス部屋進入時にロック、撃破後に解除 |
| クリアフラグ | `GameProgress` 構造体で永続管理 |
| ボス配置 | テンプレートのパレットでNPC文字を定義し、固定位置にスポーン |

### 検討したアプローチと不採用理由

- パターン学習型（Hades, Enter the Gungeon）— アクション寄り。RUINSのターン制と合わない
- リソースチェック型（Slay the Spire）— カード/デッキ系に向く
- パズル型（Noita）— 専用ギミックが必要になり、実装コストが高い
- 環境利用型（Darkest Dungeon）— パーティ編成前提。RUINSは単独探索

## コメント

- ボス戦を面白くするより、通常戦闘を面白くすることに注力すべき
- 通常戦闘の深さがそのままボス戦の深さになる
- ボスの個性はステータスと行動パターンの差で出す。専用ギミックは不要
