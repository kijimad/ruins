# 遠距離攻撃の設計

## 背景

現在の戦闘は近接のみ。遠距離攻撃を追加することで、戦闘の幅を広げる。遠距離から有利に攻撃できるが、弾薬を消費し、リロードにターンを消費する。距離が遠いほど命中率が下がる。

銃だけでなく弓も遠距離武器として扱う。弓は矢を1本ずつ装填して撃つ。銃はマガジンに複数発装填してから連続で撃てる。どちらもリロード（装填）にターンを消費する。

既存コードベースには`AttackRangeRanged`、`AttackHandgun`/`AttackRifle`/`AttackCanon`、対応するスキルマッピング（SEN依存）が定義済みで、`attack.go`に`// TODO: 遠距離武器の場合は射程を武器から取得`のコメントがある。基盤は整っている。

## 設計

### UIフロー

```
Fキー押下 → ShootingState起動
  → 装備中の武器がRanged以外 or 弾切れ → ゲームログにメッセージ、遷移しない
  → 視界内の敵対NPCを距離順にリスト化
  → 最も近い敵に自動照準（カーソル表示）
  → Tab: 次の敵に切り替え / Shift+Tab: 前の敵
  → 情報パネル: 敵名、距離、命中率、残弾数/装弾数
  → Enter: 射撃実行（ShootActivity）
  → R: リロード実行（ReloadActivity）、ShootingStateを抜ける
  → Escape: キャンセル
```

LookAroundStateとの違い: カーソルは自由移動ではなく敵間をジャンプする。

### コンポーネント変更

Weaponに弾薬管理フィールドを追加する。

```go
type Weapon struct {
    TargetType    TargetType
    Cost          int
    Magazine      int    // 現在の装弾数
    MagazineSize  int    // 最大装弾数（0 = 弾薬不要、近接武器）
    ReloadEffort  int    // リロード完了に必要な総工数
    AmmoTag       string // 使用する弾薬アイテムのタグ（例: "ammo_9mm"）
}
```

`MagazineSize == 0`なら近接武器として従来通り動作する。遠距離武器は必ず`MagazineSize > 0`。

#### リロード工数

リロードは本の読書と同じ`Effort`パターンを採用する。武器は`ReloadEffort`（完了に必要な総工数）を持ち、毎ターンの装填進捗はプレイヤーの能力値と武器スキルで決まる。

```go
// 1ターンあたりの装填工数
effortPerTurn = baseReloadEffort + dexterity + weaponSkillLevel
```

`baseReloadEffort`は基本工数（例: 10）。DEXと武器スキルが高いほど速くリロードできる。これにより同じ武器でもキャラクターの成長でリロードが速くなり、スキル育成に意味が生まれる。

#### 武器タイプごとの特性

| 武器 | MagazineSize | ReloadEffort | 特徴 |
|------|-------------|-------------|------|
| 弓 | 1 | 10 | 1発ごとに装填。軽い工数で素早く繰り返せる |
| 拳銃 | 6 | 20 | マガジン装填。バランス型 |
| 小銃 | 10 | 30 | 大容量マガジン。装填に時間がかかる |
| 大砲 | 1 | 40 | 1発ごとに装填。重い工数で高火力 |

弓と大砲はどちらもMagazineSize=1だが、ReloadEffortが10 vs 40で大きく異なる。スキルLv0・DEX10のキャラクターなら1ターンあたり工数20なので、弓は1ターンで装填完了、大砲は2ターンかかる。スキルが育てば大砲も1ターンで装填できるようになる。

### 弾薬

弾薬はインベントリ内の通常アイテムとして扱う。`Item.Count`で数量管理し、`Stackable`マーカーを付ける。武器の`AmmoTag`と一致するタグを持つアイテムを消費する。

```toml
# raw.toml での定義例
[[items]]
name = "9mm弾"
tag = "ammo_9mm"
count = 30
stackable = true
```

弾薬を独立アイテムにすることで、拾う・買う・ドロップの既存アイテムシステムをそのまま使える。

### 射撃 — ShootActivity

新規Activity。`attack.go`の既存ロジックを活用する。

```
Validate:
  - 装備武器がAttackRangeRanged
  - Magazine > 0
  - ターゲットが視界内かつ射程内
  - 射線上にBlockView=trueのタイルがないこと

DoTurn:
  - Magazine -= 1
  - 命中判定（距離補正 + 遮蔽ペナルティ）
  - ダメージ計算（既存calculateDamageを流用）
  - 武器スキル成長（既存growWeaponSkillを流用）
  - 外れた場合、射線上の遮蔽物への命中判定（将来実装）
```

1ターンで完了する。APコストは武器の`Cost`を使う。

### 命中率 — 距離補正

既存の命中率計算に距離ペナルティを追加する。

```go
hitRate = baseHitRate + statModifier + weaponAccuracy + skillBonus
hitRate -= distancePenalty(distance, weaponType)
```

各武器タイプに最適射程がある。最適射程内はペナルティなし。超過すると1タイルにつき一定%減少する。

| 武器 | 最適射程 | 最大射程 | 超過ペナルティ |
|------|----------|----------|---------------|
| 弓 | 4 | 10 | -5%/tile |
| 拳銃 | 3 | 8 | -8%/tile |
| 小銃 | 8 | 16 | -3%/tile |
| 大砲 | 6 | 12 | -5%/tile |

最大射程を超えたら射撃不可とする。

### 射線と遮蔽

#### 射線チェック

射撃時、プレイヤーとターゲットの間をBresenhamの直線アルゴリズムでトレースし、途中のタイル/エンティティを調べる。

- `BlockView=true`のタイル（壁）が射線上にある → 射撃不可
- `BlockPass=true, BlockView=false`のエンティティ（柵、家具など）が射線上にある → 射撃可能だが遮蔽ペナルティ

#### 遮蔽（カバー）

射線上にカバーオブジェクトがあれば命中率にペナルティを与える。

```go
hitRate = baseHitRate + statModifier + weaponAccuracy + skillBonus
hitRate -= distancePenalty(distance, weaponType)
hitRate -= coverPenalty(射線上の遮蔽)
```

射線上にあるBlockPass=trueかつBlockView=falseのエンティティ1つにつき命中率ペナルティを加算する。遮蔽物が多いほどペナルティが重なる。

```go
const coverPenaltyPerObject = 15 // 射線上の遮蔽物1つにつき命中率-15%
```

#### 外れ弾と物体ダメージ

射撃が外れた場合、弾は射線上のいずれかの物体に命中する可能性がある。将来的にすべてのブロック・配置物は破壊可能になるため、外れ弾のダメージ蓄積を想定しておく。

```
外れた場合:
  - 射線上の遮蔽物に命中判定を行う（遮蔽物ごとに捕捉確率を持つ）
  - 命中した物体にダメージを蓄積する
  - HPが0になった物体は破壊される
```

物体の耐久値（HP）と破壊は、破壊可能オブジェクト全般の仕組みとして別途設計する。射撃システムはそのインターフェースを呼ぶだけにする。今回は遮蔽ペナルティのみ実装し、外れ弾のダメージ蓄積は破壊可能オブジェクトの実装時に追加する。

### リロード — ReloadActivity

リロードは本の読書と同じ工数ベースの連続アクティビティで、武器の`ReloadEffort`で完了に必要な総工数が決まる。弓の「矢をつがえる」も小銃の「マガジンを交換する」も同じReloadActivityで表現する。

```
Validate:
  - 装備武器がMagazineSize > 0
  - Magazine < MagazineSize
  - インベントリにAmmoTagと一致する弾薬がある

Start:
  - TurnsTotal = 上限見積もり（ReloadEffort / baseReloadEffort）
  - 内部で装填工数の蓄積を開始

DoTurn:
  - effortPerTurn = baseReloadEffort + DEX + weaponSkillLevel を蓄積
  - 蓄積がReloadEffortに達したら弾薬アイテムのCountを減らし、Magazineを補充
  - 弾薬が足りなければ持っている分だけ装填

Canceled:
  - 途中キャンセル可能、部分装填はしない
```

スキルLv0・DEX10の場合: 弓（ReloadEffort=10）は1ターンで工数20を蓄積→即完了。大砲（ReloadEffort=40）は2ターンで完了。スキルが育てばすべての武器のリロードが速くなる。

### ShootingState

LookAroundStateをベースに、ターゲット選択に特化する。

```go
type ShootingState struct {
    targets     []ecs.Entity  // 視界内の敵対NPC（距離順）
    targetIndex int           // 現在選択中のインデックス
    weapon      *Weapon       // 装備中の遠距離武器
}
```

選択中の敵にカーソル表示し、情報パネルに命中率・距離・残弾を表示する。

### 近接と射撃の使い分け

遠距離武器はすべて近接性能も持つ。移動キーで敵の方向に入力すると、従来通り近接攻撃（殴り）が発生する。Fキーは射撃専用で、装備中の武器が遠距離武器でなければゲームログにメッセージを出すだけで遷移しない。武器の自動切り替えは行わない。

これにより、弾切れ時でも遠距離武器で殴って戦える。弓で殴る・銃で殴るという選択肢が常にある。

## 不採用にした案

### 近接攻撃と統合して自動で射撃に切り替える案

UIなしで最寄り敵を自動射撃する。シンプルだが、ターゲット選択ができず戦術性に欠ける。また弾薬を意図せず消費してしまう。

### LookAroundStateを拡張して射撃機能を追加する案

責務が混在する。観察は情報取得、射撃はアクション実行で目的が異なる。

### 弾薬を武器内蔵にする案

Weapon.Ammoだけで完結させる。リロードの概念がなくなり、弾薬管理の戦略性が失われる。
