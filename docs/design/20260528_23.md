# コンテンツ追加とSteamリリース準備

## 背景

ミニマルリリースに向けて、未使用スプライトをすべてゲーム内コンテンツとして活用し、Steamリリースに必要な要素を整理した。

## ミニマルリリース分析

現在の完成度は約85-90%。主な不足要素:

- **サウンド/オーディオシステム**: BGM・SE未実装。ミニマルリリースにおける最大のギャップ
- ストーリー/テキストの最終調整
- チュートリアル/オンボーディング

## Steamリリースに必要なこと（Ebiten）

### フェーズ1: Steamworks準備
- Steamworksアカウント登録（$100のアプリクレジット料）
- ストアページ作成（カプセル画像、スクリーンショット5枚以上、説明文）
- ストアページ公開後、レビュー審査に最低2週間

### フェーズ2: Steamworks SDK統合

EbitenでのSteamworks統合の選択肢:

1. **steamworks（CGo経由）**: `github.com/nicholasgasior/gosf` など。CGo依存
2. **steam_api DLL直接呼び出し**: `syscall.LoadDLL` で steam_api64.dll をロード。CGo不要
3. **サブプロセス方式**: 別プロセスでSteam APIを呼び、パイプ/ソケットで通信。最もポータブル

ミニマルリリースでは実績やオーバーレイは不要なので、DLL直接呼び出しで `SteamAPI_Init` と `SteamAPI_Shutdown` だけ実装すれば十分。

### フェーズ3: ビルドとリリース
- `steamcmd` でデポにビルドをアップロード
- リリースチェックリスト通過後に公開
