# 本システム

## 背景

知識を得る手段として「本」を追加する。現在はスキル経験値を毎ターン獲得するスキル本のみ実装している。

本でしか伸ばせないスキルもあり（探索、暗視、各種耐性など）、戦闘以外のスキル成長手段として重要な位置づけになる。本なしで上げられるスキルは、練習で上げてから本を読むか、より易しい本を探すことになる。

スキル値・読書進捗はラン内で完結する。毎ランゼロから成長するため、本は1ラン内で読了できる分量に設計する。基本的な本は約20ターンで読了でき、上級の本でも40ターン程度を上限とする。

要件:
- 読み通すには複数ターンが必要で、工数は本によって異なる
- スキル経験値を毎ターン獲得する（レベル適性あり）
- スキル本にはレベル範囲があり、RequiredLevel未満では読めず、MaxLevelとの差で効率が変わる
- 対応する能力値が高いほど1ターンあたりの読書工数が多い
- 危険な状態（近くに敵）になると読書は自動中断される
- プレイヤーが任意のタイミングで読書を手動中断できる
- 中断しても読書進捗は保持され、次回は続きから再開できる
- 読書中は空腹が進行する
- 読了した本は消費される
- インベントリから「読む」アクションで読書を開始できる
- 店・インベントリのspec表示に本のステータスを表示する

## 設計

### Book コンポーネント (`internal/components/book.go`)

```go
type Book struct {
    Effort Pool             // Max=読了に必要な総工数, Current=蓄積した工数
    Skill  *SkillBookEffect // スキル経験値を毎ターン獲得する効果
}

type SkillBookEffect struct {
    TargetSkill   SkillID // 経験値を獲得するスキル
    MaxLevel      int     // この本で上げられるスキル上限
    RequiredLevel int     // 読むのに必要なスキルレベル。0なら誰でも読める
}
```

- `Effort`: 既存の `Pool` 型を再利用。Max=総工数（雑誌200、専門書300、教本400）、Current=中断しても保持される蓄積工数
- `Skill`: ポインタフィールドで効果の有無を表現する

EntitySpec, Components に `Book` を追加する。`save:"true"` で永続化する。

### 工数と読書速度

1ターンあたりの読書工数は `基本工数10 + 対応する能力値` で計算する。能力値が高いほど速く読める。

```go
func calcEffortPerTurn(book, actor, world) int {
    baseEffort := 10
    abilityValue := SkillAbility[book.Skill.TargetSkill] の能力値
    return baseEffort + abilityValue
}
```

### 経験値効率

プレイヤーのスキルレベルと本のMaxLevelの差に基づく。

```
diff = MaxLevel - playerLevel（正=本が難しい）

diff > 5  → 0%   難しすぎて理解できない
diff = 5  → 50%
diff = 0  → 100% 最適
diff = -5 → 10%  易しすぎて得るものがない
diff < -5 → 10%

難しい側: efficiency = 100 - diff*10
易しい側: efficiency = 100 + diff*18
```

既存の `skill.GainExp` を再利用し、`GrowthConfig.BaseExp` を efficiency で調整する。

### ReadActivity (`internal/activity/read.go`)

RestActivityと同じパターンの多ターンアクティビティ。

```
Info:
  Name:          "読書"
  Interruptible: true
  Resumable:     true

Validate:
  - comp.Targetが設定されていること
  - 対象がBookコンポーネントを持つこと
  - 本が読了済みでないこと
  - RequiredLevel以上のスキルレベルがあること
  - 周囲に敵がいないこと（isAreaSafe共通関数）

DoTurn:
  1. 安全性チェック → 敵がいれば自動中断
  2. Effort.Current += calcEffortPerTurn()
  3. progressHunger() で空腹進行
  4. applyPerTurnEffect: 経験値効率を計算して GainExp を呼ぶ
  5. スキルアップしたら CharModifiers 再計算
  6. Effort.Current >= Effort.Max なら Complete

Finish:
  - 読了メッセージ
  - worldhelper.ChangeItemCount で本を消費

Canceled:
  - 中断メッセージ。Effort.Currentはそのまま保持
```

### 共通ユーティリティ (`internal/activity/activity.go`)

ReadActivityとRestActivityで共有する関数:
- `isAreaSafe`: 周囲3x3に敵がいないかチェック
- `progressHunger`: ターン経過による空腹進行（MoveActivityでも使用）

### spec表示 (`internal/widgets/views/spec.go`)

店・インベントリのアイテム詳細に本の情報を表示する:
- ヘッダー: 「本」（読了済みなら「本（読了）」）
- スキル: 対象スキル名
- Lv: `RequiredLevel → MaxLevel`（Nerd Fonts矢印アイコン使用）
- 進捗: パーセンテージ表示（0%の場合は非表示）

### インベントリ連携 (`internal/states/inventory_menu.go`)

- `getActionItems()`: Bookコンポーネントを持つアイテムに「読む」アクションを表示
- `executeActionItem()`: 残り工数（Effort.Max - Effort.Current）をDurationとしてReadActivityを実行

## コメント

### 採用した方法
- **ポインタフィールドで効果を表現**: `Skill *SkillBookEffect` のnil/非nilで効果の有無を判定する。deadcodeツールに検出されず、型安全
- **工数ベースの読書進捗**: ターン数固定ではなく、能力値で読書速度が変わる。能力値を上げるモチベーションになる
- **Bookを独立コンポーネントにする**: Item+Bookの組み合わせで表現。読了進捗がエンティティごとに保持される
- **RestActivityのパターンを踏襲**: 多ターン、中断可能、再開可能。isAreaSafeとprogressHungerを共通関数に抽出
- **GainExpを再利用**: 既存のスキル成長計算式のBaseExpをefficiencyで調整するだけ
- **読了時に本を消費**: 無限に経験値を得られる悪用を防ぐ
- **読書中の空腹進行**: 安全な場所でのノーリスク読書を防ぐ
- **ラン内完結モデルとの整合**: 毎ランリセットを前提に、基本的な本が約20ターンで読了できるよう工数を200〜400に設定

### 採用しなかった方法
- **Consumableとして実装**: 本は読了まで繰り返し使い、読了時に消費される。Consumableの1回使用とは異なる
- **専用のスキル成長計算式**: GainExpの仕組みで十分。BaseExpを倍率調整するだけで要件を満たせる
- **効果をフラットなフィールドで表現**: SkillIDとMaxLevelを全部Bookの直接フィールドにする方法。効果の組み合わせルールが型で表現できない
- **BookEffectインターフェース（マーカーメソッド方式）**: deadcodeツールが未使用と検出するため不採用
- **`any` 型で効果を表現**: 型安全性が低く、実行時型アサーションが必要になる
- **Type文字列による種類判定**: TOMLの構造と対応が不明瞭。ネスト構造のほうが明確
- **即時適用 + ペナルティ方式**: 本という形式の楽しさ（タイトル、概要、進捗）が失われる
- **ターン数固定の読書速度**: 能力値が活かせない。工数ベースにすることで成長の実感を得られる
