# 本システム

## 背景

知識を得る手段として「本」を追加する。本の効果は多様で、スキル経験値の獲得だけでなく、新しいレシピの習得や「やる気」の向上など、本によって読了時の効果が異なる。

本でしか伸ばせないスキルもあり（探索、暗視、各種耐性など）、戦闘以外のスキル成長手段として重要な位置づけになる。本なしで上げられるスキルは、練習で上げてから本を読むか、より易しい本を探すことになる。

要件:
- 読み通すには複数ターンが必要で、ターン数は本によって異なる
- 本の効果は種類によって異なる
  - スキル経験値を毎ターン獲得する（レベル適性あり）
  - 新しいレシピを習得する（読了時）
  - 「やる気」を増やす（読了時）
- スキル本にはレベルがあり、プレイヤーのレベルとの差で効率が変わる
  - 低すぎると得るものが少ない
  - 高すぎると難しくて理解できない
- 危険な状態（近くに敵）になると読書は自動中断される
- プレイヤーが任意のタイミングで読書を手動中断できる
- 読書中はターンが高速で進むが、中断操作を受け付けられる速度にする
- 読了状態は本に表示する
- 本ごとにタイトルや概要が異なる

## 設計

### Book コンポーネント (`internal/components/book.go`)

```go
// Book は読書可能な本のコンポーネント
type Book struct {
    Chapters     int        // 総章数。読了に必要なターン数
    ChaptersRead int        // 読了した章数。中断しても保持される
    Effect       BookEffect // 読書の効果
}

// BookEffect は本の効果を表すインターフェース
type BookEffect interface {
    bookEffect() // マーカーメソッド
}

// SkillBookEffect はスキル経験値を毎ターン獲得する効果
type SkillBookEffect struct {
    SkillID SkillID // 対応スキル
    Level   int     // 本のスキルレベル。このレベル前後が最も効果的
}

// RecipeBookEffect は読了時に新しいレシピを習得する効果
type RecipeBookEffect struct {
    RecipeName string // 習得するレシピ名
}

// MoraleBookEffect は読了時にやる気を増やす効果（将来拡張）
// type MoraleBookEffect struct {
//     Amount int // やる気の増加量
// }
```

- `Chapters`: 読了に必要なターン数。雑誌（20〜30）、マニュアル（50〜80）、専門書（100+）
- `ChaptersRead`: 読了すると効果を得られなくなる
- `Effect`: 本ごとに異なる効果。インターフェースでバリアントを表現する

EntitySpec, Components に `Book` を追加する。`save:"true"` で永続化する。

### 効果の適用タイミング

| 効果 | 毎ターン | 読了時 |
|---|---|---|
| SkillBookEffect | スキル経験値を獲得 | - |
| RecipeBookEffect | - | レシピを習得 |
| MoraleBookEffect（将来） | - | やる気を増加 |

### 経験値効率（SkillBookEffect のみ）

プレイヤーのスキルレベルと本のレベルの差に基づく。

```
diff = bookLevel - playerLevel（正=本が難しい）

diff > 5  → 0%   難しすぎて理解できない
diff = 5  → 50%
diff = 0  → 100% 最適
diff = -5 → 10%  易しすぎて得るものがない
diff < -5 → 10%

難しい側: efficiency = 100 - diff*10
易しい側: efficiency = 100 + diff*18
```

既存の `skill.GainExp` を再利用し、`GrowthConfig.BaseExp` を efficiency で調整する。

### ReadActivity (`internal/activity/read.go`)

RestActivityと同じパターンの多ターンアクティビティ。

```
Info:
  Name:          "読書"
  Interruptible: true   // プレイヤーが任意に中断できる
  Resumable:     true   // 中断後に再開できる

Validate:
  - comp.Target（本のエンティティ）が設定されていること
  - 対象がBookコンポーネントを持つこと
  - 本が読了済みでないこと
  - 周囲に敵がいないこと

DoTurn:
  1. 安全性チェック → 敵がいれば自動中断
  2. ChaptersRead++
  3. Effectの型に応じた毎ターン処理
     - SkillBookEffect: 経験値効率を計算して GainExp を呼ぶ
     - その他: 何もしない（読了時に効果を適用）
  4. スキルアップしたら CharModifiers 再計算
  5. ChaptersRead >= Chapters なら Complete

Finish:
  - Effectの型に応じた読了時処理
    - SkillBookEffect: 読了メッセージ
    - RecipeBookEffect: レシピを習得してメッセージ

Canceled:
  - 中断メッセージ。ChaptersReadはそのまま保持（次回再開可能）

ターン進行:
  - 読書中はターンが高速に自動進行する
  - ただしプレイヤーが中断操作を入力できる速度を維持する
  - 敵接近時は自動中断するため、高速進行でも安全性は保たれる
```

## コメント

### 採用した方法
- **BookEffectインターフェースで効果を多態化**: スキル経験値、レシピ習得、やる気向上を統一的に扱える。新しい効果の追加も容易
- **Bookを独立コンポーネントにする**: Item+Bookの組み合わせで表現。読了進捗がエンティティごとに保持される
- **RestActivityのパターンを踏襲**: 多ターン、中断可能、再開可能。isSafeのロジックを共通化できる
- **GainExpを再利用**: 既存のスキル成長計算式のBaseExpをefficiencyで調整するだけ
- **毎ターン効果と読了時効果の分離**: スキル経験値は読書中に徐々に得られるが、レシピやモチベーションは読了して初めて効果を発揮する。これはゲーム体験として自然

### 採用しなかった方法
- **Consumableとして実装**: 本は消費されず読了後も所持し続ける。Consumableの「使ったらなくなる」とは異なる
- **専用のスキル成長計算式**: GainExpの仕組みで十分。BaseExpを倍率調整するだけで要件を満たせる
- **読書速度をスキルや能力値で変動**: 複雑になる割にゲーム体験への貢献が小さい。Chaptersの値で本ごとに調整すれば十分
- **効果をフラットなフィールドで表現**: SkillID, RecipeName, MoraleAmount を全部Bookの直接フィールドにする方法。シンプルだが、効果の組み合わせルール（SkillIDがあるならLevelも必須、など）が型で表現できない。インターフェースにすることで各効果に必要なフィールドが明確になる
