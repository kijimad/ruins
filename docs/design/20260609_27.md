# Propのランダム配置

## 背景

現在Propはパレットで静的に配置される。マップ文字に対して固定のProp IDが割り当てられるため、同じテンプレートからは常に同じ配置のマップが生成される。ゲーム体験の多様性のためにPropのランダム配置が必要である。

Cataclysm-DDAでは以下の2つの方法でランダム配置を実現している。

1. パレット内でのランダム化: 1文字に対して重み付きで複数候補を割り当てる
2. `place_furniture`: 座標範囲+確率+繰り返し回数で事後的に配置する

## 設計

### 方針の比較

| 案 | 概要 | メリット | デメリット |
|---|---|---|---|
| A: パレット拡張 | パレットのpropsを重み付きリストにする | テンプレート作者が直感的に制御できる | マップ文字ごとの固定位置のみ |
| B: PropPlanner新設 | ItemPlannerと同様のPlannerを作る | 部屋全体にランダム散布できる | パレットと二重管理になる |
| C: A+B両方 | パレットで基本配置、Plannerで追加散布 | 最も柔軟 | 複雑 |

### 推奨: 案B（PropPlanner新設）

理由:

- ItemPlannerの実績あるパターンをそのまま踏襲でき、設計がシンプルである
- パレットのpropsは「ドア」のような確定配置に使い続ければよい
- ランダム家具（机、棚、椅子など）は部屋単位で散布する方が自然である
- CDDAのplace_furnitureも座標範囲ベースで同じ発想である

### アーキテクチャ

```
┌─────────────┐
│ mapplanner  │
│   Plan()    │
│  ┌────────┐ │    1. 部屋ごとにPropSourceを重みで選択
│  │ Prop   │ │    2. Collection/Distribution で候補を決定
│  │Planner │ │    3. 部屋内の空きfloorタイルにランダム配置
│  └────────┘ │    4. MetaPlan.Props に追加
└─────────────┘
```

PropPlannerはItemPlannerと同じパイプラインに組み込む。Plan()内のPlannerChain実行後、HostileNPCPlannerやItemPlannerと同列で呼び出す。

### データ構造

```go
// PropPlanner は部屋に家具をランダム配置する
type PropPlanner struct{}

// PropSource は配置候補グループを定義する
type PropSource struct {
    Weight  float64
    Subtype ItemGroupType // Collection / Distribution を再利用
    Entries []PropSpawnEntry
}

// PropSpawnEntry はPropの個別候補を定義する
type PropSpawnEntry struct {
    Name   string  // Prop ID（例: "desk", "shelf", "chair"）
    Weight float64 // 選択重み
}
```

### 配置ロジック

ItemPlannerを踏襲する。

1. 部屋ごとに `basePropCount + rand(0, N)` 個のPropを配置する
2. PropSourceを重みで選択する。Collectionなら全エントリ、Distributionなら1つを選ぶ
3. 部屋内の空きfloorタイルからランダムに選択する
4. 既存のProp/NPC/Portalと衝突しないか確認する
5. `MetaPlan.Props` に `PropsSpec` を追加する

### Prop定義データの例

```go
var propSources = []PropSource{
    {
        Weight:  50,
        Subtype: Distribution,
        Entries: []PropSpawnEntry{
            {Name: "desk", Weight: 60},
            {Name: "chair", Weight: 40},
        },
    },
    {
        Weight:  30,
        Subtype: Collection,
        Entries: []PropSpawnEntry{
            {Name: "shelf", Weight: 70},
            {Name: "crate", Weight: 30},
        },
    },
}
```

### 既存システムとの関係

- パレットのpropsは確定配置（ドア、階段など構造的なProp）に引き続き使用する
- PropPlannerはランダム家具（机、棚、椅子、装飾など）を担当する
- 両者の衝突はMetaPlan.Props内で重複座標チェックにより回避する

## 進捗

- 不採用。Propのランダム散布は配置が不自然になるため、すべてテンプレートベースで配置する方針に変更した

## コメント

- PropSourceの定義場所はItemPlannerに合わせてGoコードにハードコードで開始する。将来TOML化する余地はある
- 部屋タイプとの紐付けは初期実装では全部屋共通とし、必要に応じて部屋テンプレート別のPropSource選択を追加する
