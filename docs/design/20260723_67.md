# オーバーワールドの縦チャンク数を任意数化する

## 背景

現状のオーバーワールド帯は縦1行である。`Band` は東西方向の `k` チャンクだけを持ち、南北はストリーミング
しない横ストリップ。`ChunkGen(chunkIndex, offsetX)` は X の1次元、`ChunkSeed(runSeed, chunkIndex)` も
1次元、`consts.AbsTileX` はあるが `AbsTileY` は無い。

[20260723_66.md](./20260723_66.md) の街生成設計は、縦チャンク数の可変化を前提に「街判定を2次元座標で
前方互換に設計する」と決めた。その帯インフラ側の本体が本ドキュメントである。縦チャンク行数を任意数
`Rows` にし、プレイヤーが南北に分かれた複数レーンから経路を選べるようにする。1・3・5 と増やせる。

これは TODO 設計。実装は段階的に行い、`Rows=1` で現状と挙動を同値に保ったまま広げる。

## 前提の再確認

現状の1行帯の実装事実を押さえる。

- **Band**。`chunkW`・`k`(X方向のチャンク数)・`eastIndex` の3つだけ。縦の概念が無い。
- **ChunkGen**。`func(chunkIndex consts.Chunk, offsetX consts.Tile) error`。X の1次元。
- **generateBandChunks**。`for i := range k` を `offsetY=0` に置く。`Level = {TileWidth: band.Width(), TileHeight: chunkH}`。
- **ShiftEast/West**。`RemoveEntitiesInXRange(端X範囲)` で端チャンクを破棄、`TranslateAllEntities(±chunkW, 0)`、
  `rebaseCoordMaps`、`gen(新端)`。すべて X のみ。
- **SeamlessBand の serde**。`Active`・`RunSeed`・`EastIndex`・`ChunkW`・`ChunkH`・`K`・`Front`。
- **RecalcSeamAutotile**。`func(world, boundaryX consts.Tile)`。X 境界のみ。
- **前線**。`SeamlessFront` は X 一軸で東進し、全高を貫く。位置は `GameTime.TotalTurns` から決定的に導出。
- タイル系システム、描画・視界・座標索引・移動は既に `Coord{X, Y}` で2次元。**1次元なのは帯のストリーミングと
  生成・シード導出だけ**。本作業はそこを2次元化する。

## 要件

- 縦チャンク行数 `Rows` を任意数にできる。マスタ設定で持つ。
- 東西は従来どおり無限ストリーミング。**南北は `Rows` に有界でストリーミングしない**。全域2次元ストリーミングは
  しない。
- **`Rows=1` は現状と挙動同値**。回帰を防ぐ。旧セーブは `Rows=1` として読む後方互換。
- `ChunkGen` が2次元 `(cx, cy, offsetX, offsetY)` を受け、地形を Y スロットへ配置できる。
- 前線は X 一軸のまま全行を貫く。行ごとに地形が違うことでレーンの難易度差を生む。
- 街生成、doc 66 が要求する `IsTownChunk(ChunkCoord)` と `ChunkSeed2D` を使えるだけの基盤を提供する。
- serde が `Rows` を保存・復元する。

## 設計

### 1. Band に行数と高さを持たせる

`Band` に `rows` を足す。縦スロットへ配置するため `chunkH` も Band が持つ。マップ寸法は
`k*chunkW × rows*chunkH`。

```go
// before
type Band struct {
	chunkW    consts.Tile
	k         consts.Chunk // X方向のチャンク数
	eastIndex consts.Chunk
}

// after
type Band struct {
	chunkW    consts.Tile
	chunkH    consts.Tile  // 縦スロット配置に要る。従来は driver が別途保持していた
	k         consts.Chunk // X方向のチャンク数
	rows      consts.Chunk // Y方向のチャンク数。1 で現状と同値
	eastIndex consts.Chunk
}

func (b *Band) Rows() consts.Chunk  { return b.rows }
func (b *Band) Width() consts.Tile  { return b.k.Tiles(b.chunkW) }    // 既存
func (b *Band) Height() consts.Tile { return b.rows.Tiles(b.chunkH) } // 追加。高さ = rows*chunkH
```

`NewBand`・`NewBandAt` の引数に `chunkH`・`rows` を足す。`rows=1` を既定にすれば呼び出し側の移行が楽。

座標系は X だけ絶対軸を持つ。X は帯シフトで無限に伸びるため `AbsTileX` が要るが、Y は `[0, rows*chunkH)`
に有界でストリーミングしないので、Y は帯ローカルのタイル座標で足りる。**`AbsTileY` は追加しない**。`offsetY`
はこの有界ローカル Y への配置オフセット。

### 2. ChunkGen を2次元へ

```go
// before
type ChunkGen func(chunkIndex consts.Chunk, offsetX consts.Tile) error

// after
type ChunkGen func(c ChunkCoord, offsetX, offsetY consts.Tile) error

// ChunkCoord は X が絶対で東西無限、Y は [0, Rows) に有界。doc 66 と同一の型
type ChunkCoord struct{ X, Y consts.Chunk }
```

`generateBandChunks` は2重ループになる。

```go
func (dr *Driver) generateBandChunks(world w.World) error {
	query.EnsureStageField(world, gc.NewOverworldStage()).Level =
		gc.Level{TileWidth: dr.band.Width(), TileHeight: dr.band.Height()} // 高さ = rows*chunkH
	for cy := range dr.band.Rows() {
		for i := range dr.band.K() {
			c := ChunkCoord{X: dr.band.EastIndex() + i, Y: cy}
			if err := dr.gen(c, i.Tiles(chunkW), cy.Tiles(chunkH)); err != nil {
				return fmt.Errorf("チャンク生成失敗 (x=%d, y=%d): %w", c.X, c.Y, err)
			}
		}
	}
	return nil
}
```

### 3. シフトは X のみ、列すなわち全行を単位にする

`ShiftEast`/`ShiftWest` は端の X スロットの**全行**を破棄・生成する。南北移動はロード済み帯の中なので
破棄・生成を伴わない。これにより doc 66 課題1の破棄・再生成の寿命は X シフトだけに掛かる。

- `RemoveEntitiesInXRange(端X範囲)` は既に該当 X 範囲の全 Y を消すので、列破棄にそのまま流用できる。
- `TranslateAllEntities(±chunkW, 0)` は X のみ。Y は据え置き。
- 新しい端列は `rows` 行を生成する。

```go
// ShiftEast の東端生成部
newX := b.eastIndex + b.k
offsetX := (b.k - 1).Tiles(b.chunkW)
for cy := range b.rows {
	if err := gen(ChunkCoord{X: newX, Y: cy}, offsetX, cy.Tiles(b.chunkH)); err != nil {
		return err
	}
}
```

### 4. ChunkSeed2D

地形も街判定も、2次元シードに一本化する。

```go
// ChunkSeed2D は runSeed と (cx, cy) から決定的なシードを導く。既存 ChunkSeed の2次元版。
// cx と cy を別々の奇数定数で混ぜてから splitmix64 で撹拌する。
func ChunkSeed2D(runSeed uint64, cx, cy consts.Chunk) uint64
```

1行時の `ChunkSeed(runSeed, chunkIndex)` は `ChunkSeed2D(runSeed, cx, 0)` へ置換する。**後方互換は
保証しない**。`Rows` を増やす移行は世界形状そのものの変更なので、全チャンク再生成を許容する。doc 66 追記3の
判断に従う。

### 5. 縦境界の継ぎ目

現状 `RecalcSeamAutotile(world, boundaryX)` は X 境界のみ。行が増えると縦に隣接するチャンク間の Y 境界も
継ぎ目処理が要る。Y 境界版を足す。

```go
func RecalcSeamAutotile(world w.World, boundaryX consts.Tile)                // 現状。X 境界
func RecalcSeamAutotileY(world w.World, boundaryY, totalHeight consts.Tile)  // 追加。Y 境界。totalHeight=rows*chunkH
```

チャンク生成後、X 両境界に加え Y 両境界も再計算する。Y 版は `totalHeight=rows*chunkH` を受け取り、
`boundaryY=0` と `boundaryY=totalHeight` すなわちマップ上下端では隣チャンクが無いので自己スキップする。
X 版が `offsetX` と `offsetX+chunkW` を渡すだけで足りるのに対し、Y 版は端判定に全体高さが要るため引数で
渡す。帯端では現状同様に自己スキップさせ、無条件に呼べるようにする。

### 6. 前線は X 一軸のまま

`SeamlessFront` は X で東進し全行を貫く。`Rows` 化で変更不要。`ColdWidth`・`EastAbsX` は X 軸のまま。
行ごとに地形が違うので寒さの当たり方は変わるが、前線ロジック自体は不変。レーンは並走でゴールが東で
共通だから、前線も X 一軸で足りる。

### 7. serde に Rows

```go
type SeamlessBand struct {
	Active    bool
	RunSeed   uint64
	EastIndex consts.Chunk
	ChunkW    consts.Tile
	ChunkH    consts.Tile
	K         consts.Chunk
	Rows      consts.Chunk // 追加。0 や未設定は 1 として読む後方互換
	Front     SeamlessFront
}
```

`restoreFromSave` は `Rows` を読み、`NewBandAt(chunkW, chunkH, k, rows, eastIndex)` で再構築する。旧セーブは
`Rows=0` を `1` へ正規化して読む。

新規保存側の `startNewBand` も、`sb.K` と並べて `sb.Rows = dr.band.Rows()` を書く。ここを書き忘れると
保存直後のロードで `Rows=0` になり、正規化で `1` に落ちて `Rows>1` が失われる。書き込みは `sb.K` のすぐ隣に
置き、レビュー観点でも対にして見落とさないようにする。

## データ定義

上記の struct 前後がすべて。骨子は「1次元の帯・生成・シード導出に Y を足す」ことで、タイル系はすでに
2次元なので触らない。

## 採用しなかった方法

- **全域2次元ストリーミング、Y も無限**。南北も無限に生成・破棄する。プレイヤーはレーンを選べれば足り、
  無限縦スクロールは要らない。座標系と寿命管理が過剰に複雑化する。却下。
- **縦もシフトする**。プレイヤーが南北端に達したら縦シフトする。`Rows` 有界なら不要で、X と Y の二重
  シフトは座標変換とリベースを著しく複雑にする。Y は固定窓に留める。却下。
- **`ChunkGen` シグネチャを据え置き内部で Y を暗黙計算**。呼び出し側が `offsetY` を制御できず、2次元配置の
  意図が型に出ない。誤配置を型で防げない。明示引数にする。
- **`Rows` を固定 3 や 5 で決め打ち**。任意数化の要件に反する。マスタ設定で持ち、テストは任意行数を組む。
- **1行時の `ChunkSeed` 値を後方互換で保つ**。doc 66 追記3の判断どおり保証しない。`Rows` 移行は世界形状の
  変更なので全再生成を許容し、シード定数の自由度を縛らない。

## コメント

- 大半のタイル系は既に2次元なので、本作業の実質は「帯のストリーミングと生成・シード導出の2次元化」に絞られる。
  影響範囲は `worldstream`・`overworld` にほぼ閉じる。
- `Rows=1` 同値を最初に固定し、常に緑を保って `Rows>1` を足す。doc 66 と同じ「代替を入れてから広げる」方針。
- 前線が X 一軸で済むのはレーンが並走しゴールが東で共通だから。もし縦にもゴールや目的地を分けるなら前線設計を
  見直す。今は不要。
- doc 66 の街生成は本基盤の上に乗る。`ChunkSeed2D` と `ChunkCoord` は両ドキュメントで同一の型・関数を指す。
  実装順序は本ドキュメントの基盤を先、または `ChunkSeed2D` だけ先行させ街判定から使う。
- `(chunkX, chunkZ)` と seed からチャンクを決定的生成する `ChunkSeed2D` の方式は、Minecraft のチャンク生成と
  同型で実績がある。詳細は doc 66 追記(2026-07-24)。CDDA のグローバル乱数依存を避けた本方針の裏付けになる。

## 実行計画

各段は単独で `make check` 緑を保つ。**`Rows=1` と `cy=0` を現状同値に保つ**のが全段の不変条件で、これが
大改造を小さな緑の積み重ねに分解する退避路になる。

推奨順序は「B0 → doc 66 で街生成を `Rows=1` で完成 → 本計画 B1〜B8 で `Rows>1` へ一般化」。

### B0. 共有前提

- **ファイル**: `internal/overworld/chunkgen.go`(or 新規)
- **実装**: `ChunkCoord{X, Y consts.Chunk}`、`ChunkSeed2D(runSeed, cx, cy)` を splitmix64 で。`cx`・`cy` を
  別々の奇数定数で混ぜる。既存 `ChunkSeed` は当面残す。
- **テスト**: 転置 `(x,y)≠(y,x)`、隣接で散る、同一入力同一出力。
- **注**: doc 66 の街生成がこの `ChunkSeed2D`/`ChunkCoord` を共有する。後方互換は保証しない。

### B1. Band に `chunkH`・`rows`、アクセサ

- **ファイル**: `internal/worldstream/band.go`、呼び出し元 driver
- **実装**: `chunkH`・`rows` を追加。`NewBand`/`NewBandAt` 引数拡張(`rows=1` 既定)。`Rows() consts.Chunk`・
  `Height() consts.Tile = rows.Tiles(chunkH)`。
- **緑**: `rows=1` を渡すだけで `Height()==chunkH`。
- **テスト**: `rows=1` で `Width`/`Height` が現状値、既存 band テスト全緑。

### B2. `ChunkGen` を2次元化、`ChunkSeed2D` へ置換

- **ファイル**: `internal/overworld/chunkgen.go`、`internal/worldstream/band.go`(gen 呼び出し)
- **実装**: `ChunkGen` を `(ChunkCoord, offsetX, offsetY)` へ。地形の `ChunkSeed(runSeed, idx)` を
  `ChunkSeed2D(runSeed, cx, 0)` に置換。`AbsTileY` は追加しない。
- **緑**: `cy=0`・`offsetY=0` 固定で現状同値。
- **テスト**: `Rows=1` の生成結果が置換前と同値(golden 維持)。

### B3. `generateBandChunks` 2重ループ、Level 高さ

- **ファイル**: `internal/overworld/driver.go`
- **実装**: `for cy := range rows { for i := range k {...} }`。`Level.TileHeight = band.Height()`。
- **緑**: `rows=1` で1重ループ相当。
- **テスト**: `rows=1` 同値、`rows=3` で3行生成。

### B4. シフトを列(全行)単位に

- **ファイル**: `internal/worldstream/band.go`
- **実装**: 端破棄は `RemoveEntitiesInXRange`(全 Y を消すので流用可)、`TranslateAllEntities(±chunkW, 0)`、
  新端列は `for cy := range rows { gen(...) }`。X のみ。
- **緑**: `rows=1` で現状の1チャンク破棄・生成と同値。
- **テスト**: `rows=1` 同値、`rows=3` で東シフト後に3行復元、南北移動では破棄が起きない。

### B5. 縦境界の継ぎ目

- **ファイル**: `internal/overworld/seam.go`、chunkgen 呼び出し
- **実装**: `RecalcSeamAutotileY(world, boundaryY, totalHeight=rows*chunkH)`。生成後に X・Y 両境界を再計算。
  上下端は自己スキップ。
- **緑**: `rows=1` は Y 境界が無く自己スキップ、現状同値。
- **テスト**: `rows=3` で行間の継ぎ目が消える。

### B6. serde に `Rows`

- **ファイル**: `internal/components/dungeon.go`、`internal/overworld/driver.go`
- **実装**: `SeamlessBand.Rows` 追加。`startNewBand` で `sb.K` の隣に `sb.Rows = band.Rows()`。
  `restoreFromSave` で読み `Rows=0→1` 正規化。
- **緑**: 旧セーブは `Rows=1` として復元。
- **テスト**: `rows=3` を保存→別 world ロードで3行復元、旧セーブ互換。

### B7. マスタに行数

- **ファイル**: `internal/dungeon/definition.go`
- **実装**: `OverworldKind.BandShape` に `rows` を足す。本番は 1、テストは任意行数。
- **緑**: 既定 1 で現状同値。
- **テスト**: `NewOverworldKind` で任意 `rows` を組める。

### B8. `Rows>1` の通し

- 新規ゲーム→南北移動→東シフト→セーブ/ロード→継ぎ目、を `rows=3` で通す統合テスト。doc 66 の街が複数
  レーンに散ることも確認する。
