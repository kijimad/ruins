# 収納システム

## 背景

ダンジョン内に宝箱・棚・ロッカーなどの収納propをランダムスポーンし、プレイヤーがアイテムを出し入れできるようにする。7 Days to Die のように収納をスカベンジする楽しみを提供する。

## 設計

### コンポーネント

```go
// Storage は収納機能を持つPropに付与するコンポーネント。
// 格納アイテムはLocationInStorageで参照する。
// 容量はプレイヤーのCarryWeightと同じくkg単位の重量で制限する
type Storage struct {
    MaxWeight float64 // 最大格納重量（kg）
}
```

```go
// LocationInStorage は収納内のアイテム位置を表す
type LocationInStorage struct {
    Owner ecs.Entity // 収納Propのエンティティ
}
```

`LocationInStorage`は既存の`LocationInBackpack`/`LocationEquipped`/`LocationOnField`と排他的に管理する（`setLocation()`で保証済み）。

### Interactable拡張

```go
// StorageInteraction は収納の相互作用
type StorageInteraction struct{}

func (s StorageInteraction) Config() InteractionConfig {
    return InteractionConfig{
        ActivationRange: ActivationRangeAdjacent,  // 隣接タイルから操作
        ActivationWay:   ActivationWayManual,      // Enterキーで開く
    }
}
```

### UI（StorageMenuState）

タブで「取得」と「収納」を切り替える。常に左側が操作対象で、Enterで左のアイテムに対してアクションを実行する。既存の`tabmenu`パッケージを使用する。

#### 「取得」タブ（収納から取り出す）

左に収納の中身、右にバックパックを表示する。Enterで左のアイテムをバックパックに移動する。

```
┌──────────────────────────────────────┐
│ 古びた木箱      [5.2 / 20.0 kg]     │
│ [取得]  収納                         │
├──────────────┬───────────────────────┤
│ 収納の中身     │ バックパック          │
│               │                     │
│ > 回復薬       │   鉄の剣             │
│   松明         │   革の盾             │
│   矢 x5       │   回復薬 x3          │
│               │   松明 x2            │
├──────────────┴───────────────────────┤
│ Enter:取り出す  ←→:タブ切替  Esc:閉じる│
└──────────────────────────────────────┘
```

#### 「収納」タブ（バックパックから入れる）

左にバックパック、右に収納の中身を表示する。Enterで左のアイテムを収納に移動する。

```
┌──────────────────────────────────────┐
│ 古びた木箱      [5.2 / 20.0 kg]     │
│  取得  [収納]                        │
├──────────────┬───────────────────────┤
│ バックパック   │ 収納の中身            │
│               │                     │
│ > 鉄の剣       │   回復薬             │
│   革の盾       │   松明               │
│   回復薬 x3    │   矢 x5             │
│   松明 x2      │                     │
├──────────────┴───────────────────────┤
│ Enter:収納する  ←→:タブ切替  Esc:閉じる│
└──────────────────────────────────────┘
```

- 左側が常に操作対象。カーソルは左カラムのみ動く
- 右側は参照用の表示のみ
- タブ切替で左右の内容が入れ替わり、アクションの方向が変わる
- 重量超過時はEnterで収納不可のフィードバックを表示する

### マップスポーン

`mapspawner/spawner.go`のPropスポーン処理で`Storage`付きpropを生成する。初期アイテムの格納は既存の`itemTables`を再利用する。`StorageRaw.lootTableName`で`ItemTable`名を参照する。

```toml
# 既存のitemTablesで収納の初期アイテムを定義する
[[itemTables]]
name = "wooden_chest_loot"
[[itemTables.entries]]
  groupName = "回復アイテム"
  weight = 10
  minDepth = 1
  maxDepth = 30
[[itemTables.entries]]
  groupName = "消耗品"
  weight = 5
  minDepth = 1
  maxDepth = 30
```

`mapspawner`は収納propスポーン時に`lootTableName`で指定された`ItemTable`からアイテムを抽選し、`MoveToStorage`で格納する。

### 採用しなかった方法

| 方法 | 不採用の理由 |
|------|-------------|
| pocket/contents多層構造 | ruinsのアイテムシステムは単純なので過剰。重量制限のみで十分 |
| スロット数（個数）による容量制限 | プレイヤーのCarryWeightが重量ベースなので、収納も重量で統一した方が一貫性がある |
| 収納を破壊して中身を取得する方式 | UIで出し入れする方が操作性が高い。破壊時のドロップは将来的に追加可能 |
| LocationInStorageを使わずStorageに直接Entity配列を持つ方式 | 既存のECSロケーション管理（setLocation排他制御）と整合しない |

## コメント

- 収納propのスプライトはassets追加が必要（wooden_chest, metal_locker等）
- 収納の中身はダンジョン深度に応じてルートテーブルを変えると探索の楽しみが増す
- 将来的に鍵付き収納（KeyItem必要）や壊れる収納（HP付き）も検討できる
- 街のNPCショップも内部的にStorageとして実装を統一できる可能性がある
- `GetStorageCurrentWeight`はメニュー表示中に毎フレーム呼ばれるため、アイテム数が増えた場合はStorage側にCurrentWeightをキャッシュしてダーティフラグで更新する方式を検討する。プレイヤーのCarryWeightが`InventoryChangedSystem`でフラグベースに更新される既存パターンと揃えられる
- Storage propが破壊される場合、格納アイテムの孤立を防ぐため破壊前にフィールドへドロップする処理が必要になる
