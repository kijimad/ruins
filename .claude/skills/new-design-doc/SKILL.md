# 設計ドキュメント作成

## 手順

1. **採番する**: `docs/design/YYYYMMDD_NN.md`。日付は当日、`NN` は同日既存の最大 + 1（2桁ゼロ埋め）。
   - `ls docs/design/$(date +%Y%m%d)_*.md` で当日の既存を確認する。
2. **雛形を使う**: `docs/design/tmpl.md` を土台にする。
3. **必須セクションを埋める**: 背景 / 要件 / 設計 / 変更箇所 / 採用しなかった方法 / コメント。
   - **データ定義はコードで示す**。struct・型・コンポーネント・フィールドを伴う設計は、散文でなく Go のコードブロックで書き、データの形を一目でわかるようにする。変更を伴うなら変更前後の struct を並べる。振る舞いより「どんなデータを持つか」が設計の骨子なので、まず形を確定させる。
4. **変更箇所テーブル**にはファイルパスと変更内容を書く。
5. **末尾に `## 進捗`** を設け、変更箇所テーブルの各行に対応するタスクを `- [ ]` / `- [x]` で管理する。粒度が粗ければサブタスクに分解する。

## 注意

- ロジック・コメントは日本語で書く。
- データ定義はコードで示す。文章でフィールドを列挙するより、struct をそのまま書くほうが形が正確に伝わり、実装との齟齬も減る。
- 採用しなかった案も理由付きで残す（後から判断根拠を辿れるように）。
